## Thumb (16-bit) Instruction Set in ARM
The **Thumb instruction set** is a **16-bit compressed version** of the standard 32-bit ARM instruction set. It was introduced to improve **code density** (i.e., reduce program size) while maintaining reasonable performance.
## What is Thumb Instruction Set?
- A subset of the ARM instruction set using **16-bit instructions**
- Designed for **memory-constrained systems**
- Executes on ARM processors that support **Thumb mode**
- Provides **higher code density** (smaller programs)
## Why Thumb Mode?
| Feature                  | Benefit                            |
| ------------------------ | ---------------------------------- |
| 16-bit instructions      | Reduced memory usage               |
| Smaller code size        | Better cache utilization           |
| Lower power consumption  | Ideal for embedded/mobile systems  |
| Faster fetch from memory | Improved performance in some cases |
## ARM vs Thumb
| Feature          | ARM Mode      | Thumb Mode        |
| ---------------- | ------------- | ----------------- |
| Instruction size | 32-bit        | 16-bit            |
| Code density     | Lower         | Higher            |
| Performance      | Higher        | Slightly lower    |
| Register access  | All registers | Limited registers |
| Instruction set  | Rich          | Reduced           |
## Key Characteristics of Thumb Instructions
1. **Fixed 16-bit instruction length**
2. Uses **subset of ARM registers**
   - Mostly: R0–R7 (low registers)
3. Limited instruction functionality compared to ARM
4. No direct conditional execution (unlike ARM)
5. Requires **BX instruction** to switch between ARM and Thumb
## Thumb Register Usage
- **Low registers**: R0–R7 (frequently used)
- **High registers**: R8–R15 (limited access)
- PC (R15), SP (R13), LR (R14) still play special roles
## Example Instructions
```assembly
MOV R0, #5       ; Load immediate value
ADD R1, R0, #3   ; Add immediate
SUB R2, R1, R0   ; Subtract
LDR R3, [R0]     ; Load from memory
STR R3, [R1]     ; Store to memory
B label          ; Branch
```
## Switching Between ARM and Thumb
- Use **BX (Branch and Exchange)** instruction:
```assembly
BX R0   ; Switch mode based on LSB of R0
```
- If LSB = 1 → Thumb mode
- If LSB = 0 → ARM mode
## Thumb-2 Extension
- Later ARM processors introduced **Thumb-2**
- Combines **16-bit + 32-bit instructions**
- Offers:
  - Better performance
  - More functionality
  - Nearly full ARM capability
## Advantages
- Smaller code size
- Better memory efficiency
- Lower power consumption
- Ideal for embedded systems
## Disadvantages
- Limited instruction set
- Reduced register access
- Slightly lower performance than ARM
## Applications
- Mobile phones
- Embedded systems
- IoT devices
- Low-power microcontrollers
## Summary
The **Thumb instruction set** is a **compact and efficient version of ARM instructions**, mainly used where **memory and power are limited**. While it sacrifices some performance and flexibility, it greatly improves **code density**, making it ideal for embedded applications.

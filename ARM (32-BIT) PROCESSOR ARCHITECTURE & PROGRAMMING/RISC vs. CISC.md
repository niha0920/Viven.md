## What is RISC?
**RISC (Reduced Instruction Set Computer)** focuses on **simple, fast instructions**.
### Key Features:
- Simple and limited number of instructions
- Each instruction executes in **one clock cycle** (mostly)
- Fixed instruction length
- Uses **load/store architecture** (operations only on registers)
- Easier to pipeline
- More reliance on software (compiler)
### Examples:
- ARM architecture
- MIPS architecture
- SPARC architecture
## What is CISC?
**CISC (Complex Instruction Set Computer)** focuses on **powerful, complex instructions**.
### Key Features:
- Large number of instructions
- Instructions can take **multiple clock cycles**
- Variable instruction length
- Can perform complex operations in a single instruction
- Harder to pipeline
- Less reliance on software
### Examples:
- x86 architecture
- VAX architecture
## RISC vs CISC (Comparison Table)
| Feature             | RISC             | CISC                     |
| ------------------- | ---------------- | ------------------------ |
| Instruction Set     | Small & simple   | Large & complex          |
| Execution Time      | Faster (1 cycle) | Slower (multi-cycle)     |
| Instruction Length  | Fixed            | Variable                 |
| Memory Access       | Load/Store only  | Direct memory operations |
| Pipelining          | Easy             | Difficult                |
| Hardware Complexity | Simple           | Complex                  |
| Code Size           | Larger           | Smaller                  |
| Power Consumption   | Low              | Higher                   |
## Simple Understanding
- **RISC** → “Do simple things very fast”
- **CISC** → “Do complex things in fewer instructions”

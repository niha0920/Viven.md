The **ARM (32-bit) Instruction Set** is designed around the principles of **RISC (Reduced Instruction Set Computing)**—meaning instructions are simple, fast, and mostly execute in a single clock cycle.
## Overview of ARM 32-bit Instruction Set
ARM uses a **fixed 32-bit instruction length** (in ARM mode), which makes decoding simple and efficient.
### Key features:
- Load/Store architecture (memory accessed only via special instructions)
- Large register set (R0–R15)
- Conditional execution (almost every instruction can be conditionally executed)
- Orthogonal instruction design (flexible operand usage)
## Classification of ARM Instructions
### 1. Data Processing Instructions
These operate on registers (no direct memory access).
#### Examples:
- `ADD R1, R2, R3` → R1 = R2 + R3
- `SUB R4, R5, R6` → R4 = R5 − R6
- `MOV R0, R1` → Copy value
- `CMP R1, R2` → Compare (sets flags)
#### Features:
- Operate on registers only
- Update condition flags (N, Z, C, V)
- Flexible second operand (can include shifts)
## 2. Load/Store Instructions
Used to transfer data between memory and registers.
### Examples:
- `LDR R0, [R1]` → Load from memory
- `STR R2, [R3]` → Store to memory
### Variants:
- `LDRB / STRB` → Byte
- `LDRH / STRH` → Halfword
### Important:
- ARM does **not allow memory-to-memory operations**
## 3. Branch Instructions
Used for program control flow.
### Examples:
- `B label` → Unconditional branch
- `BL label` → Branch with link (function call)
- `BX R0` → Branch to address in register
### Special:
- Uses **Link Register (R14)** to store return address
## 4. Multiply Instructions
### Examples:
- `MUL R0, R1, R2` → R0 = R1 × R2
- `MLA R0, R1, R2, R3` → R0 = (R1 × R2) + R3
## 5. Software Interrupt (Exception) Instructions
### Example:
- `SWI 0x11` → Software interrupt
### Purpose:
- Used for system calls
- Switches processor to **Supervisor Mode**
## 6. Status Register Instructions
Used to access **CPSR (Current Program Status Register)**
- `MRS R0, CPSR` → Read status register
- `MSR CPSR, R0` → Write status register
## 7. Conditional Execution (Unique Feature)
Almost every ARM instruction can be conditionally executed.
### Example:
- `ADDNE R0, R1, R2` → Execute only if Z = 0 (Not Equal)
- `BEQ label` → Branch if equal
### Common condition codes:
- EQ (Equal)
- NE (Not Equal)
- GT (Greater than)
- LT (Less than)
## General Instruction Format
```
<opcode>{cond}{S} Rd, Rn, Operand2
```
### Example:
```
ADDNE R0, R1, R2
```
- `ADD` → Operation
- `NE` → Condition
- `R0` → Destination
- `R1, R2` → Operands
## Key Characteristics
- Fixed 32-bit instruction size
- Conditional execution reduces branches
- Barrel shifter integrated with ALU
- Efficient pipeline execution
- Load/store architecture
## Simple Example Program
```assembly
MOV R0, #5
MOV R1, #10
ADD R2, R0, R1
STR R2, [R3]
```
### Operation:
- Load values into registers
- Perform addition
- Store result in memory
## Summary
The ARM 32-bit instruction set is:
- **Simple** (few instruction types)
- **Power-efficient**
- **Highly optimized for embedded systems**
- Based on **RISC principles**

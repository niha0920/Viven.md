# Socket concepts
**Socket concepts** are fundamental to network programming—they define how two systems communicate over a network using endpoints called sockets.
## What is a Socket?
A **socket** is an endpoint for communication between two machines. It allows processes to send and receive data across a network.
- Think of it like a **door** through which data enters and leaves.
- Each socket is uniquely identified by:
  - **IP address**
  - **Port number**
###
Example: `172.29.59.230 : 6000`
## Basic Components of a Socket
Every socket connection involves:
1. **IP Address** – Identifies the device
2. **Port Number** – Identifies the application/process
3. **Protocol** – Defines communication rules (TCP or UDP)
## Types of Sockets
### 1. TCP Socket (Stream Socket)
- Uses Transmission Control Protocol
- Connection-oriented
- Reliable (guarantees delivery)
- Data arrives in order
#### Used in:
- Web browsing
- File transfer
- Email
## 2. UDP Socket (Datagram Socket)
- Uses User Datagram Protocol
- Connectionless
- Faster but unreliable
- No guarantee of delivery/order
### Used in:
- Video streaming
- Online gaming
- VoIP

#### Server Side Steps
1. Create socket
2. Bind (attach IP + port)
3. Listen for connections
4. Accept connection
5. Receive & send data
6. Close socket
## Client Side Steps
1. Create socket
2. Connect to server
3. Send & receive data
4. Close socket
## Socket Lifecycle
1. Creation → `socket()`
2. Connection → `connect()` / `accept()`
3. Data Transfer → `send()` / `recv()`
4. Termination → `close()`
## Key Concepts
### Port Numbers
- Range: 0–65535
- Example:
  - 80 → HTTP
  - 443 → HTTPS
  - 6000 → Custom app (like your TCP experiment)
## Blocking vs Non-blocking Sockets
- **Blocking**: waits until operation completes
- **Non-blocking**: continues execution without waiting
## Full Duplex Communication
- Both client and server can send/receive data simultaneously
## Socket Address Structure
In programming (like C/Python), sockets use structures like:
```c
struct sockaddr_in {
    int sin_family;
    int sin_port;
    struct in_addr sin_addr;
};
```
## Real-Life Analogy
- **IP address** = House address
- **Port number** = Room number
- **Socket** = Door of the room
- **Protocol (TCP/UDP)** = Rules of communication
## Simple Python Example (TCP)
### Server
```c
import socket
server = socket.socket()
server.bind(("0.0.0.0", 6000))
server.listen(1)
conn, addr = server.accept()
print("Connected:", addr)
data = conn.recv(1024)
print("Received:", data.decode())
conn.send(b"Hello Client")
conn.close()
```
## Client
```c
import socket
client = socket.socket()
client.connect(("172.29.59.230", 6000))
client.send(b"Hello Server")
print(client.recv(1024).decode())
client.close()
```
## Summary
- Socket = communication endpoint
- Works with TCP or UDP
- Enables client-server communication
- Core concept in network programming

# Socket concepts
**Socket concepts** are fundamental to network programming—they define how two systems communicate over a network using endpoints called sockets.
## What is a Socket?
A **socket** is an endpoint for communication between two machines. It allows processes to send and receive data across a network.
- Think of it like a **door** through which data enters and leaves.
- Each socket is uniquely identified by:
  - **IP address**
  - **Port number**
###
Example: `172.29.59.230 : 6000`
## Basic Components of a Socket
Every socket connection involves:
1. **IP Address** – Identifies the device
2. **Port Number** – Identifies the application/process
3. **Protocol** – Defines communication rules (TCP or UDP)
## Types of Sockets
### 1. TCP Socket (Stream Socket)
- Uses Transmission Control Protocol
- Connection-oriented
- Reliable (guarantees delivery)
- Data arrives in order
#### Used in:
- Web browsing
- File transfer
- Email
## 2. UDP Socket (Datagram Socket)
- Uses User Datagram Protocol
- Connectionless
- Faster but unreliable
- No guarantee of delivery/order
### Used in:
- Video streaming
- Online gaming
- VoIP

## Key Concepts
### Port Numbers
- Range: 0–65535
- Example:
  - 80 → HTTP
  - 443 → HTTPS
  - 6000 → Custom app (like your TCP experiment)
## Blocking vs Non-blocking Sockets
- **Blocking**: waits until operation completes
- **Non-blocking**: continues execution without waiting
## Full Duplex Communication
- Both client and server can send/receive data simultaneously
## Socket Address Structure
In programming (like C/Python), sockets use structures like:
```c
struct sockaddr_in {
    int sin_family;
    int sin_port;
    struct in_addr sin_addr;
};
```
## Real-Life Analogy
- **IP address** = House address
- **Port number** = Room number
- **Socket** = Door of the room
- **Protocol (TCP/UDP)** = Rules of communication
## Simple Python Example (TCP)
### Server
```c
import socket
server = socket.socket()
server.bind(("0.0.0.0", 6000))
server.listen(1)
conn, addr = server.accept()
print("Connected:", addr)
data = conn.recv(1024)
print("Received:", data.decode())
conn.send(b"Hello Client")
conn.close()
```
## Client
```c
import socket
client = socket.socket()
client.connect(("172.29.59.230", 6000))
client.send(b"Hello Server")
print(client.recv(1024).decode())
client.close()
```
## Summary
- Socket = communication endpoint
- Works with TCP or UDP
- Enables client-server communication
- Core concept in network programming

# Socket API Interface
The **Socket API Interface** (often called the **Berkeley Sockets API**) is a set of functions that applications use to communicate over a network. It acts as a bridge between your program and the operating system’s networking stack.
## What is Socket API?
The Socket API provides system calls that allow programs to:
- Create sockets
- Establish connections
- Send and receive data
- Close connections
###
It is widely used in C, C++, Python, Java, etc.
## Key Socket API Functions
### 1. `socket()`
Creates a new socket.
```c
int sockfd = socket(domain, type, protocol);
```
- domain → Address family (e.g., IPv4 → AF_INET)
- type → Communication type (SOCK_STREAM / SOCK_DGRAM)
- protocol → Usually 0 (auto-select)
## 2. `bind()`
Assigns IP address and port to the socket (used by server).
```c
bind(sockfd, (struct sockaddr*)&addr, sizeof(addr));
```
## 3. `listen()`
Marks the socket as passive (ready to accept connections).
```c
listen(sockfd, backlog);
```
## 4. `accept()`
Accepts a connection request from a client.
```c
int new_sock = accept(sockfd, (struct sockaddr*)&client, &len);
```
## 5. `connect()`
Used by client to connect to server.
```c
connect(sockfd, (struct sockaddr*)&server, sizeof(server));
```
## 6. `send()` / `write()`
Sends data.
```c
send(sockfd, buffer, size, 0);
```
## 7. `recv()` / `read()`
Receives data.
```c
recv(sockfd, buffer, size, 0);
```
## 8. `close()`
Closes the socket.
```c
close(sockfd);
```

## Address Families (Domains)
- **AF_INET** → IPv4
- **AF_INET6** → IPv6
- **AF_UNIX** → Local communication (same machine)
## Socket Types
- **SOCK_STREAM** → Uses Transmission Control Protocol (reliable)
- **SOCK_DGRAM** → Uses User Datagram Protocol (fast, unreliable)
## Example: Simple TCP Server (C)
```c
#include <stdio.h>
#include <sys/socket.h>
#include <netinet/in.h>
int main() {
    int sockfd = socket(AF_INET, SOCK_STREAM, 0);
    struct sockaddr_in server;
    server.sin_family = AF_INET;
    server.sin_port = htons(6000);
    server.sin_addr.s_addr = INADDR_ANY;
    bind(sockfd, (struct sockaddr*)&server, sizeof(server));
    listen(sockfd, 5);
    int client = accept(sockfd, NULL, NULL);
    char buffer[100];
    recv(client, buffer, sizeof(buffer), 0);
    printf("Received: %s\n", buffer);
    send(client, "Hello Client", 12, 0);
    close(client);
    close(sockfd);
}
```
## Important Notes
- Socket API is **OS-level interface** (provided by Linux/Unix/Windows)
- Works with both **TCP and UDP**
- Uses **file descriptors** (in Unix-like systems)
- Supports **blocking and non-blocking modes**
## Summary
- Socket API = set of functions for network communication
- Core functions: `socket`, `bind`, `listen`, `accept`, `connect`, `send`, `recv`, `close`
- Enables client-server communication
- Used in real-world networking applications

## Definition
**Process Synchronization** is a technique used in an operating system to coordinate multiple processes or threads so that they can share resources and execute without conflicts or inconsistencies.

It ensures that shared data is accessed in a controlled manner and prevents problems such as data corruption, race conditions, and deadlocks.
### Simple Definition
When multiple processes try to access the same resource (CPU, memory, file, printer, variable, etc.) at the same time, synchronization helps them work properly without interfering with each other.
## Need for Process Synchronization
Synchronization is needed because many processes execute concurrently and may access shared resources.

Without synchronization:
* Data inconsistency occurs
* Race conditions happen
* Shared resources get corrupted
* Unexpected program behavior occurs
### Example
Suppose two processes are updating the same bank account balance.

Initial balance = ₹1000
* Process 1 adds ₹500
* Process 2 withdraws ₹200

Without synchronization:

Both processes may read the old value simultaneously and overwrite each other’s result.

Final balance may become incorrect.

With synchronization:

Processes access the balance one at a time and produce the correct result.
## Critical Section Problem
The most important concept in synchronization is the **Critical Section**.

A **Critical Section** is a part of a program where shared data/resources are accessed.

Example:
```c
balance = balance + 500;
```
If multiple processes execute this simultaneously, incorrect results may occur.
### Structure of Process Execution
```
Entry Section
Critical Section
Exit Section
Remainder Section
```
### Explanation
1. **Entry Section**
   Process requests permission to enter the critical section.
2. **Critical Section**
   Shared resource is accessed.
3. **Exit Section**
   Process releases the resource.
4. **Remainder Section**
   Remaining code executes.
## Requirements of Critical Section Problem
A synchronization solution must satisfy these three conditions:
### 1. Mutual Exclusion
Only one process can enter the critical section at a time.

Example:
If Process P1 is updating a file, Process P2 must wait.
### 2. Progress
If no process is inside the critical section, waiting processes should be allowed to enter without unnecessary delay.
### 3. Bounded Waiting
A process should not wait forever to enter the critical section.

Each process gets a fair chance.
## Race Condition
A **Race Condition** occurs when multiple processes access shared data simultaneously and the final result depends on execution order.

Example:

Shared variable:
```c
int count = 5;
```
Process P1:
```c
count++;
```
Process P2:
```c
count--;
```
Expected result = 5

But due to simultaneous execution, result may become 4 or 6.
## Synchronization Tools / Mechanisms
### 1. Mutex (Mutual Exclusion)
A mutex lock allows only one process/thread to access a resource.
```
lock()
Critical Section
unlock()
```
### 2. Semaphore
A semaphore is a synchronization variable used to control access to shared resources.

Two types:
#### Binary Semaphore
Works like a lock (0 or 1).
#### Counting Semaphore
Allows multiple resources.

Example:
Printer pool with 3 printers.
## Producer–Consumer Problem
A famous synchronization problem.
* Producer adds data into buffer
* Consumer removes data from buffer

Need synchronization to avoid:
* Buffer overflow
* Buffer underflow
## Readers–Writers Problem
Multiple readers can read simultaneously.

But writing must be exclusive.

Rules:
* Many readers → allowed
* One writer → exclusive access
## Dining Philosophers Problem
A classic synchronization problem.

Several philosophers share forks.

Problem:
If all pick one fork simultaneously → deadlock occurs.

Solution:
Control resource allocation.
## Busy Waiting
A process continuously checks whether it can enter the critical section.

Example:
```c
while(lock == 0);
```
This wastes CPU time.
## Advantages of Process Synchronization
* Prevents race conditions
* Maintains data consistency
* Prevents corruption of shared data
* Proper resource sharing
* Improves system reliability
## Disadvantages
* Increased complexity
* Deadlock possibility
* Starvation possibility
* Performance overhead
## Real-Time Example
ATM system:

Multiple users access the same bank database.

Synchronization ensures:
* Correct balance updates
* No transaction conflicts
* Data consistency

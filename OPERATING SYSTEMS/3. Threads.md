## Threads in Operating System
A **thread** is the **smallest unit of execution inside a process**. It is often called a **lightweight process** because multiple threads can exist within the same process and share resources.

In simple words:

> A process is a running program, and a thread is a task inside that process.

For example, in a web browser:
* One thread handles the user interface
* One thread downloads files
* One thread plays videos
* One thread manages tabs

All these threads run inside the same browser process.
### Definition
A thread is a **path of execution within a process**. Each thread has:
* Program Counter (PC)
* Register Set
* Stack Space

Threads within the same process **share**:
* Code section
* Data section
* Heap memory
* Open files and resources
### Structure of Threads
Each thread contains:
1. **Thread ID (TID)** – unique identification
2. **Program Counter (PC)** – next instruction to execute
3. **Registers** – CPU state information
4. **Stack** – local variables and function calls

Shared among threads:
* Memory
* Variables
* Files
* Process resources
### Types of Threads
#### 1. User-Level Threads (ULT)
* Managed by user-level libraries
* Kernel is unaware of these threads
* Faster creation and switching
* If one thread blocks, entire process may block

Example:
* POSIX Pthreads library
#### 2. Kernel-Level Threads (KLT)
* Managed directly by OS kernel
* Kernel schedules threads
* Better concurrency
* More overhead due to kernel involvement

Examples:
* Modern Linux, Windows threads
### Multithreading
Executing multiple threads simultaneously within a process is called **multithreading**.

Benefits:
* Faster execution
* Better CPU utilization
* Responsiveness
* Resource sharing
* Parallelism

Example:

A music app:
* One thread → play song
* One thread → download songs
* One thread → UI interaction
### Thread Life Cycle
A thread goes through different states:
1. **New** → Thread created
2. **Ready** → Waiting for CPU
3. **Running** → Executing instructions
4. **Blocked/Waiting** → Waiting for I/O/event
5. **Terminated** → Execution completed
### Difference Between Process and Thread
| Process                  | Thread                   |
| ------------------------ | ------------------------ |
| Heavyweight              | Lightweight              |
| Separate memory          | Shared memory            |
| Creation is slow         | Creation is fast         |
| Communication expensive  | Communication easy       |
| Context switching costly | Context switching faster |
### Advantages of Threads
* Improved performance
* Better responsiveness
* Efficient communication
* Resource sharing
* Parallel task execution
### Disadvantages of Threads
* Synchronization issues
* Data inconsistency
* Debugging difficulty
* Security risks due to shared memory
### Real-Time Example
In a chat application:
* One thread sends messages
* One thread receives messages
* One thread updates UI

So the application works smoothly without freezing.

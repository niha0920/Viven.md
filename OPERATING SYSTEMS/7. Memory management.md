## Memory Management in Operating System
### Definition
**Memory Management** is a function of an Operating System that manages the computer’s main memory (RAM). It controls how memory is allocated, used, and deallocated by processes.

In simple terms, memory management ensures that multiple programs can run efficiently without interfering with each other.
### Example
Suppose you open:
* Browser
* Music player
* Text editor

The OS allocates memory separately to each application so they can run simultaneously without crashing or overwriting one another.
## Objectives of Memory Management
1. **Efficient memory utilization** – Use RAM effectively.
2. **Allocation and deallocation** – Assign memory to processes and free it after execution.
3. **Protection** – Prevent one process from accessing another process’s memory.
4. **Sharing** – Allow processes to share memory when needed.
5. **Logical organization** – Organize programs in memory properly.
## Functions of Memory Management
### 1. Memory Allocation
Assigns memory to processes when they start execution.

**Example:**

If a process needs 200 MB RAM, the OS allocates that space.
### 2. Memory Deallocation
Frees memory after a process terminates.

**Example:**

Closing a browser releases its memory.
### 3. Memory Protection
Prevents unauthorized access between processes.

**Example:**

A music player cannot modify browser memory.
### 4. Address Translation
Converts logical address to physical address.
* **Logical Address** → Generated by CPU
* **Physical Address** → Actual memory location in RAM
### 5. Memory Sharing
Allows multiple processes to share common memory.

Example: Shared libraries.
## Types of Memory Management Techniques
### 1. Contiguous Memory Allocation
Memory is allocated in one continuous block.

Types:
* Single partition allocation
* Multiple partition allocation

**Advantage:** Simple implementation

**Disadvantage:** Memory wastage due to fragmentation
## 2. Paging
Memory is divided into fixed-size blocks:
* **Page** → Process divided into pages
* **Frame** → RAM divided into frames

Pages are loaded into available frames.

**Advantages**
* Eliminates external fragmentation
* Better memory utilization

**Disadvantage**
* Internal fragmentation may occur
## 3. Segmentation
Program divided into logical parts:
* Code segment
* Data segment
* Stack segment

Each segment has different size.

**Advantage**
* Easy logical organization

**Disadvantage**
* External fragmentation
## 4. Virtual Memory
Allows execution of processes larger than RAM using disk space.

Common technique:
* Demand Paging

Example:

A 10 GB application can run even if RAM is 8 GB.
## Fragmentation
Memory wastage due to improper allocation.
### External Fragmentation
Free memory exists but scattered.

Example:
```
100 KB free + 200 KB free + 300 KB free
```
A process requiring 500 KB cannot fit contiguously.
### Internal Fragmentation
Allocated memory contains unused space.

Example:

A process needs 18 KB but gets 20 KB.

Unused 2 KB = wastage.
## Swapping
A process is temporarily moved from RAM to disk and brought back later.

Purpose:
* Improve multiprogramming
* Free memory
## Page Replacement Algorithms
Used when memory is full.

Examples:
* FIFO (First In First Out)
* LRU (Least Recently Used)
* Optimal Page Replacement

## Advantages of Memory Management
* Efficient RAM utilization
* Supports multiprogramming
* Prevents memory conflicts
* Enables virtual memory
* Improves system performance
## Disadvantages
* Complex implementation
* Fragmentation issues
* Overhead in address translation

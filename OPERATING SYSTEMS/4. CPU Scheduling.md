## CPU Scheduling
### Introduction
**CPU Scheduling** is a process used by an operating system to decide **which process gets CPU time and for how long** when multiple processes are ready to execute.

The main goal of CPU scheduling is to **maximize CPU utilization and improve system performance**.

In a computer system, many processes may be waiting for execution, but only one process can use the CPU at a time (in a single processor system). Therefore, the OS scheduler selects the next process to execute.
### Why CPU Scheduling is Needed
* To keep the CPU busy (avoid idle time)
* To improve system performance
* To reduce waiting time of processes
* To provide fair CPU allocation
* To increase throughput (number of completed processes)
### Basic Terms in CPU Scheduling
#### 1. Process
A program in execution is called a process.
#### 2. Ready Queue
A queue containing processes that are ready to execute and waiting for CPU allocation.
#### 3. Scheduler
The OS component that selects the next process for execution.
#### 4. CPU Burst
The time during which a process executes on the CPU.
#### 5. I/O Burst
The time spent waiting for input/output operations.
### Types of CPU Scheduling
#### 1. Non-Preemptive Scheduling
Once a process gets the CPU, it continues execution until completion or waiting for I/O.

**Examples:**
* First Come First Serve (FCFS)
* Shortest Job First (SJF)
* Priority Scheduling (Non-preemptive)

**Advantages**
* Simple to implement
* Less overhead

**Disadvantages**
* Long waiting time for short processes
#### 2. Preemptive Scheduling
The OS can interrupt a running process and allocate CPU to another process.

**Examples:**
* Round Robin (RR)
* Shortest Remaining Time First (SRTF)
* Priority Scheduling (Preemptive)

**Advantages**
* Better response time
* Suitable for multitasking systems

**Disadvantages**
* More context switching overhead
### Scheduling Criteria
CPU scheduling algorithms are evaluated based on:
1. **CPU Utilization** – Keep CPU busy as much as possible
2. **Throughput** – Number of completed processes
3. **Turnaround Time** – Total time taken for process completion
4. **Waiting Time** – Time process waits in ready queue
5. **Response Time** – Time taken to respond to a request
### Formulae Used
**Turnaround Time (TAT)**

= Completion Time − Arrival Time

**Waiting Time (WT)**

= Turnaround Time − Burst Time

**Response Time (RT)**

= First CPU Allocation Time − Arrival Time

### CPU Scheduling Process
1. Process enters ready queue
2. Scheduler selects a process
3. CPU executes the process
4. Process completes or moves to waiting state
5. Scheduler selects the next process
### Common CPU Scheduling Algorithms
1. FCFS (First Come First Serve)
2. SJF (Shortest Job First)
3. SRTF (Shortest Remaining Time First)
4. Priority Scheduling
5. Round Robin Scheduling
6. Multilevel Queue Scheduling
### Example
Suppose there are three processes:
| Process | Burst Time |
| ------- | ---------: |
| P1      |          5 |
| P2      |          3 |
| P3      |          2 |

In **FCFS**, execution order depends on arrival:

**P1 → P2 → P3**

In **SJF**, shortest job executes first:

**P3 → P2 → P1**
### Real-Time Example
Imagine a printer queue:
* Documents waiting to print are like processes
* Printer is like CPU
* The order of printing is decided using scheduling
### Conclusion
CPU Scheduling is an important operating system function that decides **which process executes next**, ensuring efficient CPU utilization, reduced waiting time, and better system performance. Different scheduling algorithms are used depending on system requirements.

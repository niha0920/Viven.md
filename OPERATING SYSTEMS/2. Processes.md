## Processes in Operating System
A **process** is a **program in execution**.
When a program is loaded into memory and starts running, it becomes a process.

In simple words:

> **Program + Execution = Process**
### Example
* A file like `calculator.exe` stored on disk → **Program**
* When you open Calculator and it starts running → **Process**

Similarly:
* Opening a browser → process
* Playing music → process
* Running a C program → process
### Definition
A process is an active entity that contains:
1. **Program code (instructions)**
2. **Program counter** – address of next instruction to execute
3. **CPU registers**
4. **Memory space**
5. **Stack and Heap**
6. **Input/Output information**
7. **Process state**
### Components of a Process
1. **Text Section (Code Section)**
   Contains the program instructions.
2. **Data Section**
   Stores global and static variables.
3. **Heap Section**
   Used for dynamic memory allocation (`malloc`, `calloc`, `realloc`).
4. **Stack Section**
   Stores function calls, local variables, return addresses.

### Process States
A process goes through different states during execution:
1. **New** – Process is being created
2. **Ready** – Waiting for CPU allocation
3. **Running** – Currently executing
4. **Waiting/Blocked** – Waiting for I/O or event
5. **Terminated** – Execution completed

### Process Control Block (PCB)
The operating system stores information about every process in a **PCB (Process Control Block)**.

PCB contains:
* Process ID (PID)
* Process state
* Program counter
* CPU registers
* Memory management info
* Scheduling information
* I/O status information
### Characteristics of a Process
* Dynamic in nature (executes instructions)
* Requires system resources (CPU, memory, I/O)
* Has its own execution state
* Can run concurrently with other processes
### Example in Linux
If you run:
```
gedit
```
A new process is created for the text editor.

To see running processes:
```
ps
```
or
```
top
```
### Difference Between Program and Process
| Program             | Process                |
| ------------------- | ---------------------- |
| Passive entity      | Active entity          |
| Stored on disk      | Running in memory      |
| Static              | Dynamic                |
| Set of instructions | Executing instructions |

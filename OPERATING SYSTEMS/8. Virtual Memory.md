## Virtual Memory
### What is Virtual Memory?
**Virtual Memory** is a memory management technique used by an Operating System that gives a process the illusion of having a large continuous memory space, even if the physical RAM is smaller.

It uses part of the hard disk/SSD as an extension of RAM.

In simple words:

> When RAM becomes full, the OS temporarily moves less-used data from RAM to disk and brings it back when needed.

This allows programs larger than physical memory to run.
### Why is Virtual Memory Needed?
* To run **large applications** even when RAM is limited
* To support **multitasking**
* To improve **memory utilization**
* To provide **process isolation and protection**
* To avoid loading the entire program into RAM at once
### Example
Suppose:
* RAM = 4 GB
* Running applications need = 7 GB

Without virtual memory → system crashes or cannot open more applications.

With virtual memory → OS keeps active data in RAM and inactive data in disk space (swap area).
## How Virtual Memory Works
1. Program is divided into small blocks called **pages**
2. RAM is divided into equal-sized blocks called **frames**
3. OS loads only required pages into RAM
4. Remaining pages stay in disk (swap space/backing store)
5. When a needed page is missing in RAM, OS loads it from disk
### Flow
```
Program Execution
       ↓
Required page in RAM?
    Yes → Execute
    No
    ↓
Page Fault occurs
    ↓
OS loads page from Disk → RAM
    ↓
Continue Execution
```
## Key Terms in Virtual Memory
### 1. Page
Fixed-size block of logical memory.

Example: 4 KB
### 2. Frame
Fixed-size block of physical memory (RAM).
### 3. Paging
Technique of mapping pages to frames.
### 4. Page Table
Stores mapping between logical addresses and physical addresses.
### 5. Page Fault
Occurs when a required page is not present in RAM.
### 6. Swap Space
Disk area used as temporary memory.
## Address Translation in Virtual Memory
Logical address generated by CPU:
```
Page Number + Offset
```
OS checks page table:
```
Page Number → Frame Number
```
Physical address:
```
Frame Number + Offset
```
## Types of Virtual Memory
### 1. Paging
Memory divided into pages and frames.

Advantages:
* No external fragmentation
* Efficient memory usage

Disadvantage:
* Internal fragmentation possible
### 2. Segmentation
Memory divided logically:

Examples:
* Code segment
* Data segment
* Stack segment

Advantage:
* Logical division of memory

Disadvantage:
* External fragmentation
## Demand Paging
A technique where pages are loaded into RAM **only when needed**.

Instead of loading the whole program:
```
Load page only when requested
```
Benefits:
* Faster startup
* Less RAM usage
## Page Fault Handling Steps
1. CPU requests page
2. OS checks page table
3. Page not found → page fault
4. OS searches free frame
5. Page loaded from disk to RAM
6. Page table updated
7. Execution resumes
## Advantages of Virtual Memory
- Run large programs
- Better multitasking
- Efficient RAM utilization
- Increased system responsiveness
- Process isolation and protection
## Disadvantages of Virtual Memory
- Slower than RAM access
- Excessive paging causes performance degradation (**Thrashing**)
- Requires disk space
## Thrashing
When the system spends more time swapping pages between disk and RAM than executing processes.

Symptoms:
* Slow performance
* High disk usage
* CPU waiting

Cause:
* Too many processes
* Insufficient RAM
## Real-Time Example
Imagine your study table:
* Table = RAM
* Cupboard = Hard Disk
* Books on table = active data
* Books in cupboard = inactive data

When the table is full:

You keep some books in cupboard and bring them back when needed.

This is exactly how virtual memory works.
## Difference Between Physical Memory and Virtual Memory
| Physical Memory | Virtual Memory  |
| --------------- | --------------- |
| Actual RAM      | Logical memory  |
| Faster          | Slower          |
| Limited size    | Appears larger  |
| Hardware memory | Uses disk + RAM |

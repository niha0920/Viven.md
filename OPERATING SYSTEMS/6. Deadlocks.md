## Deadlocks in Operating System
A **deadlock** is a situation in an operating system where **two or more processes are unable to proceed because each is waiting for a resource held by another process**.

In simple words:

> **A deadlock occurs when processes wait for each other forever, and none of them can continue execution.**
### Real-Life Example
Imagine two people:
* Person A has a pen and needs a notebook.
* Person B has a notebook and needs a pen.

Both keep waiting for the other to release the resource.
- Neither can proceed → **Deadlock**
### Operating System Example
Suppose:
* Process P1 holds **Printer** and waits for **Scanner**
* Process P2 holds **Scanner** and waits for **Printer**

Both processes wait forever.
### Diagram
```
P1 → Waiting for Scanner (held by P2)
P2 → Waiting for Printer (held by P1)
```
So execution stops → **Deadlock**
### Necessary Conditions for Deadlock
Deadlock occurs only when the following **4 conditions** happen together (called Coffman conditions):
#### 1. Mutual Exclusion
At least one resource must be non-shareable.

Example:

Only one process can use a printer at a time.
#### 2. Hold and Wait
A process holds one resource and waits for another.

Example:

P1 holds printer and waits for scanner.
#### 3. No Preemption
Resources cannot be forcibly taken away.

Example:

OS cannot remove printer access from a process suddenly.
#### 4. Circular Wait
Processes form a circular chain of waiting.

Example:
```
P1 waits for P2
P2 waits for P3
P3 waits for P1
```
This creates a cycle.
### Resource Allocation Graph (RAG)
A graph used to represent deadlocks.
* **Process** → Circle
* **Resource** → Rectangle
* Arrow from process to resource → Request
* Arrow from resource to process → Allocation

Example:
```
P1 ----> R2
^         |
|         v
R1 <---- P2
```
Cycle present → Deadlock may exist.
### Methods to Handle Deadlock
#### 1. Deadlock Prevention
Prevent one of the four conditions from occurring.

Examples:
* Remove hold-and-wait
* Allow resource preemption
* Avoid circular wait
#### 2. Deadlock Avoidance
OS carefully allocates resources to avoid unsafe states.

Common algorithm:
* Banker’s Algorithm
#### 3. Deadlock Detection
Allow deadlock to happen and detect it later.

Methods:
* Wait-for graph
* Detection algorithms
#### 4. Deadlock Recovery
After detection:
* Terminate process(es)
* Preempt resources
* Roll back process
### Deadlock Prevention Techniques
| Condition        | Prevention Method                      |
| ---------------- | -------------------------------------- |
| Mutual exclusion | Make resources shareable (if possible) |
| Hold and wait    | Request all resources at once          |
| No preemption    | Force release of resources             |
| Circular wait    | Give resource ordering                 |
### Safe State vs Unsafe State
#### Safe State
System can allocate resources safely without deadlock.
#### Unsafe State
May lead to deadlock in future.

Note:

> **Unsafe state does not always mean deadlock, but deadlock is possible.**
### Advantages of Deadlock Handling
* Better resource utilization
* Prevents system freeze
* Improves reliability
### Disadvantages
* Prevention/avoidance increases overhead
* Detection algorithms consume resources
* Recovery may terminate important processes

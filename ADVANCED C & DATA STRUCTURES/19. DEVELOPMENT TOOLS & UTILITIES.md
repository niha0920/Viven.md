# LINUX COMMANDS USEFUL IN DEVELOPMENT
Linux commands are essential for software development, embedded systems, debugging, automation, version control, and system administration.
## 1. Basic File and Directory Commands
| Command         | Purpose                  | Example            |
| --------------- | ------------------------ | ------------------ |
| `pwd`           | Show current directory   | `pwd`              |
| `ls`            | List files/directories   | `ls -l`            |
| `cd`            | Change directory         | `cd /home/user`    |
| `mkdir`         | Create directory         | `mkdir test`       |
| `rmdir`         | Remove empty directory   | `rmdir test`       |
| `rm`            | Remove files/directories | `rm -rf test`      |
| `cp`            | Copy files               | `cp a.txt b.txt`   |
| `mv`            | Move/Rename files        | `mv old.c new.c`   |
| `touch`         | Create empty file        | `touch file.c`     |
| `cat`           | Display file content     | `cat file.txt`     |
| `more` / `less` | View large files         | `less log.txt`     |
| `head`          | First lines of file      | `head -5 file.txt` |
| `tail`          | Last lines of file       | `tail -f log.txt`  |
## 2. File Permission Commands
| Command | Purpose             | Example               |
| ------- | ------------------- | --------------------- |
| `chmod` | Change permissions  | `chmod 755 app.sh`    |
| `chown` | Change owner        | `chown user file.txt` |
| `umask` | Default permissions | `umask 022`           |
### Permission Representation
| Permission | Value |
| ---------- | ----- |
| Read       | 4     |
| Write      | 2     |
| Execute    | 1     |
Example:
```
chmod 755 file.sh
```
Meaning:
* Owner → rwx
* Group → r-x
* Others → r-x
## 3. Searching Commands
| Command  | Purpose              | Example              |
| -------- | -------------------- | -------------------- |
| `find`   | Search files         | `find . -name "*.c"` |
| `grep`   | Search text pattern  | `grep "main" file.c` |
| `locate` | Quickly locate files | `locate stdio.h`     |
| `which`  | Path of command      | `which gcc`          |
### Example

* `-n` → line number

# Vi editor 
`vi` (Visual Editor) is a powerful text editor available in almost all Linux/Unix systems.
`vim` (Vi Improved) is the enhanced version of `vi`.

It is widely used in:
* Embedded Linux development
* Shell scripting
* Kernel development
* Configuration file editing
* Remote server editing through SSH
## Opening a File in Vi
```
vi filename.txt
```
Example:
```
vi test.c
```
* Opens existing file
* Creates new file if not present
## Modes in Vi Editor
Vi works mainly in **3 modes**:
| Mode           | Purpose                              |
| -------------- | ------------------------------------ |
| Command Mode   | Default mode for commands            |
| Insert Mode    | Used for typing/editing              |
| Last Line Mode | Used for save/quit/search operations |
## 1. Command Mode
This is the default mode when vi opens.

Used for:
* Copy
* Paste
* Delete
* Navigation
* Undo
* Search

Press `Esc` anytime to return to command mode.
## 2. Insert Mode
Used to enter text.
### Enter Insert Mode
| Command | Meaning                     |
| ------- | --------------------------- |
| `i`     | Insert before cursor        |
| `I`     | Insert at beginning of line |
| `a`     | Append after cursor         |
| `A`     | Append at end of line       |
| `o`     | Open new line below         |
| `O`     | Open new line above         |
Example:
```
Press i
Type text
Press Esc
```
## 3. Last Line Mode
Accessed using:
```
:
```
Used for:
* Save
* Quit
* Search
* Replace

# GCC compiler
The GNU Compiler Collection (GCC) is a compiler system used to compile programs written in languages like:
* C
* C++
* Java
* Fortran
* Go

In development and embedded Linux systems, GCC is mainly used for compiling C/C++ programs.
## What is a Compiler?
A compiler converts:
```
High-Level Language  →  Machine Code
```
Example:
```c
printf("Hello");
```
is converted into executable binary instructions.
## Basic GCC Compilation Process
The GCC compilation process has 4 stages:
```
Source Code
   ↓
Preprocessor
   ↓
Compiler
   ↓
Assembler
   ↓
Linker
   ↓
Executable File
```
## GCC Compilation Stages
### 1. Preprocessing
Processes:
* `#include`
* `#define`
* Macros
* Header files

Command:
```
gcc -E test.c -o test.i
```
Output:
```
test.i
```
(preprocessed file)
## 2. Compilation
Converts C code into assembly code.

Command:
```
gcc -S test.i -o test.s
```
Output:
```
test.s
```
(assembly file)
## 3. Assembly
Converts assembly code into object code.

Command:
```
gcc -c test.s -o test.o
```
Output:
```
test.o
```
(object file)
## 4. Linking
Combines object files and libraries.

Command:
```
gcc test.o -o test
```
Output:
```
Executable file
```
## Simple Compilation
### Example Program
```c
#include<stdio.h>
int main()
{
    printf("Hello GCC\n");
    return 0;
}
```
### Compile
```
gcc hello.c -o hello
```
### Run
```
./hello
```
## Common GCC Options
| Option    | Meaning                 |
| --------- | ----------------------- |
| `-o`      | Output filename         |
| `-c`      | Compile only            |
| `-Wall`   | Enable warnings         |
| `-g`      | Generate debugging info |
| `-O`      | Optimization            |
| `-I`      | Include header path     |
| `-L`      | Library path            |
| `-l`      | Link library            |
| `-E`      | Preprocess only         |
| `-S`      | Generate assembly       |
| `-shared` | Create shared library   |
## Important GCC Commands
### Compile with Warnings
```
gcc -Wall test.c -o test
```
Warnings help detect:
* unused variables
* syntax mistakes
* logic problems
## Debugging Support
```
gcc -g test.c -o test
```
Used with `gdb`.
## Optimization
```
gcc -O2 test.c -o test
```
Optimization levels:
| Level | Meaning               |
| ----- | --------------------- |
| `-O0` | No optimization       |
| `-O1` | Basic optimization    |
| `-O2` | Moderate optimization |
| `-O3` | High optimization     |
## Multiple File Compilation
### Example
#### main.c
```c
#include<stdio.h>
void display();
int main()
{
    display();
    return 0;
}
```
#### display.c
```c
#include<stdio.h>
void display()
{
    printf("Welcome\n");
}
```
### Compile
```
gcc main.c display.c -o app
```
## Separate Compilation
### Compile Individually
```
gcc -c main.c
gcc -c display.c
```
Creates:
```
main.o
display.o
```
### Link
```
gcc main.o display.o -o app
```
## Header File Inclusion
### Custom Header Example
#### math.h
```c
int add(int, int);
```
#### math.c
```c
int add(int a, int b)
{
    return a+b;
}
```
#### main.c
```c
#include<stdio.h>
#include "math.h"
int main()
{
    printf("%d", add(2,3));
}
```
### Compile
```
gcc main.c math.c -o app
```
## Include Header Path
```
gcc -I/include/path main.c
```
## Library Linking
### Math Library
```c
#include<math.h>
sqrt(25);
```
Compile:
```
gcc test.c -lm
```
* `-lm` → links math library
## Static and Shared Libraries
### Static Library
Extension:
```
.a
```
### Shared Library
Extension:
```
.so
```
## Creating Static Library
### Create Object File
```
gcc -c math.c
```
### Create Library
```
ar rcs libmath.a math.o
```
### Link Library
```
gcc main.c -L. -lmath
```
## Creating Shared Library
```
gcc -shared -fPIC math.c -o libmath.so
```
## Cross Compilation
Used in embedded systems.

Example:
```
arm-linux-gnueabi-gcc test.c -o test
```
Compile on PC for ARM target.
## GCC and Makefile
Instead of typing long commands repeatedly:
### Makefile Example
```makefile
app: main.o display.o

main.o: main.c
	gcc -c main.c
display.o: display.c
	gcc -c display.c
```
Run:
```
make
```

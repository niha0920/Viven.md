# The purpose of pointers
Pointers are one of the most powerful features in **C programming**, and they serve several important purposes:
## 1. Direct Access to Memory
A pointer stores the **address of a variable**, allowing you to directly work with memory locations.
```c
int x = 10;
int *p = &x;  // p stores address of x
```
- This is useful for low-level programming and efficient memory handling.
## 2. Efficient Parameter Passing (Call by Reference)
Pointers allow functions to modify original variables.
```c
void update(int *a) {
    *a = 20;
}
```
- Instead of copying values, the function works on the actual data → saves memory & time.
## 3. Dynamic Memory Allocation
Pointers are essential for allocating memory at runtime using functions like:
- `malloc()`
- `calloc()`
- `free()`
```c
int *p = (int *)malloc(sizeof(int));
```
- Helps in creating flexible data structures.
## 4. Implementing Data Structures
Pointers are used to build:
- Linked Lists
- Trees
- Graphs
- Stacks and Queues
###
These structures rely on storing addresses to connect elements.
## 5. Array and String Handling
Arrays and pointers are closely related.
```c
int arr[3] = {1,2,3};
int *p = arr;
```
- Enables efficient traversal and manipulation of arrays.
## 6. Function Pointers
Pointers can store addresses of functions.
```c
int add(int a, int b) { return a + b; }
int (*fp)(int, int) = add;
```
- Useful for callbacks, dynamic function calls.
## 7. Returning Multiple Values from Functions
Using pointers, functions can modify multiple variables.
## 8. System-Level Programming
Pointers are heavily used in:
- Operating systems
- Embedded systems
- Device drivers
###
Because they interact directly with memory and hardware.
## In Short
Pointers are used to:
- Access memory directly
- Improve performance
- Enable dynamic memory usage
- Build complex data structures

# Defining pointers
## Defining Pointers in C
A **pointer** is a variable that stores the **memory address of another variable**.
To define a pointer, you use the `*` (asterisk) symbol.
## Basic Syntax
```c
data_type *pointer_name;
```
### Here:
- `data_type` → type of variable the pointer will point to
- `*` → indicates that the variable is a pointer
- `pointer_name` → name of the pointer
## Example 1: Simple Pointer Definition
```c
int *p;
```
### This means:
- `p` is a pointer to an integer (`int`)
- It can store the address of an integer variable
## Example 2: Definition + Initialization
```c
int x = 10;
int *p = &x;
```
### Explanation:
- `&x` → gives the address of `x`
- `p` stores that address
- `*p` → gives the value stored at that address (10)
## Example 3: Multiple Pointer Declarations
```c
int *p1, *p2;
```
- Both `p1` and `p2` are pointers to integers
### Important:
```c
int *p1, p2;   // WRONG understanding
```
- `p1` is a pointer
- `p2` is a normal integer
## Pointer Types
| Pointer Type      | Example    | Description               |
| ----------------- | ---------- | ------------------------- |
| Integer pointer   | `int *p`   | Points to an integer      |
| Float pointer     | `float *p` | Points to a float         |
| Character pointer | `char *p`  | Points to a character     |
| Void pointer      | `void *p`  | Generic pointer (no type) |
## Special Pointer Initialization
```c
int *p = NULL;
```
- `NULL` means the pointer is not pointing to any valid memory location.
## Key Points to Remember
- `*` is used **only while defining or dereferencing**
- `&` is used to **get the address of a variable**
- Pointer must match the data type it points to
- Always initialize pointers (avoid garbage values)
## Quick Summary
- Pointer = variable that stores address
- Defined using `*`
- Initialized using `&`
- Used to access value indirectly using `*p`

# The & and * operators
## The `&` and `*` Operators in C
These two operators are fundamental to understanding pointers.
## 1. Address-of Operator (`&`)
The `&` operator is used to get the memory address of a variable.
### Example:
```c
int x = 10;
int *p = &x;
```
### Explanation:
- `&x`→ gives the address of variable `x`
- `p` stores that address
####
So:
- `x` = 10
- `&x` = address of `x` (like 1000, just an example)
- `p` = address of `x`
## 2. Dereference Operator (`*`)
The `*` operator is used to **access the value stored at a memory address**.
### Example:
```c
int x = 10;
int *p = &x;
printf("%d", *p);
```
### Output:
```
10
```
### Explanation:
- `p` contains the address of `x`
- `*p` → value stored at that address → 10
## Combined Understanding
```c
int x = 10;
int *p = &x;
printf("%d\n", x);   // 10
printf("%p\n", &x);  // address of x
printf("%p\n", p);   // same address
printf("%d\n", *p);  // 10
```
## Memory Concept (Simple View)
```
Variable   Value   Address
   x        10      1000
   p       1000     2000
```
- `&x` → 1000
- `p` → 1000
- `*p` → 10
## Important Notes
- `&` → “Give me the address”
- `*` → “Give me the value at that address”
## Dual Role of `*`
The `*` symbol has **two meanings**:
### 1. Declaration
```c
int *p;   // p is a pointer
```
### 2. Dereferencing
```c
*p = 20;  // change value at address
```
## Modifying Value Using Pointer
```c
int x = 10;
int *p = &x;
*p = 50;
printf("%d", x);  // Output: 50
```
- Changing `*p` changes `x` because both refer to the same memory.
## Summary
| Operator | Meaning             | Example | Result              |
| -------- | ------------------- | ------- | ------------------- |
| `&`      | Address of variable | `&x`    | Address of `x`      |
| `*`      | Value at address    | `*p`    | Value stored at `p` |

# Pointer assignment
## Pointer Assignment in C
**Pointer assignment** means assigning a **memory address** to a pointer variable so that it points to a specific location.
## Basic Pointer Assignment
```c
int x = 10;
int *p;
p = &x;   // pointer assignment
```
### Explanation:
- `&x` → address of variable `x`
- `p` → now holds that address
- `p` is said to “point to `x`”
## Declaration + Assignment Together
```c
int x = 10;
int *p = &x;
```
- This is the most common way to assign a pointer.
## Assigning One Pointer to Another
```c
int x = 10;
int *p1 = &x;
int *p2;
p2 = p1;
```
### Now:
- Both `p1` and `p2` point to the **same variable** `x`
- Changing `*p2` will also affect `x`
## Example: Value Modification via Pointer
```c
int x = 10;
int *p = &x;
*p = 25;
```
### Now:
- `x = 25`
- Because `p` points to `x`
## Important Rules for Pointer Assignment
### 1. Type Compatibility
Pointers must be of the same type:
```c
int x;
int *p;
p = &x;   // ✔ correct
```
```c
float y;
p = &y;   // ❌ incorrect (type mismatch)
```
## 2. Assign Only Addresses (or NULL)
```c
int *p;
p = NULL;     // ✔ valid
```
```c
p = 100;      // ❌ invalid (not a valid address)
```
## 3. Pointer to Pointer Assignment
```c
int x = 10;
int *p = &x;
int **pp = &p;
```
- `pp` stores address of pointer `p`
## Common Mistake
```c
int *p;
*p = 10;   // ❌ ERROR (p is uninitialized)
```
- Always assign an address before dereferencing.
## Summary
- Pointer assignment = assigning an **address to a pointer**
- Use `&` to get address
- Use `*` to access/modify value
- Pointers can be assigned to other pointers
- Always initialize pointers before use

# Pointers with functions
## Pointers with Functions in C
Pointers become very powerful when used with functions. They allow functions to **modify original variables, return multiple values, and work efficiently with memory**.
## 1. Passing Pointers to Functions (Call by Reference)
Instead of passing values, we pass **addresses**.
### Example:
```c
#include <stdio.h>
void update(int *a) {
    *a = 50;
}
int main() {
    int x = 10;
    update(&x);
    printf("%d", x);   // Output: 50
    return 0;
}
```
### Explanation:
- `&x` → address of `x` is passed
- `*a = 50` modifies the original variable
####
This is called **Call by Reference**
## 2. Swapping Two Numbers Using Pointers
Without pointers, swapping won’t affect original values.
### Example:
```c
#include <stdio.h>
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
int main() {
    int x = 5, y = 10;
    swap(&x, &y);
    printf("x = %d, y = %d", x, y);
    return 0;
}
```
### Output:
```
x = 10, y = 5
```
## 3. Returning Multiple Values Using Pointers
Functions normally return one value, but pointers help return multiple.
### Example:
```c
#include <stdio.h>
void calc(int a, int b, int *sum, int *diff) {
    *sum = a + b;
    *diff = a - b;
}
int main() {
    int s, d;
    calc(10, 5, &s, &d);
    printf("Sum = %d, Diff = %d", s, d);
    return 0;
}
```
## 4. Passing Arrays to Functions (Using Pointers)
Arrays are passed as pointers.
```c
#include <stdio.h>
void display(int *arr, int n) {
    for(int i = 0; i < n; i++) {
        printf("%d ", *(arr + i));
    }
}
```
- `arr` acts like a pointer to the first element.
## 5. Returning Pointers from Functions
A function can return a pointer.
```c
#include <stdio.h>
int* getValue() {
    static int x = 10;
    return &x;
}
```
- Use `static` to avoid returning address of a local variable.
## 6. Function Pointers
Pointers can store addresses of functions.
```c
#include <stdio.h>
int add(int a, int b) {
    return a + b;
}
int main() {
    int (*fp)(int, int) = add;
    printf("%d", fp(2, 3));  // Output: 5
    return 0;
}
```
## Important Notes
- Always pass **address using** `&`
- Use `*` inside function to access/modify value
- Avoid returning pointer to local variables
- Arrays automatically behave like pointers
## Summary
Pointers with functions are used for:
- Call by reference
- Swapping values
- Returning multiple values
- Efficient array handling
- Function pointers (advanced use)

# Pointer to Pointer
## Pointer to Pointer in C
A **pointer to pointer** is a variable that stores the **address of another pointer**.
## Definition
```c
data_type **pointer_name;
```
### It means:
- First `*` → pointer to a variable
- Second `*` → pointer to another pointer
## Basic Example
```c
#include <stdio.h>
int main() {
    int x = 10;
    int *p = &x;     // pointer to int
    int **pp = &p;   // pointer to pointer
    printf("%d\n", x);    // 10
    printf("%d\n", *p);   // 10
    printf("%d\n", **pp); // 10
    return 0;
}
```
## How It Works
```
x   = 10        (value)
&p  = address of p
&p  = stored in pp
Memory Representation:
Variable   Value        Address
x          10           1000
p          1000         2000
pp         2000         3000
```
### Relationships:
- `p = &x`
- `pp = &p`
- `*p = x`
- `**pp = x`
## Accessing Values
| Expression | Meaning                           |
| ---------- | --------------------------------- |
| `p`        | Address of `x`                    |
| `*p`       | Value of `x`                      |
| `pp`       | Address of `p`                    |
| `*pp`      | Value of `p` (i.e., address of x) |
| `**pp`     | Value of `x`                      |
## Example: Modifying Value
```c
#include <stdio.h>
int main() {
    int x = 5;
    int *p = &x;
    int **pp = &p;
    **pp = 20;
    printf("%d", x);  // Output: 20
    return 0;
}
- Changing `**pp` changes `x`.
## Why Use Pointer to Pointer?
### 1. Dynamic Memory Allocation (2D arrays)
```c
int **matrix;
```
### 2. Modifying a Pointer inside a Function
```c
void allocate(int **p) {
    *p = (int *)malloc(sizeof(int));
}
```
### 3. Handling Complex Data Structures
- Linked lists (double pointers)
- Trees and graphs
## Important Notes
- Each `*` level = one level of indirection
- `**pp` means “value at the address stored in pointer p”
- Always initialize pointers properly
## Summary
- Pointer → stores address of variable
- Pointer to pointer → stores address of another pointer
- `*p` → value
- `**pp` → value of original variable

# Pointers to Arrays
## Pointers to Arrays in C
Pointers and arrays are closely related. A **pointer to an array** is a pointer that stores the **address of an entire array**, not just a single element.
## 1. Basic Idea
In C, the name of an array itself acts like a pointer to its first element.
```c
int arr[5] = {10, 20, 30, 40, 50};
```
- `arr` → address of first element (`&arr[0]`)
## 2. Pointer to Array (Whole Array)
### Syntax:
```c
data_type (*pointer_name)[size];
```
### Example:
```c
int arr[5] = {10, 20, 30, 40, 50};
int (*p)[5] = &arr;
```
### Explanation:
- `p` is a pointer to the **entire array of 5 integers**
- `&arr` → address of the whole array
## 3. Accessing Elements Using Pointer to Array
```c
#include <stdio.h>
int main() {
    int arr[3] = {1, 2, 3};
    int (*p)[3] = &arr;
    printf("%d\n", (*p)[0]); // 1
    printf("%d\n", (*p)[1]); // 2
    printf("%d\n", (*p)[2]); // 3
    return 0;
}
```
- `(*p)[i]` is used to access elements
## 4. Difference: Pointer to Array vs Normal Pointer
| Concept          | Example               | Meaning                 |
| ---------------- | --------------------- | ----------------------- |
| Normal pointer   | `int *p = arr;`       | Points to first element |
| Pointer to array | `int (*p)[5] = &arr;` | Points to whole array   |
## 5. Pointer Arithmetic
```c
int arr[5] = {1,2,3,4,5};
int (*p)[5] = &arr;
p++;  // moves to next array of 5 integers
```
### Important:
- `p + 1` moves by **entire array size**
- Not just one element
## 6. Pointer to Array with Functions
```c
#include <stdio.h>
void display(int (*p)[3]) {
    for(int i = 0; i < 3; i++) {
        printf("%d ", (*p)[i]);
    }
}
int main() {
    int arr[3] = {10, 20, 30};
    display(&arr);
    return 0;
}
```
## 7. Pointer to Multidimensional Array
```c
int arr[2][3] = {{1,2,3}, {4,5,6}};
int (*p)[3] = arr;
```
### Here:
- `p` points to each row (array of 3 elements)
####
Access:
```c
printf("%d", p[1][2]); // 6
```
## Common Mistake
```c
int *p[5];   // ❌ Not pointer to array
```
### This means:
- Array of 5 pointers (not pointer to array)
## Summary
- `arr` → pointer to first element
- `int *p` → pointer to element
- `int (*p)[n]` → pointer to entire array
- Use `(*p)[i]` to access elements
- Useful in multidimensional arrays and functions

# Void Pointers
## Void Pointers in C
A **void pointer** (`void *`) is a **generic pointer** that can store the address of **any data typ**e.
## Definition
```c
void *ptr;
```
### This means:
- `ptr` can point to **int, float, char, structure, etc**.
- It has **no specific type**
## Example: Assigning Different Types
```c
#include <stdio.h>
int main() {
    int x = 10;
    float y = 5.5;
    void *ptr;
    ptr = &x;   // points to int
    ptr = &y;   // now points to float
    return 0;
}
```
- A void pointer can **hold any address**, but…
## Important Rule: Type Casting Required
You **cannot directly dereference** a void pointer.
### Wrong:
```c
printf("%d", *ptr);
```
### Correct:
```c
printf("%d", *(int *)ptr);
```
- You must **cast** it to the correct type before dereferencing.
## Example with Dereferencing
```c
#include <stdio.h>
int main() {
    int x = 10;
    void *ptr = &x;
    printf("%d", *(int *)ptr);  // Output: 10
    return 0;
}
```
## Why Use Void Pointers?
### 1. Generic Functions
Used when a function should work with **any data type**
#### Example:
Standard library function like `malloc()` returns `void *`
```c
int *p = (int *)malloc(sizeof(int));
```
## 2. Data Structures
Useful in:
- Linked lists
- Generic containers
## 3. Flexibility
Same pointer can point to different types at different times
## Pointer Arithmetic on Void Pointer
Not allowed directly:
```c
ptr++;
```
- Because size of data is unknown
### Solution:
```c
(char *)ptr + 1;
```
## Summary Table
| Feature       | Description                          |
| ------------- | ------------------------------------ |
| Type          | Generic pointer                      |
| Declaration   | `void *ptr;`                         |
| Dereferencing | Requires type casting                |
| Arithmetic    | Not allowed directly                 |
| Use case      | Generic programming, memory handling |
## Key Points
- `void *` → can point to any data type
- Must **typecast before dereferencing**
- Widely used in **dynamic memory allocation**
- Provides **flexibility but less type safety**

# Call By value and Call by reference
## Call by Value vs Call by Reference (in C)
These terms describe **how arguments are passed to functions**.
## 1. Call by Value
In **call by value**, a **copy of the variable** is passed to the function.
### Example:
```c
#include <stdio.h>
void change(int a) {
    a = 50;
}
int main() {
    int x = 10;
    change(x);
    printf("%d", x);   // Output: 10
    return 0;
}
```
### Explanation:
- `x` is copied into `a`
- Changes inside the function **do not affect original variable**
## 2. Call by Reference (Using Pointers)
In C, this is implemented using **pointers**. We pass the **address** of the variable.
### Example:
```c
#include <stdio.h>
void change(int *a) {
    *a = 50;
}
int main() {
    int x = 10;
    change(&x);
    printf("%d", x);   // Output: 50
    return 0;
}
```
### Explanation:
- Address of `x` is passed
- Function directly modifies original variable
## Key Differences
| Feature             | Call by Value           | Call by Reference          |
| ------------------- | ----------------------- | -------------------------- |
| What is passed      | Copy of value           | Address of variable        |
| Changes affect      | Original not affected   | Original is modified       |
| Memory usage        | More (copy created)     | Less (no copy)             |
| Safety              | Safer (no side effects) | Risk of unintended changes |
| Implementation in C | Default                 | Using pointers             |
## Example: Swap
### Call by Value (Fails)
```c
void swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}
```
- No effect on original variables
### Call by Reference (Works)
```c
void swap(int *a, int *b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
```
## Important Note
-  C **does NOT have true call by reference** like C++
- It only uses **call by value**, but pointers simulate reference behavior
## Summary
- **Call by Value** → function works on copy
- **Call by Referenc**e → function works on original (via pointers)
- Use pointers when you need to modify actual data

# Advanced pointer types
## Advanced Pointer Types in C
Once you understand basic pointers, C offers several **advanced pointer types** that are heavily used in real programs, data structures, and system-level coding.
## 1. Pointer to Pointer (Double Pointer)
A pointer that stores the address of another pointer.
```c
int x = 10;
int *p = &x;
int **pp = &p;
printf("%d", **pp);  // 10
```
### Used in:
- Dynamic memory allocation (2D arrays)
- Modifying pointers inside functions
## 2. Function Pointers
Pointers that store the address of a function.
```c
#include <stdio.h>
int add(int a, int b) {
    return a + b;
}
int main() {
    int (*fp)(int, int) = add;
    printf("%d", fp(2, 3));  // 5
    return 0;
}
```
### Used in:
- Callbacks
- Event handling
- Menu-driven programs
## 3. Void Pointers (Generic Pointers)
A pointer that can point to any data type.
```c
int x = 10;
void *ptr = &x;
printf("%d", *(int *)ptr);
```
- Must be typecast before dereferencing.
## 4. Pointer to Array
Pointer that points to an entire array.
```c
int arr[5] = {1,2,3,4,5};
int (*p)[5] = &arr;
printf("%d", (*p)[2]);  // 3
```
- Useful in multidimensional arrays.
## 5. Array of Pointers
An array where each element is a pointer.
```c
int a = 10, b = 20, c = 30;
int *arr[3] = {&a, &b, &c};
printf("%d", *arr[1]);  // 20
```
### Used in:
- Strings (`char *arr[]`)
- Jagged arrays
## 6. Null Pointer
A pointer that points to nothing.
```c
int *p = NULL;
```
### Used for:
- Initialization
- Error handling
## 7. Wild Pointer
An uninitialized pointer.
```c
int *p;  // wild pointer
```
- Dangerous: may point anywhere (garbage address)
## 8. Dangling Pointer
Pointer pointing to freed or invalid memory.
```c
int *p = (int *)malloc(sizeof(int));
free(p);
```
- `p` is now a dangling pointer
## 9. Constant Pointer
### (a) Pointer to Constant
```c
const int *p;
```
- Cannot modify value through pointer
### (b) Constant Pointer
```c
int *const p = &x;
```
- Cannot change the pointer itself
## 10. Pointer Arithmetic
Operations on pointers:
```c
int arr[3] = {10,20,30};
int *p = arr;
p++;  // moves to next element
```
### Used for:
- Array traversal
- Memory navigation
## 11. Near, Far, Huge Pointers (Legacy)
Used in older systems (like DOS memory models)
- Near pointer → 16-bit
- Far pointer → access outside segment
- Huge pointer → normalized far pointer
### 
Mostly obsolete today
## Summary
Advanced pointer types include:
- Double pointers
- Function pointers
- Void pointers
- Pointer to array / Array of pointers
- Null, wild, dangling pointers
- Constant pointers
- Pointer arithmetic

# Pointers to functions
## Pointers to Functions in C (Function Pointers)
A **function pointer** is a pointer that stores the **address of a function** instead of a variable.
## 1. Why Function Pointers?
They allow you to:
- Call functions dynamically
- Pass functions as arguments (callbacks)
- Write flexible and reusable code
## 2. Syntax
```c
return_type (*pointer_name)(parameter_types);
```
### Example:
```c
int (*fp)(int, int);
```
- `fp` is a pointer to a function
- Function takes two `int` arguments and returns `int`
## 3. Assigning a Function to Pointer
```c
int add(int a, int b) {
    return a + b;
}
int (*fp)(int, int) = add;
```
You can also write:
```c
fp = &add;   // same meaning
```
## 4. Calling Function Using Pointer
```c
printf("%d", fp(2, 3));     // 5
```
Or:
```c
printf("%d", (*fp)(2, 3));  // 5
```
Both are valid.
## 5. Example Program
```c
#include <stdio.h>
int multiply(int a, int b) {
    return a * b;
}
int main() {
    int (*fp)(int, int);
    fp = multiply;
    printf("%d", fp(4, 5));  // Output: 20
    return 0;
}
```
## 6. Passing Function Pointer to Another Function
```c
#include <stdio.h>
int add(int a, int b) {
    return a + b;
}
void operate(int (*f)(int, int)) {
    printf("%d", f(3, 4));
}
int main() {
    operate(add);   // passing function as argument
    return 0;
}
```
- This is called a **callback function**
## 7. Array of Function Pointers
```c
#include <stdio.h>
int add(int a, int b) { return a + b; }
int sub(int a, int b) { return a - b; }
int main() {
    int (*fp[2])(int, int) = {add, sub};
    printf("%d\n", fp[0](5, 3)); // add
    printf("%d\n", fp[1](5, 3)); // sub
    return 0;
}
```
## 8. Real-World Uses
- Callback functions (e.g., sorting functions like `qsort()`)
- Event handling systems
- Menu-driven programs
- Implementing function tables
## Important Notes
- Function name itself acts as pointer
- Signature must match exactly
- No need for `&` while assigning (optional)
## Summary
- Function pointer = pointer to function
- Used to call functions indirectly
- Enables callbacks and flexible code design

# Pointers and Strings
## Pointers and Strings in C
In C, **strings are closely related to pointers** because a string is essentially an **array of characters ending with `'\0'` (null character).**
## 1. String as a Pointer
```c
char str[] = "Hello";
```
Internally:
- Stored as: `H e l l o \0`
- `str` acts like a pointer to the first character
###
Equivalent pointer form:
```c
char *p = "Hello";
```
## 2. Accessing Characters Using Pointer
```c
#include <stdio.h>
int main() {
    char *p = "Hello";
    printf("%c\n", *p);      // H
    printf("%c\n", *(p+1));  // e
    return 0;
}
```
- Pointer arithmetic helps traverse the string.
## 3. Traversing a String Using Pointer
```c
#include <stdio.h>
int main() {
    char *p = "Hello";
    while(*p != '\0') {
        printf("%c ", *p);
        p++;
    }
    return 0;
}
```
- Output:
```
H e l l o
```
## 4. String vs Pointer Difference
| Concept           | Example              | Behavior                   |
| ----------------- | -------------------- | -------------------------- |
| Character array   | `char str[] = "Hi";` | Modifiable                 |
| Pointer to string | `char *p = "Hi";`    | Not modifiable (read-only) |
- Important:
```c
char *p = "Hello";
p[0] = 'h';   // ❌ Undefined behavior
```

#include <stdio.h>
void display(char *str) {

int main() {

```

# Pointers and Dynamic memory
## Pointers and Dynamic Memory in C
**Dynamic memory allocation** means allocating memory **at runtime** using pointers.
Instead of fixed-size variables, you can create memory when needed and free it later.
## 1. Why Dynamic Memory?
- Memory is allocated **during program execution**
- Useful when size is **not known in advance**
- Helps build **flexible data structures** (linked lists, trees, etc.)
## 2. Memory Allocation Functions
From the Standard C Library:
| Function    | Purpose                                      |
| ----------- | -------------------------------------------- |
| `malloc()`  | Allocates single block                       |
| `calloc()`  | Allocates multiple blocks (initialized to 0) |
| `realloc()` | Resizes allocated memory                     |
| `free()`    | Deallocates memory                           |
## 3. Using `malloc()`
```c
#include <stdio.h>
#include <stdlib.h>
int main() {
    int *p;
    p = (int *)malloc(sizeof(int));
    *p = 10;
    printf("%d", *p);
    free(p);
    return 0;
}
```
Key Points:
- `malloc()` returns `void *` → needs typecasting
- Memory contains **garbage value initially**
## 4. Using `calloc()`
```c
int *p = (int *)calloc(5, sizeof(int));
```
Allocates memory for **5 integers**
- Initializes all values to **0**
## 5. Using `realloc()`
```c
p = (int *)realloc(p, 10 * sizeof(int));
```
- Changes memory size (expand/shrink)
## 6. Freeing Memory
```c
free(p);
```
- Prevents **memory leaks**
## 7. Dynamic Array Example
```c
#include <stdio.h>
#include <stdlib.h>
int main() {
    int n;
    scanf("%d", &n);
    int *arr = (int *)malloc(n * sizeof(int));
    for(int i = 0; i < n; i++) {
        arr[i] = i + 1;
    }
    for(int i = 0; i < n; i++) {
        printf("%d ", arr[i]);
    }
    free(arr);
    return 0;
}
```
## 8. Common Problems
### Memory Leak
```c
int *p = malloc(sizeof(int));
// forgot free(p);
```
- Memory is never released
## Dangling Pointer
```c
free(p);
// p still points to freed memory
```
- Fix:
```c
p = NULL;
```
## Uninitialized Pointer
```c
int *p;
*p = 10;   // ERROR
```
## 9. Advantages
- Efficient memory usage
- Flexible size allocation
- Essential for dynamic data structures
## 10. Disadvantages
- Manual memory management
- Risk of memory leaks
- More complex than static allocation
## Summary
- Dynamic memory is allocated using pointers
- Functions: `malloc()`, `calloc()`, `realloc()`, `free()`
- Always free memory after use
- Helps create flexible and efficient programs

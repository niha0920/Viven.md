# Concept of a FILE data type
In C programming, a **FILE data type** is used to handle files.
It is defined in the standard library header:
```c
#include<stdio.h>
```
The `FILE` type is actually a **structure** defined by the C standard library to store information about a file such as:
* File name
* File position
* Buffer information
* File mode (read/write)
* Error status
## Declaration of FILE Pointer
A file is accessed using a **FILE pointer**.
### Syntax
```c
FILE *fp;
```
Here:
* `FILE` → predefined data type
* `*fp` → pointer to the file
## Why FILE Pointer is Needed
The operating system cannot directly understand C variables.
So C uses a `FILE` structure to communicate with files.

The file pointer helps to:
* Open a file
* Read data
* Write data
* Move file position
* Close file
## Opening a File
A file is opened using `fopen()`.
### Syntax
```c
fp = fopen("filename","mode");
```
Example:
```c
FILE *fp;
fp = fopen("data.txt","r");
```
## File Opening Modes
| Mode   | Meaning                  |
| ------ | ------------------------ |
| `"r"`  | Read                     |
| `"w"`  | Write                    |
| `"a"`  | Append                   |
| `"r+"` | Read & Write             |
| `"w+"` | Read & Write (overwrite) |
| `"a+"` | Read & Append            |
## Example Program
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("sample.txt", "w");
    fprintf(fp, "Hello File");
    fclose(fp);
    return 0;
}
```
## Explanation
1. `FILE *fp;`
   * Creates a file pointer.
2. `fopen()`
   * Opens the file.
3. `fprintf()`
   * Writes data into the file.
4. `fclose()`
   * Closes the file.
## Important File Functions
| Function    | Purpose              |
| ----------- | -------------------- |
| `fopen()`   | Open file            |
| `fclose()`  | Close file           |
| `fprintf()` | Write formatted data |
| `fscanf()`  | Read formatted data  |
| `fgetc()`   | Read character       |
| `fputc()`   | Write character      |
| `fgets()`   | Read string          |
| `fputs()`   | Write string         |
## Real-Time Example
When applications save:
* Student records
* Logs
* Configuration data
* Images
* Reports

they use the `FILE` data type internally for file handling.
## Key Point
`FILE` is **not a primitive data type** like `int` or `char`.

It is a **user-defined structure type** provided by the C standard library for file management.

# File Input, Output operations
File handling in C allows data to be stored permanently in files instead of temporary memory.

Using file I/O operations, we can:
* Create files
* Read data from files
* Write data into files
* Append data
* Modify existing data
## Steps in File Handling
### 1. Declare File Pointer
```c 
FILE *fp;
```
## 2. Open the File
```c 
fp = fopen("file.txt","mode");
```
## 3. Perform Read/Write Operations
Using functions like:
* `fprintf()`
* `fscanf()`
* `fgetc()`
* `fputc()`
* `fgets()`
* `fputs()`
## 4. Close the File
```c 
fclose(fp);
```
## File Opening Modes
| Mode   | Description              |
| ------ | ------------------------ |
| `"r"`  | Open for reading         |
| `"w"`  | Open for writing         |
| `"a"`  | Open for appending       |
| `"r+"` | Read and write           |
| `"w+"` | Read/write and overwrite |
| `"a+"` | Read and append          |
## File Output Operations (Writing)
Output operations write data into a file.
## 1. fprintf()
Writes formatted output to a file.
### Syntax
```c 
fprintf(fp,"format",variables);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt","w");
    fprintf(fp,"Name: Niharika\n");
    fprintf(fp,"Age: 22");
    fclose(fp);
    return 0;
}
```
## 2. fputc()
Writes a single character.
### Syntax
```c 
fputc(character, fp);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("char.txt","w");
    fputc('A', fp);
    fclose(fp);
    return 0;
}
```
## 3. fputs()
Writes a string into a file.
### Syntax
```c 
fputs(string, fp);
```
### Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("text.txt","w");
    fputs("Welcome to C Programming", fp);
    fclose(fp);
    return 0;
}
```
## File Input Operations (Reading)
Input operations read data from a file.
## 1. fscanf()
Reads formatted input.
### Syntax
```c 
fscanf(fp,"format",variables);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char name[20];
    fp = fopen("data.txt","r");
    fscanf(fp,"%s",name);
    printf("%s",name);
    fclose(fp);
    return 0;
}
```
## 2. fgetc()
Reads one character at a time.
### Syntax
```c 
ch = fgetc(fp);
```
### Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("char.txt","r");
    ch = fgetc(fp);
    printf("%c", ch);
    fclose(fp);
    return 0;
}
```
## 3. fgets()
Reads a string or line from a file.
### Syntax
```c 
fgets(string, size, fp);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char str[50];
    fp = fopen("text.txt","r");
    fgets(str,50,fp);
    printf("%s",str);
    fclose(fp);
    return 0;
}
```
## Append Operation
Append mode adds data at the end of a file without deleting old content.
### Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt","a");
    fprintf(fp,"\nNew Data Added");
    fclose(fp);
    return 0;
}
```
## Important Functions in File I/O
| Function    | Purpose               |
| ----------- | --------------------- |
| `fopen()`   | Opens file            |
| `fclose()`  | Closes file           |
| `fprintf()` | Writes formatted data |
| `fscanf()`  | Reads formatted data  |
| `fputc()`   | Writes character      |
| `fgetc()`   | Reads character       |
| `fputs()`   | Writes string         |

## Real-Time Applications
File I/O is used in:
* Banking systems
* Student databases
* Log files
* Employee management systems
* Configuration files
* Data storage systems

Example:

# Sequential Files
A **Sequential File** is a type of file where data is stored and accessed in a sequence, one record after another from beginning to end.

The records are processed in the same order in which they are stored.
## Definition
In sequential file organization:
* Data is stored line by line or record by record.
* Access starts from the beginning of the file.
* To reach a specific record, previous records must be read first.
## Characteristics of Sequential Files
* Records are stored continuously.
* Easy to create and maintain.
* Suitable for serial processing.
* Access speed is slower for large files because searching is sequential.
* Mostly used for text files.
## Working of Sequential File
Example File Content:
```
101 Ravi
102 Priya
103 Kiran
```
The system reads:
1. First record → `101 Ravi`
2. Second record → `102 Priya`
3. Third record → `103 Kiran`

in sequence.
## Sequential File Operations
### 1. Creating/Writing Sequential File
#### Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("student.txt","w");
    fprintf(fp,"101 Ravi\n");
    fprintf(fp,"102 Priya\n");
    fprintf(fp,"103 Kiran\n");
    fclose(fp);
    return 0;
}
```
## Reading Sequential File
Data is read one by one in order.
### Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("student.txt","r");
    while((ch = fgetc(fp)) != EOF)
    {
        printf("%c", ch);
    }
    fclose(fp);
    return 0;
}
```
## Explanation
### Writing Phase
```c 
fprintf(fp,"101 Ravi\n");
```
Stores records sequentially into the file.
## Reading Phase
```c 
while((ch = fgetc(fp)) != EOF)
```
Reads characters sequentially until the end of file (`EOF`).
## Advantages of Sequential Files
* Simple to implement
* Requires less memory
* Good for batch processing
* Easy for backup storage
* Efficient for reading complete files
## Disadvantages of Sequential Files
* Slow searching
* Difficult to modify records in the middle
* Inefficient for random access
* Entire file may need to be scanned
## Real-Time Applications
Sequential files are used in:
* Log files
* Text documents
* Payroll processing
* Student records
* Bank transaction history
* Report generation systems
## Sequential vs Random Access Files
| Sequential File              | Random Access File     |
| ---------------------------- | ---------------------- |
| Access in order              | Direct access possible |
| Slower searching             | Faster searching       |
| Simple structure             | More complex           |
| Suitable for text processing | Suitable for databases |
## Important Point
Sequential files process data from:
```
Beginning → End
```
without skipping records directly.

# Random Access Files
A **Random Access File** allows data to be accessed directly at any position in the file without reading all previous records.

Unlike sequential files, records can be:
* Read directly
* Modified directly
* Updated directly

using file positioning functions.
## Definition
In random access file organization:
* Records are stored in fixed locations.
* Any record can be accessed using its position.
* File pointer can move forward or backward.
## Why Random Access is Needed
In sequential files:
* To access record 100,
* records 1 to 99 must be read first.

Random access solves this problem by directly jumping to the required location.
## Important File Positioning Functions
| Function   | Purpose                   |
| ---------- | ------------------------- |
| `fseek()`  | Move file pointer         |
| `ftell()`  | Get current position      |
| `rewind()` | Move pointer to beginning |
## 1. fseek()
Moves the file pointer to a specific location.
### Syntax
```c 
fseek(fp, offset, position);
```
## Parameters
### `offset`
Number of bytes to move.
### `position`
Starting point for movement.
| Position   | Meaning           |
| ---------- | ----------------- |
| `SEEK_SET` | Beginning of file |
| `SEEK_CUR` | Current position  |
| `SEEK_END` | End of file       |
## Example of Random Access
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("data.txt","r");
    fseek(fp, 5, SEEK_SET);
    ch = fgetc(fp);
    printf("%c", ch);
    fclose(fp);
    return 0;
}
```
## Explanation
```c 
fseek(fp, 5, SEEK_SET);
```
Moves the pointer to the 6th byte from the beginning.

Then:
```c 
ch = fgetc(fp);
```
reads that character directly.
## 2. ftell()
Returns the current file pointer position.
### Syntax
```c 
ftell(fp);
```
## Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt","r");
    printf("%ld", ftell(fp));
    fgetc(fp);
    printf("\n%ld", ftell(fp));
    fclose(fp);
    return 0;
}
```
## Output Example
```
0
1
```
## 3. rewind()
Moves the file pointer back to the beginning.
### Syntax
```c
rewind(fp);
```
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("data.txt","r");
    ch = fgetc(fp);
    rewind(fp);
    ch = fgetc(fp);
    printf("%c", ch);
    fclose(fp);
    return 0;
}
```
## Random Access with Structures
Random access is commonly used with structures.
## Example: Student Record Access
```c 
#include<stdio.h>
struct student
{
    int id;
    char name[20];
};
int main()
{
    FILE *fp;
    struct student s;
    fp = fopen("student.dat","rb");
    fseek(fp, sizeof(s)*2, SEEK_SET);
    fread(&s, sizeof(s), 1, fp);
    printf("%d %s", s.id, s.name);
    fclose(fp);
    return 0;
}
```
## Explanation
```c 
fseek(fp, sizeof(s)*2, SEEK_SET);
```
Moves directly to the 3rd record.
## Advantages of Random Access Files
* Faster data retrieval
* Direct record modification
* Efficient for large databases
* Saves processing time
* Easy record updates
## Disadvantages
* More complex than sequential files
* Requires careful file management
* Fixed-size records preferred
## Real-Time Applications
Random access files are used in:
* Banking systems
* Employee databases
* Library management systems
* Airline reservation systems
* Student management systems
## Sequential vs Random Access
| Sequential Access   | Random Access      |
| ------------------- | ------------------ |
| Read in order       | Direct access      |
| Slower searching    | Faster searching   |
| Simple              | Complex            |
| Good for text files | Good for databases |
## Important Point
Random access files use:
```
File Pointer Manipulation
```
to jump directly to required data locations.

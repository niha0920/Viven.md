# fopen, fread, fwrite, fclose, fseek
## File Handling Functions in C
These functions are used for handling files in C programming.
## 1. `fopen()`
Used to open a file.
### Syntax
```c
FILE *fopen(const char *filename, const char *mode);
```
### Modes
| Mode   | Meaning                    |
| ------ | -------------------------- |
| `"r"`  | Open for reading           |
| `"w"`  | Open for writing           |
| `"a"`  | Open for appending         |
| `"r+"` | Read and write             |
| `"w+"` | Read and write (overwrite) |
| `"a+"` | Read and append            |
| `"rb"` | Read binary                |
| `"wb"` | Write binary               |
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt", "w");
    if(fp == NULL)
    {
        printf("File not opened");
        return 1;
    }
    printf("File opened successfully");
    fclose(fp);
    return 0;
}
```
## 2. `fread()`
Used to read data from a file in binary format.
### Syntax
```c
fread(buffer, size, count, file_pointer);
```
### Parameters
| Parameter      | Meaning                |
| -------------- | ---------------------- |
| `buffer`       | Variable to store data |
| `size`         | Size of each element   |
| `count`        | Number of elements     |
| `file_pointer` | Pointer to file        |
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char str[20];
    fp = fopen("data.txt", "rb");
    fread(str, sizeof(char), 5, fp);
    str[5] = '\0';
    printf("%s", str);
    fclose(fp);
    return 0;
}
```
## 3. `fwrite()`
Used to write data into a file in binary format.
### Syntax
```c
fwrite(buffer, size, count, file_pointer);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char str[] = "Hello";
    fp = fopen("data.txt", "wb");
    fwrite(str, sizeof(char), 5, fp);
    fclose(fp);
    return 0;
}
```
## 4. `fclose()`
Used to close an opened file.
### Syntax
```c
fclose(file_pointer);
```
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt", "r");
    fclose(fp);
    printf("File closed");
    return 0;
}
```
## 5. `fseek()`
Used to move the file pointer to a specific location.
### Syntax
```c
fseek(file_pointer, offset, position);
```
### Positions
| Position   | Meaning           |
| ---------- | ----------------- |
| `SEEK_SET` | Beginning of file |
| `SEEK_CUR` | Current position  |
| `SEEK_END` | End of file       |
### Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("data.txt", "r");
    fseek(fp, 3, SEEK_SET);
    ch = fgetc(fp);
    printf("%c", ch);
    fclose(fp);
    return 0;
}
```
## Combined Example Program
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char data[] = "HELLO";
    char buffer[10];
    fp = fopen("sample.txt", "wb");
    fwrite(data, sizeof(char), 5, fp);
    fclose(fp);
    fp = fopen("sample.txt", "rb");
    fread(buffer, sizeof(char), 5, fp);
    buffer[5] = '\0';
    printf("Data : %s\n", buffer);
    fseek(fp, 2, SEEK_SET);
    printf("Pointer moved");
    fclose(fp);
    return 0;
}
```
## Important Points
* `fopen()` returns `NULL` if file opening fails.
* `fclose()` should always be used after file operations.
* `fread()` and `fwrite()` are mainly used for binary files.
* `fseek()` helps in random access of files.
* `FILE` is a predefined structure in `stdio.h`.

# Relationship between file descriptor
## Relationship Between File Descriptor and FILE Pointer in C
In C programming, both **File Descriptor** and **FILE Pointer** are used for file handling, but they belong to different levels.
## 1. File Descriptor
A **File Descriptor (FD)** is a low-level integer handle used by the operating system to access files.
* Managed by the OS kernel
* Used in **system calls**
* Works with functions like:
  * `open()`
  * `read()`
  * `write()`
  * `close()`
### Example
```c 
int fd;
fd = open("data.txt", O_RDONLY);
```
Here:
* `fd` may contain values like `3`, `4`, `5`, etc.
* `0` → stdin
* `1` → stdout
* `2` → stderr
## 2. FILE Pointer
A **FILE pointer** is a high-level stream used by the C standard library.
* Managed by `stdio.h`
* Buffered I/O
* Used with:
  * `fopen()`
  * `fread()`
  * `fprintf()`
  * `fclose()`
### Example
```c 
FILE *fp;
fp = fopen("data.txt", "r");
```
## Relationship Between Them
When `fopen()` is used:
1. Internally it calls the OS `open()`
2. OS creates a file descriptor
3. C library creates a `FILE` structure
4. `FILE *` stores the file descriptor internally

So:
```
FILE Pointer  --->  contains File Descriptor
```
## Diagram
```
User Program
     |
     v
 FILE *fp        (High Level Stream)
     |
     v
File Descriptor (Integer FD)
     |
     v
Operating System Kernel
     |
     v
Actual File on Disk
```
## Conversion Between FILE Pointer and File Descriptor
### Convert FILE Pointer → File Descriptor
Using `fileno()`
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    int fd;
    fp = fopen("data.txt", "r");
    fd = fileno(fp);
    printf("File Descriptor = %d", fd);
    fclose(fp);
    return 0;
}
```
## Convert File Descriptor → FILE Pointer
Using `fdopen()`
```c 
#include<stdio.h>
#include<fcntl.h>
#include<unistd.h>
int main()
{
    int fd;
    FILE *fp;
    fd = open("data.txt", O_RDONLY);
    fp = fdopen(fd, "r");
    printf("Conversion done");
    fclose(fp);
    return 0;
}
```
## Difference Between FILE Pointer and File Descriptor
| Feature     | File Descriptor       | FILE Pointer            |
| ----------- | --------------------- | ----------------------- |
| Type        | Integer               | Structure Pointer       |
| Level       | Low-level             | High-level              |
| Header File | `unistd.h`, `fcntl.h` | `stdio.h`               |
| Buffering   | No buffering          | Buffered                |
| Speed       | Faster                | Slightly slower         |
| Functions   | `read()`, `write()`   | `fprintf()`, `fscanf()` |
| Managed By  | OS Kernel             | C Library               |
## Important Interview Points
* `fopen()` internally uses `open()`
* Every `FILE *` contains a file descriptor
* File descriptors are integers
* `stdin = 0`, `stdout = 1`, `stderr = 2`
* `fileno()` gets FD from `FILE *`
* `fdopen()` creates `FILE *` from FD

# and FILE pointer
A **FILE pointer** is a pointer to a structure used for managing files in C.

It is defined in the header file:
```c
#include<stdio.h>
```
## Definition
```c
FILE *fp;
```
Here:
* `FILE` → predefined structure in `stdio.h`
* `fp` → pointer variable pointing to the file
## Purpose of FILE Pointer
A FILE pointer is used to:
* Open a file
* Read data from file
* Write data into file
* Move file position
* Close the file
## Internal Working
When a file is opened using `fopen()`:
1. Memory is allocated for a `FILE` structure
2. File information is stored:
   * File descriptor
   * Buffer
   * File mode
   * Current position
3. Address of structure is returned
```
FILE *fp
    |
    v
+------------------+
| File Descriptor  |
| Buffer           |
| File Position    |
| File Mode        |
+------------------+
```
## Syntax
### Declaration
```c 
FILE *fp;
```
### Opening File
```c 
fp = fopen("data.txt", "r");
```
### Closing File
```c 
fclose(fp);
```
## Example Program
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("sample.txt", "w");
    if(fp == NULL)
    {
        printf("File opening failed");
        return 1;
    }
    fprintf(fp, "Hello");
    fclose(fp);
    return 0;
}
```
## Common File Operations Using FILE Pointer
| Function    | Purpose              |
| ----------- | -------------------- |
| `fopen()`   | Open file            |
| `fclose()`  | Close file           |
| `fprintf()` | Write formatted data |
| `fscanf()`  | Read formatted data  |
| `fgetc()`   | Read character       |
| `fputc()`   | Write character      |
| `fread()`   | Binary read          |
| `fwrite()`  | Binary write         |
| `fseek()`   | Move file pointer    |
## Types of File Opening Modes
| Mode   | Meaning       |
| ------ | ------------- |
| `"r"`  | Read          |
| `"w"`  | Write         |
| `"a"`  | Append        |
| `"r+"` | Read & Write  |
| `"w+"` | Read & Write  |
| `"a+"` | Read & Append |
## Relationship With File Descriptor
```
FILE Pointer  --->  File Descriptor  --->  Actual File
```
* `FILE *` is high-level
* File descriptor is low-level integer
* `FILE *` internally stores file descriptor
## Important Interview Points
* `FILE` is a structure defined in `stdio.h`
* `FILE *` is mandatory for standard file handling
* `fopen()` returns `NULL` on failure
* FILE pointer supports buffered I/O
* Easier to use than low-level system calls
* Internally connected to file descriptor

# Character at a time I/O
Character-at-a-time I/O means reading or writing **one character at a time** from/to a file.

These functions are useful for:
* Text file processing
* Parsing characters
* Counting words/lines
* Copying files character by character
## Functions Used
| Function  | Purpose             |
| --------- | ------------------- |
| `fgetc()` | Read one character  |
| `fputc()` | Write one character |
| `getc()`  | Read one character  |
| `putc()`  | Write one character |
## 1. `fgetc()`
Reads a single character from a file.
### Syntax
```c 
int fgetc(FILE *fp);
```
* Returns character if successful
* Returns `EOF` at end of file
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("data.txt", "r");
    if(fp == NULL)
    {
        printf("File not found");
        return 1;
    }
    while((ch = fgetc(fp)) != EOF)
    {
        printf("%c", ch);
    }
    fclose(fp);
    return 0;
}
```
## 2. `fputc()`
Writes one character into a file.
### Syntax
```c
int fputc(int ch, FILE *fp);
```
## Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt", "w");
    fputc('H', fp);
    fputc('E', fp);
    fputc('L', fp);
    fputc('L', fp);
    fputc('O', fp);
    fclose(fp);
    return 0;
}
```
## 3. `getc()`
Works similar to `fgetc()`.
### Syntax
```c 
getc(FILE *fp);
```
## Example
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    char ch;
    fp = fopen("data.txt", "r");
    while((ch = getc(fp)) != EOF)
    {
        printf("%c", ch);
    }
    fclose(fp);
    return 0;
}
```
## 4. `putc()`
Works similar to `fputc()`.
### Syntax
```c 
putc(character, FILE *fp);
```
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt", "w");
    putc('A', fp);
    putc('B', fp);
    putc('C', fp);
    fclose(fp);
    return 0;
}
```
## Copy File Using Character-at-a-Time I/O
```c 
#include<stdio.h>
int main()
{
    FILE *source, *target;
    char ch;
    source = fopen("source.txt", "r");
    target = fopen("target.txt", "w");
    while((ch = fgetc(source)) != EOF)
    {
        fputc(ch, target);
    }
    printf("File copied successfully");
    fclose(source);
    fclose(target);
    return 0;
}
```
## Difference Between Character and Binary I/O
| Character I/O           | Binary I/O            |
| ----------------------- | --------------------- |
| One character at a time | Block of data         |
| Slower                  | Faster                |
| `fgetc()`, `fputc()`    | `fread()`, `fwrite()` |
| Mainly text files       | Binary files          |
## Important Interview Points
* `EOF` means End Of File
* `fgetc()` returns integer value
* Character I/O is slower because it processes one character at a time
* `getc()` and `putc()` may be implemented as macros
* Suitable for text processing applications

# Line at a time I/O
Line-at-a-time I/O means reading or writing an entire line from/to a file at once.

These functions are mainly used for:
* Reading text files line by line
* Processing sentences
* Handling strings in files
## Functions Used
| Function  | Purpose                 |
| --------- | ----------------------- |
| `fgets()` | Read one line from file |
| `fputs()` | Write one line to file  |
## 1. `fgets()`
Reads a line (string) from a file.
### Syntax
```c 
char *fgets(char *str, int size, FILE *fp);
```
### Parameters
| Parameter | Meaning                    |
| --------- | -------------------------- |
| `str`     | Buffer to store line       |
| `size`    | Maximum characters to read |
| `fp`      | File pointer               |
## Features
* Reads until:
  * Newline `\n`
  * EOF
  * Size-1 characters
* Automatically adds `'\0'`
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    char str[100];
    fp = fopen("data.txt", "r");
    if(fp == NULL)
    {
        printf("File not found");
        return 1;
    }
    while(fgets(str, sizeof(str), fp) != NULL)
    {
        printf("%s", str);
    }
    fclose(fp);
    return 0;
}
```
## 2. `fputs()`
Writes a line/string into a file.
### Syntax
```c 
int fputs(const char *str, FILE *fp);
```
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    fp = fopen("data.txt", "w");
    fputs("Hello World\n", fp);
    fputs("Line I/O Example\n", fp);
    fclose(fp);
    return 0;
}
```
## Program: Copy File Line by Line
```c 
#include<stdio.h>
int main()
{
    FILE *source, *target;
    char str[100];
    source = fopen("source.txt", "r");
    target = fopen("target.txt", "w");
    while(fgets(str, sizeof(str), source) != NULL)
    {
        fputs(str, target);
    }
    printf("File copied successfully");
    fclose(source);
    fclose(target);
    return 0;
}
```
## Difference Between Character I/O and Line I/O
| Character I/O                 | Line I/O                 |
| ----------------------------- | ------------------------ |
| One character at a time       | One line at a time       |
| `fgetc()`, `fputc()`          | `fgets()`, `fputs()`     |
| Slower                        | Faster                   |
| Good for character processing | Good for text processing |
## Important Interview Points
* `fgets()` is safer than `gets()`
* `gets()` is dangerous and removed from modern C
* `fgets()` includes newline character if space exists
* `fputs()` does not automatically add newline
* Line I/O is efficient for text files

# Formatted I/O
Formatted I/O means reading and writing data in a specific format using format specifiers.

These functions are used for:
* Reading formatted input
* Writing formatted output
* Handling integers, floats, strings, etc.
## Functions Used
| Function    | Purpose                       |
| ----------- | ----------------------------- |
| `fprintf()` | Write formatted data to file  |
| `fscanf()`  | Read formatted data from file |
## 1. `fprintf()`
Used to write formatted output into a file.
### Syntax
```c 
fprintf(FILE *fp, "format string", variables);
```
## Example
```c
#include<stdio.h>
int main()
{
    FILE *fp;
    int id = 101;
    float salary = 25000.5;
    fp = fopen("data.txt", "w");
    fprintf(fp, "ID = %d\n", id);
    fprintf(fp, "Salary = %.2f\n", salary);
    fclose(fp);
    return 0;
}
```
## 2. `fscanf()`
Used to read formatted data from a file.
### Syntax
```c 
fscanf(FILE *fp, "format string", &variables);
```
## Example
```c 
#include<stdio.h
int main()
{
    FILE *fp;
    int id;
    float salary;
    fp = fopen("data.txt", "r");
    fscanf(fp, "%d %f", &id, &salary);
    printf("ID = %d\n", id);
    printf("Salary = %.2f", salary);
    fclose(fp)
    return 0;
}
```
## Common Format Specifiers
| Specifier | Data Type   |
| --------- | ----------- |
| `%d`      | Integer     |
| `%f`      | Float       |
| `%c`      | Character   |
| `%s`      | String      |
| `%lf`     | Double      |
| `%x`      | Hexadecimal |
## Program Example
### Write and Read Employee Data
```c 
#include<stdio.h>
int main()
{
    FILE *fp;
    int id;
    char name[20];
    float salary;
    fp = fopen("employee.txt", "w");
    fprintf(fp, "%d %s %f\n", 101, "Ravi", 25000.50);
    fclose(fp);
    fp = fopen("employee.txt", "r");
    fscanf(fp, "%d %s %f", &id, name, &salary);
    printf("ID : %d\n", id);
    printf("Name : %s\n", name);
    printf("Salary : %.2f\n", salary);
    fclose(fp);
    return 0;
}
```
## Difference Between Formatted I/O and Character I/O
| Formatted I/O               | Character I/O                   |
| --------------------------- | ------------------------------- |
| Reads/writes formatted data | Reads/writes single character   |
| `fprintf()`, `fscanf()`     | `fgetc()`, `fputc()`            |
| Easier for structured data  | Easier for character processing |
## Difference Between Formatted I/O and Binary I/O
| Formatted I/O           | Binary I/O            |
| ----------------------- | --------------------- |
| Human-readable          | Machine-readable      |
| Slower                  | Faster                |
| Text format             | Raw bytes             |
| `fprintf()`, `fscanf()` | `fread()`, `fwrite()` |
## Important Interview Points
* `fprintf()` works like `printf()` but writes to file
* `fscanf()` works like `scanf()` but reads from file
* Formatted I/O is mainly used with text files
* Easier to debug because file content is readable
* Binary I/O is faster than formatted I/O

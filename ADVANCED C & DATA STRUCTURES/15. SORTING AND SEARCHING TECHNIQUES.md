# Insertion sort
Insertion Sort is a simple sorting technique that builds the final sorted array one element at a time.

It works similarly to arranging playing cards in your hand:
* Pick one element
* Compare it with previous elements
* Insert it into the correct position
## Algorithm Steps
1. Assume the first element is already sorted.
2. Take the next element.
3. Compare it with elements before it.
4. Shift larger elements one position to the right.
5. Insert the element in the correct place.
6. Repeat until the array is sorted.
## Example
Array:
```
5 3 4 1 2
```
### Pass 1
Insert `3` into sorted part:
```
3 5 4 1 2
```
### Pass 2
Insert `4`:
```
3 4 5 1 2
```
### Pass 3
Insert `1`:
```
1 3 4 5 2
```
### Pass 4
Insert `2`:
```
1 2 3 4 5
```
## C Program
```c
#include <stdio.h>
int main()
{
    int arr[] = {5, 3, 4, 1, 2};
    int n = 5;
    int i, j, key;
    for(i = 1; i < n; i++)
    {
        key = arr[i];
        j = i - 1;
        while(j >= 0 && arr[j] > key)
        {
            arr[j + 1] = arr[j];
            j--;
        }
        arr[j + 1] = key;
    }
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Time Complexity
| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n)            |
| Average Case | O(n²)           |
| Worst Case   | O(n²)           |
## Advantages
* Simple and easy to implement
* Efficient for small datasets
* Stable sorting algorithm
* Requires less memory
## Disadvantages
* Slow for large datasets
* More comparisons and shifts
## Real-Time Uses
* Sorting small lists
* Used when data is nearly sorted
* Used in hybrid sorting algorithms like TimSort
## Key Point
Insertion Sort maintains:
* Left side → sorted
* Right side → unsorted

Each iteration inserts one element into its correct sorted position.

# Selection sort
Selection Sort is a simple sorting algorithm that repeatedly selects the smallest element from the unsorted part and places it at the correct position.
## Working Principle
1. Find the smallest element in the array.
2. Swap it with the first element.
3. Find the next smallest element.
4. Swap it with the next position.
5. Repeat until the array is sorted.
## Example
Array:
```
64 25 12 22 11
```
### Pass 1
Smallest element = `11`

Swap with `64`
```
11 25 12 22 64
```
### Pass 2
Smallest element = `12`

Swap with `25`
```
11 12 25 22 64
```
### Pass 3
Smallest element = `22`

Swap with `25`
```
11 12 22 25 64
```
### Pass 4
Array already sorted
```
11 12 22 25 64
```
## C Program
```c
#include <stdio.h>
int main()
{
    int arr[] = {64, 25, 12, 22, 11};
    int n = 5;
    int i, j, min, temp;
    for(i = 0; i < n - 1; i++)
    {
        min = i;
        for(j = i + 1; j < n; j++)
        {
            if(arr[j] < arr[min])
            {
                min = j;
            }
        }
        temp = arr[i];
        arr[i] = arr[min];
        arr[min] = temp;
    }
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Time Complexity
| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n²)           |
| Average Case | O(n²)           |
| Worst Case   | O(n²)           |
## Space Complexity
```
O(1)
```
## Advantages
* Simple and easy to understand
* Performs fewer swaps
* Works well for small datasets
## Disadvantages
* Inefficient for large datasets
* Time complexity is always O(n²)
* Not a stable sorting algorithm
## Real-Time Uses
* Used when memory writes are costly
* Suitable for small embedded systems
* Helpful in teaching sorting concepts
## Key Point
Selection Sort divides the array into:
* Sorted part
* Unsorted part

It repeatedly selects the minimum element from the unsorted part and places it in the sorted part.

# Bubble sort
Bubble Sort is a simple sorting algorithm that repeatedly compares adjacent elements and swaps them if they are in the wrong order.

Larger elements “bubble” toward the end of the array after each pass.
## Working Principle
1. Compare adjacent elements.
2. Swap them if they are in the wrong order.
3. Continue for the entire array.
4. Repeat the process until no swaps are needed.
## Example
Array:
```
5 1 4 2 8
```
### Pass 1
```
1 5 4 2 8
```
```
1 4 5 2 8
```
```
1 4 2 5 8
```
```
1 4 2 5 8
```
Largest element `8` moved to the end.
## Pass 2
```
1 4 2 5 8
```
```
1 2 4 5 8
```
```
1 2 4 5 8
```
## Pass 3
No swaps needed.

Array is sorted:
```
1 2 4 5 8
```
## C Program
```c
#include <stdio.h>
int main()
{
    int arr[] = {5, 1, 4, 2, 8};
    int n = 5;
    int i, j, temp;
    for(i = 0; i < n - 1; i++)
    {
        for(j = 0; j < n - i - 1; j++)
        {
            if(arr[j] > arr[j + 1])
            {
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Optimized Bubble Sort
This version stops early if the array becomes sorted.
```c 
#include <stdio.h>
int main()
{
    int arr[] = {5, 1, 4, 2, 8};
    int n = 5;
    int i, j, temp, swapped;
    for(i = 0; i < n - 1; i++)
    {
        swapped = 0;
        for(j = 0; j < n - i - 1; j++)
        {
            if(arr[j] > arr[j + 1])
            {
                temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = 1;
            }
        }
        if(swapped == 0)
        {
            break;
        }
    }
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Time Complexity
| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n)            |
| Average Case | O(n²)           |
| Worst Case   | O(n²)           |
## Space Complexity
```
O(1)
```
## Advantages
* Simple to understand
* Easy to implement
* Stable sorting algorithm
* Detects already sorted arrays in optimized version
## Disadvantages
* Very slow for large datasets
* Large number of swaps
* Inefficient compared to advanced sorting algorithms
## Real-Time Uses
* Educational purposes
* Small datasets
* Detecting whether a list is already sorted
## Key Point
After every pass:
* Largest unsorted element moves to the end
* Comparison happens between adjacent elements only

# Merge sort
Merge Sort is a Divide and Conquer sorting algorithm.

It divides the array into smaller subarrays, sorts them, and then merges them back together in sorted order.
## Working Principle
1. Divide the array into two halves.
2. Recursively divide the subarrays until each contains one element.
3. Merge the subarrays in sorted order.
4. Repeat until the complete array is sorted.
## Example
Array:
```
38 27 43 3 9 82 10
```
## Step 1: Divide
```
38 27 43 3 | 9 82 10
```
```
38 27 | 43 3 | 9 82 | 10
```
```
38 | 27 | 43 | 3 | 9 | 82 | 10
```
## Step 2: Merge
```
27 38
```
```
3 43
```
```
9 82
```
## Step 3: Merge Again
```
3 27 38 43
```
```
9 10 82
```
## Final Merge
```
3 9 10 27 38 43 82
```
## C Program
```c 
#include <stdio.h>
void merge(int arr[], int low, int mid, int high)
{
    int i, j, k;
    int n1 = mid - low + 1;
    int n2 = high - mid;
    int L[n1], R[n2];
    for(i = 0; i < n1; i++)
        L[i] = arr[low + i];
    for(j = 0; j < n2; j++)
        R[j] = arr[mid + 1 + j];
    i = 0;
    j = 0;
    k = low;
    while(i < n1 && j < n2)
    {
        if(L[i] <= R[j])
        {
            arr[k] = L[i];
            i++;
        }
        else
        {
            arr[k] = R[j];
            j++;
        }
        k++;
    }
    while(i < n1)
    {
        arr[k] = L[i];
        i++;
        k++;
    }
    while(j < n2)
    {
        arr[k] = R[j];
        j++;
        k++;
    }
}
void mergeSort(int arr[], int low, int high)
{
    if(low < high)
    {
        int mid = (low + high) / 2;
        mergeSort(arr, low, mid);
        mergeSort(arr, mid + 1, high);
        merge(arr, low, mid, high);
    }
}
int main()
{
    int arr[] = {38, 27, 43, 3, 9, 82, 10};
    int n = 7;
    int i;
    mergeSort(arr, 0, n - 1);
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Time Complexity
| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n log n)      |
| Average Case | O(n log n)      |
| Worst Case   | O(n log n)      |
## Space Complexity
```
O(n)
```
## Advantages
* Efficient for large datasets
* Stable sorting algorithm
* Guaranteed O(n log n) performance
* Works well with linked lists
## Disadvantages
* Requires extra memory
* Recursive calls increase overhead
## Real-Time Uses
* External sorting
* Sorting linked lists
* Large data processing systems
* Database sorting operations
## Key Point
Merge Sort follows:
* Divide → Sort → Merge

The array is repeatedly divided into halves and merged back in sorted order.

# Quick sort
Quick Sort is a fast Divide and Conquer sorting algorithm.

It selects a pivot element and partitions the array such that:
* Elements smaller than the pivot go to the left
* Elements greater than the pivot go to the right

Then the same process is repeated recursively.
## Working Principle
1. Choose a pivot element.
2. Partition the array around the pivot.
3. Place pivot in its correct position.
4. Recursively sort left and right subarrays.
## Example
Array:
```
10 7 8 9 1 5
```
## Step 1: Choose Pivot
Pivot = `5`

Partition:
```
1 5 10 7 8 9
```
## Step 2: Sort Left and Right Parts
Left side already sorted:
```
1
```
Right side:
```
10 7 8 9
```
Pivot = `9`

Partition:
```
7 8 9 10
```
## Final Sorted Array
```
1 5 7 8 9 10
```
## C Program
```c 
#include <stdio.h>
void swap(int *a, int *b)
{
    int temp;
    temp = *a;
    *a = *b;
    *b = temp;
}
int partition(int arr[], int low, int high)
{
    int pivot, i, j;
    pivot = arr[high];
    i = low - 1;
    for(j = low; j < high; j++)
    {
        if(arr[j] < pivot)
        {
            i++;
            swap(&arr[i], &arr[j]);
        }
    }
    swap(&arr[i + 1], &arr[high]);
    return i + 1;
}
void quickSort(int arr[], int low, int high)
{
    if(low < high)
    {
        int pi;
        pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);
        quickSort(arr, pi + 1, high);
    }
}
int main()
{
    int arr[] = {10, 7, 8, 9, 1, 5};
    int n = 6;
    int i;
    quickSort(arr, 0, n - 1);
    printf("Sorted array:\n");
    for(i = 0; i < n; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Time Complexity
| Case         | Time Complexity |
| ------------ | --------------- |
| Best Case    | O(n log n)      |
| Average Case | O(n log n)      |
| Worst Case   | O(n²)           |
Worst case occurs when:
* Pivot is always smallest or largest element
* Example: already sorted array
## Space Complexity
```
O(log n)
```
## Advantages
* Very fast in practice
* Efficient for large datasets
* In-place sorting algorithm
* Requires less memory
## Disadvantages
* Worst-case time complexity is O(n²)
* Recursive implementation
* Not a stable sorting algorithm
## Real-Time Uses
* Used in standard library sorting functions
* Database systems
* Operating systems
* Large-scale data sorting
## Key Point
Quick Sort follows:
* Pick Pivot
* Partition
* Recursively Sort

The pivot divides the array into smaller subarrays for efficient sorting.

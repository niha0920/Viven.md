# Introduction
## Introduction to Data Structures
A **Data Structure** is a way of organizing, storing, and managing data in a computer so that it can be accessed and modified efficiently.

It helps in:
* Storing large amounts of data
* Performing operations quickly
* Managing memory efficiently
* Improving program performance
## Why Data Structures are Important
Data structures are used to:
* Search data quickly
* Insert and delete data efficiently
* Organize information logically
* Reduce execution time and memory usage
Real-time examples:
* Contact list in mobile → Arrays/Linked Lists
* Browser back button → Stack
* Printer queue → Queue
* GPS navigation → Graph
* Database indexing → Trees
## Classification of Data Structures
### 1. Primitive Data Structures
Basic built-in data types.

Examples:
* `int`
* `char`
* `float`
* `double`
* `pointer`
Example:
```c
int a = 10;
char ch = 'A';
```
## 2. Non-Primitive Data Structures
These are derived from primitive data types.

They are divided into:
### A. Linear Data Structures
Elements are arranged sequentially.

Examples:
1. Array
2. Linked List
3. Stack
4. Queue
### B. Non-Linear Data Structures
Elements are connected hierarchically or randomly.

Examples:
1. Tree
2. Graph
## Common Data Structures
| Data Structure | Description                                          | Real-Time Example   |
| -------------- | ---------------------------------------------------- | ------------------- |
| Array          | Collection of same type elements stored continuously | Student marks       |
| Linked List    | Nodes connected using pointers                       | Music playlist      |
| Stack          | Follows LIFO (Last In First Out)                     | Browser history     |
| Queue          | Follows FIFO (First In First Out)                    | Ticket booking line |
| Tree           | Hierarchical structure                               | File system         |
| Graph          | Network-like structure                               | Google Maps         |
## Operations on Data Structures
Common operations:
1. Traversal → Access elements
2. Insertion → Add elements
3. Deletion → Remove elements
4. Searching → Find elements
5. Sorting → Arrange elements
6. Merging → Combine structures
## Characteristics of a Good Data Structure
* Efficient memory usage
* Fast processing
* Simplicity
* Scalability
* Reusability
## Difference Between Data Type and Data Structure
| Data Type            | Data Structure              |
| -------------------- | --------------------------- |
| Defines type of data | Organizes data              |
| Example: int, float  | Example: array, stack       |
| Stores single value  | Stores collection of values |
## Applications of Data Structures
* Operating Systems

* Artificial Intelligence

# Structured Representation
A **Structured Representation** means organizing data in a systematic and logical format so that it becomes easy to:
* Store
* Access
* Process
* Modify
Instead of keeping data randomly, data structures represent data in an organized manner.
## Need for Structured Representation
Structured representation helps in:
* Efficient memory usage
* Faster searching and sorting
* Easy data management
* Better program readability
* Reduced complexity
Example:

Instead of storing student data separately,
```
101
Ravi
85
```
we organize it as:
```
Student
{
   RollNo = 101
   Name = Ravi
   Marks = 85
}
```
This is a structured representation.
## Types of Structured Representation
### 1. Linear Representation
Data elements are arranged sequentially.
#### Characteristics
* One-by-one arrangement
* Easy traversal
* Single level organization
#### Examples
* Array
* Linked List
* Stack
* Queue
#### Example Array Representation
```
Index : 0   1   2   3   4
Value : 10  20  30  40  50
```
## 2. Non-Linear Representation
Data elements are arranged hierarchically or interconnected.
### Characteristics
* Multiple relationships
* Complex organization
* Efficient for large systems
### Examples
* Tree
* Graph
### Example Tree Representation
```
        A
      /   \
     B     C
    / \
   D   E
```
## Structured Representation Using Structures in C
C language provides `struct` for structured representation.
## Example
```c
#include<stdio.h>
struct Student
{
    int rollno;
    char name[20];
    float marks;
};
int main()
{
    struct Student s1 = {101, "Ravi", 85.5};
    printf("Roll No : %d\n", s1.rollno);
    printf("Name    : %s\n", s1.name);
    printf("Marks   : %.2f\n", s1.marks);
    return 0;
}
```
Output:
```
Roll No : 101
Name    : Ravi
Marks   : 85.50
```

# Statements and Control Structures
A **statement** in C is an instruction given to the computer to perform a specific task.

A **control structure** determines the flow of execution of statements in a program.

They help in:
* Decision making
* Repetition of tasks
* Sequential execution
* Controlling program flow
## Types of Statements in C
### 1. Declaration Statement
Used to declare variables.
```c 
int a;
float b;
char ch;
```
## 2. Assignment Statement
Used to assign values.
```c
a = 10;
b = 5.5;
```
## 3. Input/Output Statements
Used for user interaction.
```c
printf("Hello");
scanf("%d", &a);
```
## 4. Compound Statement
Group of statements enclosed within `{ }`.
```c 
{
    int a = 10;
    printf("%d", a);
}
```
## Control Structures in C
Control structures are divided into:
1. Sequential Control
2. Selection Control
3. Iteration Control
4. Jump Control
## 1. Sequential Control Structure
Statements execute one after another in order.
## Example
```c
#include<stdio.h>
int main()
{
    int a = 10;
    int b = 20;
    int c;
    c = a + b;
    printf("%d", c);
    return 0;
}
```
Output:
```
30
```
## 2. Selection (Decision-Making) Control Structure
Used to make decisions based on conditions.

Types:
* if
* if-else
* nested if
* else-if ladder
* switch
## if Statement
Executes block only if condition is true.
### Syntax
```c
if(condition)
{
    statements;
}
```
### Example
```c 
#include<stdio.h>
int main()
{
    int age = 20;
    if(age >= 18)
    {
        printf("Eligible to vote");
    }
    return 0;
}
```
## if-else Statement
Executes one block if condition is true, otherwise another block.
### Syntax
```c 
if(condition)
{
    statements;
}
else
{
    statements;
}
```
### Example
```c
#include<stdio.h>
int main()
{
    int n = 5;
    if(n % 2 == 0)
    {
        printf("Even");
    }
    else
    {
        printf("Odd");
    }
    return 0;
}
```
## else-if Ladder
Used for multiple conditions.
### Example
```c
#include<stdio.h>
int main()
{
    int marks = 75;
    if(marks >= 90)
    {
        printf("Grade A");
    }
    else if(marks >= 70)
    {
        printf("Grade B");
    }
    else
    {
        printf("Grade C");
    }
    return 0;
}
```
## switch Statement
Used for multiple fixed choices.
### Syntax
```c 
switch(expression)
{
    case value:
        statements;
        break;
    default:
        statements;
}
```
### Example
```c 
#include<stdio.h>
int main()
{
    int day = 2;
    switch(day)
    {
        case 1:
            printf("Monday");
            break;
        case 2:
            printf("Tuesday");
            break;
        default:
            printf("Invalid");
    }
    return 0;
}
```
## 3. Iteration (Looping) Control Structure
Used to repeat statements multiple times.

Types:
* for loop
* while loop
* do-while loop
## for Loop
Used when number of iterations is known.
### Syntax
```c 
for(initialization; condition; increment)
{
    statements;
}
```
### Example
```c
#include<stdio.h>
int main()
{
    int i;
    for(i = 1; i <= 5; i++)
    {
        printf("%d ", i);
    }
    return 0;
}
```
Output:
```
1 2 3 4 5
```
## while Loop
Checks condition before execution.
### Syntax
```c 
while(condition)
{
    statements;
}
```
### Example
```c 
#include<stdio.h>
int main()
{
    int i = 1;
    while(i <= 5)
    {
        printf("%d ", i);
        i++;
    }
    return 0;
}
```
## do-while Loop
Executes at least once.
### Syntax
```c 
do
{
    statements;
}while(condition);
```
### Example
```c
#include<stdio.h>
int main()
{
    int i = 1;
    do
    {
        printf("%d ", i);
        i++;
    }while(i <= 5);
    return 0;
}
```
## 4. Jump Control Structures
Used to transfer program control.

Types:
* break
* continue
* goto
* return
## break Statement
Terminates loop or switch.
```c 
for(int i = 1; i <= 10; i++)
{
    if(i == 5)
        break;
    printf("%d ", i);
}
```
Output:
```
1 2 3 4
```
## continue Statement
Skips current iteration.
```c 
for(int i = 1; i <= 5; i++)
{
    if(i == 3)
        continue;
    printf("%d ", i);
}
```
Output:
```
1 2 4 5
```
## goto Statement
Transfers control to labeled statement.
```c 
#include<stdio.h>
int main()
{
    int i = 1;
label:
    printf("%d ", i);
    i++;
    if(i <= 5)
        goto label;
    return 0;
}
```
## return Statement
Exits from function.
```c 
return 0;
```
## Difference Between Looping Statements
| Loop     | Condition Check | Executes Minimum |
| -------- | --------------- | ---------------- |
| for      | Beginning       | 0 times          |
| while    | Beginning       | 0 times          |
| do-while | End             | 1 time           |
## Real-Time Uses of Control Structures
| Control Structure | Real-Time Example        |
| ----------------- | ------------------------ |
| if-else           | ATM balance check        |

| while loop        | Reading data until EOF   |

# Abstract Data Types
An **Abstract Data Type (ADT)** is a logical model of a data structure that defines:
* What data can be stored
* What operations can be performed on the data
but **does not specify how the operations are implemented**.

ADT focuses on:
* **Behavior**
* **Operations**
* **Functionality**
instead of internal implementation details.
## Simple Definition
An ADT tells:
* **What to do**
  but not
* **How to do it**
## Real-Time Example
### ATM Machine
You can:
* Withdraw money
* Deposit money
* Check balance
But you do not know:
* Internal banking server logic
* Database implementation
This is abstraction.
## Components of ADT
An ADT mainly contains:
1. **Data**
2. **Operations on data**
## Characteristics of ADT
* Hides implementation details
* Provides only essential operations
* Improves security
* Increases modularity
* Easy maintenance
* Reusable code
## Common Abstract Data Types
| ADT   | Operations                     |
| ----- | ------------------------------ |
| Stack | push(), pop(), peek()          |
| Queue | enqueue(), dequeue()           |
| List  | insert(), delete(), search()   |
| Tree  | insert(), traverse(), delete() |
| Graph | addVertex(), addEdge()         |
## Example: Stack ADT
A stack follows **LIFO** principle.
### Operations of Stack ADT
| Operation | Purpose            |
| --------- | ------------------ |
| push()    | Insert element     |
| pop()     | Remove top element |
| peek()    | View top element   |
| isEmpty() | Check empty stack  |
| isFull()  | Check full stack   |
## Stack ADT Representation
```
        TOP
         ↓
       -------
       | 30 |
       -------
       | 20 |
       -------
       | 10 |
       -------
```
## ADT vs Data Structure
| Abstract Data Type                  | Data Structure                   |
| ----------------------------------- | -------------------------------- |
| Logical view                        | Physical implementation          |
| What operations are performed       | How operations are implemented   |
| Independent of programming language | Language dependent               |
| Example: Stack                      | Array/Linked List implementation |
## Example of ADT in C
### Stack Implementation Using Array
```c
#include<stdio.h>
#define SIZE 5
int stack[SIZE];
int top = -1;
void push(int value)
{
    if(top == SIZE - 1)
    {
        printf("Stack Overflow\n");
    }
    else
    {
        top++;
        stack[top] = value;
    }
}
void pop()
{
    if(top == -1)
    {
        printf("Stack Underflow\n");
    }
    else
    {
        printf("Deleted: %d\n", stack[top]);
        top--;
    }
}
void display()
{
    int i;
    for(i = top; i >= 0; i--)
    {
        printf("%d\n", stack[i]);
    }
}
int main()
{
    push(10);
    push(20);
    push(30);
    display();
    pop();
    return 0;
}
```
## Types of ADT
### 1. Linear ADT
Elements arranged sequentially.

Examples:
* Stack
* Queue
* List
## 2. Non-Linear ADT
Elements arranged hierarchically.

Examples:
* Tree
* Graph
## Advantages of ADT
* Data hiding
* Security
* Flexibility
* Easy debugging
* Code reusability
* Modular programming
## Disadvantages of ADT
* Extra complexity
* Slight performance overhead
* Requires proper design
## Real-Time Applications of ADT
| ADT   | Application                     |
| ----- | ------------------------------- |
| Stack | Function calls, browser history |
| Queue | Printer queue, scheduling       |
| Tree  | File systems, databases         |
| Graph | Social networks, maps           |
| List  | Music playlist                  |
## Importance of ADT in Data Structures
ADT helps programmers:
* Separate logic from implementation
* Build scalable applications
* Improve software design
* Simplify complex systems

# Static & Dynamic Data Types
Data types in programming can be classified based on how memory is allocated and managed.

They are mainly divided into:
1. Static Data Types
2. Dynamic Data Types
## 1. Static Data Types
A **Static Data Type** is a data type whose memory size is fixed before program execution and cannot be changed during runtime.

Memory is allocated at:
* Compile time
  or
* Before execution
## Characteristics of Static Data Types
* Fixed memory size
* Faster access
* Simple implementation
* Less flexible
* Memory may be wasted
## Examples of Static Data Types
* Integer
* Character
* Float
* Arrays with fixed size
* Structures with fixed members
## Example of Static Data Type
```c
#include<stdio.h>
int main()
{
    int a = 10;
    float b = 5.5;
    char ch = 'A';
    printf("%d\n", a);
    printf("%.1f\n", b);
    printf("%c\n", ch);
    return 0;
}
```
## Static Array Example
```c
#include<stdio.h>
int main()
{
    int arr[5] = {10,20,30,40,50};
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
## Memory Representation of Static Array
```
Index : 0   1   2   3   4
Value : 10  20  30  40  50
```
Size is fixed and cannot be increased during execution.
## Advantages of Static Data Types
* Faster execution
* Easy to implement
* Better performance
* Direct memory access
## Disadvantages of Static Data Types
* Fixed size
* Memory wastage possible
* Less flexible
* Difficult to resize
## 2. Dynamic Data Types
A **Dynamic Data Type** is a data type whose memory size can change during program execution.

Memory is allocated:
* During runtime
using:
* `malloc()`
* `calloc()`
* `realloc()`
* `free()`
## Characteristics of Dynamic Data Types
* Flexible size
* Memory allocated as needed
* Efficient memory utilization
* More complex implementation
# Examples of Dynamic Data Types
* Dynamic arrays
* Linked Lists
* Trees
* Graphs
* Queues using linked lists
## Dynamic Memory Allocation Functions
| Function  | Purpose                   |
| --------- | ------------------------- |
| malloc()  | Allocates single block    |
| calloc()  | Allocates multiple blocks |
| realloc() | Resizes memory            |
| free()    | Deallocates memory        |
# Example Using malloc()
```c 
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    int n = 5;
    ptr = (int *)malloc(n * sizeof(int));
    if(ptr == NULL)
    {
        printf("Memory not allocated");
        return 0;
    }
    for(int i = 0; i < n; i++)
    {
        ptr[i] = (i + 1) * 10;
    }
    for(int i = 0; i < n; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
Output:
```
10 20 30 40 50
```
## Dynamic Memory Representation
```
Heap Memory
-------------
| 10 |
| 20 |
| 30 |
| 40 |
| 50 |
-------------
```
Memory is allocated during runtime from heap memory.
## Difference Between Static and Dynamic Data Types
| Static Data Type                    | Dynamic Data Type    |
| ----------------------------------- | -------------------- |
| Fixed size                          | Variable size        |
| Compile-time allocation             | Runtime allocation   |
| Faster access                       | Slightly slower      |
| Less flexible                       | Highly flexible      |
| May waste memory                    | Efficient memory use |
| Stored mainly in stack/data segment | Stored in heap       |
## Real-Time Examples
| Static                 | Dynamic              |
| ---------------------- | -------------------- |
| Fixed employee records | Social media feeds   |
| Array of 100 students  | Dynamic contact list |
| Static menu system     | Browser tabs         |
| Fixed sensor buffer    | Linked list queues   |
## Memory Areas in C
```
-----------------
| Code Segment  |
-----------------
| Global/Data   |
-----------------
| Stack Memory  | ← Static variables/local variables
-----------------
| Heap Memory   | ← Dynamic allocation
-----------------
```
## When to Use Static Data Types
Use when:
* Size is known beforehand
* Fast access required
* Simplicity is important
Example:
* Fixed-size buffers
* Embedded systems arrays
## When to Use Dynamic Data Types
Use when:
* Size changes frequently
* Memory optimization needed
* Large data handling required
Example:
* Linked lists
* Trees
* Dynamic databases

# Arrays & Pointers
Arrays and pointers are closely related concepts in C programming.
Understanding their relationship is very important for:
* Data structures
* Dynamic memory allocation
* String handling
* Function arguments
## Arrays
An **array** is a collection of elements of the same data type stored in contiguous memory locations.
## Declaration of Array
### Syntax
```c 
datatype array_name[size];
```
### Example
```c
int arr[5];
```
This creates an integer array of size 5.
## Initialization of Array
```c 
int arr[5] = {10,20,30,40,50};
```
## Accessing Array Elements
Array elements are accessed using index values.
```c 
arr[0] = 10;
arr[1] = 20;
```
## Example Program Using Array
```c 
#include<stdio.h>
int main()
{
    int arr[5] = {10,20,30,40,50};
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
Output:
```
10 20 30 40 50
```
## Memory Representation of Array
```
Address → 1000 1004 1008 1012 1016
Value   →  10   20   30   40   50
```
Each integer typically occupies 4 bytes.
## Pointers
A **pointer** is a variable that stores the address of another variable.
## Declaration of Pointer
### Syntax
```c
datatype *pointer_name;
```
### Example
```c
int *ptr;
```
## Example of Pointer
```c 
#include<stdio.h>
int main()
{
    int a = 10;
    int *ptr;
    ptr = &a;
    printf("Value of a = %d\n", a);
    printf("Address of a = %p\n", &a);
    printf("Value stored in ptr = %p\n", ptr);
    printf("Value pointed by ptr = %d\n", *ptr);
    return 0;
}
```
## Operators Used with Pointers
| Operator | Meaning                        |
| -------- | ------------------------------ |
| `&`      | Address-of operator            |
| `*`      | Value at address (dereference) |
## Relationship Between Arrays and Pointers
The name of an array acts like a pointer to the first element.

Example:
```c 
int arr[5] = {10,20,30,40,50};
```
Here:
```c 
arr == &arr[0]
```
Both point to the first element.
## Example Showing Array-Pointer Relationship
```c 
#include<stdio.h>
int main()
{
    int arr[5] = {10,20,30,40,50};
    printf("%d\n", arr[0]);
    printf("%d\n", *arr);
    return 0;
}
```
Output:
```
10
10
```
## Pointer Arithmetic
Pointers can move between memory locations.
## Example
```c 
#include<stdio.h>
int main()
{
    int arr[5] = {10,20,30,40,50};
    int *ptr = arr;
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", *(ptr + i));
    }
    return 0;
}
```
Output:
```
10 20 30 40 50
```
## Array Access Using Pointers
| Array Form | Pointer Form |
| ---------- | ------------ |
| `arr[i]`   | `*(arr + i)` |
| `&arr[i]`  | `(arr + i)`  |
## Example
```c 
#include<stdio.h>
int main()
{
    int arr[3] = {100,200,300};
    printf("%d\n", arr[1]);
    printf("%d\n", *(arr + 1));
    return 0;
}
```
Output:
```
200
200
```
## Passing Arrays to Functions
Arrays are passed as pointers to functions.
## Example
```c 
#include<stdio.h>
void display(int *arr, int size)
{
    for(int i = 0; i < size; i++)
    {
        printf("%d ", arr[i]);
    }
}
int main()
{
    int arr[5] = {1,2,3,4,5};
    display(arr, 5);
    return 0;
}
```
## Array of Pointers
An array can store multiple pointers.
## Example
```c
#include<stdio.h>
int main()
{
    char *names[] = {"Ram", "Sam", "John"};
    for(int i = 0; i < 3; i++)
    {
        printf("%s\n", names[i]);
    }
    return 0;
}
```
## Pointer to Array
A pointer can point to an entire array.
## Example
```c 
#include<stdio.h>
int main()
{
    int arr[3] = {10,20,30};
    int (*ptr)[3] = &arr;
    printf("%d\n", (*ptr)[1]);
    return 0;
}
```
## Difference Between Array and Pointer
| Array                          | Pointer                   |
| ------------------------------ | ------------------------- |
| Collection of elements         | Stores address            |
| Fixed size                     | Can point anywhere        |
| Memory allocated automatically | Needs separate assignment |
| Cannot be reassigned           | Can be reassigned         |
## Advantages of Arrays
* Fast access
* Easy traversal
* Efficient memory usage
## Disadvantages of Arrays
* Fixed size
* Insertion/deletion difficult
* Memory wastage possible
## Advantages of Pointers
* Dynamic memory allocation
* Efficient parameter passing
* Useful for linked structures
## Real-Time Applications
| Arrays           | Pointers          |
| ---------------- | ----------------- |
| Student marks    | Dynamic memory    |
| Image processing | Linked lists      |
| Buffers          | Trees and graphs  |
| Matrices         | Function pointers |
## Important Points
1. Array name is a constant pointer.
2. Arrays store elements sequentially.
3. Pointer arithmetic depends on data type size.
4. `arr[i]` and `*(arr+i)` are equivalent.

# Linked List
A **Linked List** is a dynamic linear data structure where elements are stored in separate memory locations and connected using pointers.

Each element in a linked list is called a **node**.
## Structure of a Node
A node contains:
1. Data
2. Address of next node
```
 -----------------
| Data | Next     |
 -----------------
```
## Why Linked List?
Arrays have limitations:
* Fixed size
* Memory wastage
* Difficult insertion/deletion
Linked lists solve these problems using dynamic memory allocation.
## Characteristics of Linked List
* Dynamic size
* Non-contiguous memory allocation
* Efficient insertion/deletion
* Sequential access only
## Representation of Linked List
```
[10|1004] -> [20|1020] -> [30|1040] -> NULL
```
Each node stores:
* Data
* Address of next node
## Types of Linked Lists
1. Singly Linked List
2. Doubly Linked List
3. Circular Linked List
4. Circular Doubly Linked List
## 1. Singly Linked List
Each node points to the next node only.
```
[10| ] -> [20| ] -> [30| ] -> NULL
```
## Structure Definition in C
```c 
struct Node
{
    int data;
    struct Node *next;
};
```
## Creating a Node
```c 
struct Node *newNode;
newNode = (struct Node *)malloc(sizeof(struct Node));
```
## Example: Creating and Displaying Linked List
```c 
#include<stdio.h>
#include<stdlib.h>
struct Node
{
    int data;
    struct Node *next;
};
int main()
{
    struct Node *head = NULL;
    struct Node *second = NULL;
    struct Node *third = NULL;
    head = (struct Node *)malloc(sizeof(struct Node));
    second = (struct Node *)malloc(sizeof(struct Node));
    third = (struct Node *)malloc(sizeof(struct Node));
    head->data = 10;
    head->next = second;
    second->data = 20;
    second->next = third;
    third->data = 30;
    third->next = NULL;
    struct Node *temp = head;
    while(temp != NULL)
    {
        printf("%d ", temp->data);
        temp = temp->next;
    }
    return 0;
}
```
Output:
```
10 20 30
```
## Operations on Linked List
Common operations:
1. Traversal
2. Insertion
3. Deletion
4. Searching
5. Updating
## Traversal
Visiting each node one by one.
```
HEAD → Node1 → Node2 → Node3 → NULL
```
## Insertion Operations
Insertion can be done:
1. At beginning
2. At end
3. At specific position
## Insertion at Beginning
```
Before:
10 -> 20 -> 30

After inserting 5:
5 -> 10 -> 20 -> 30
```
# Example: Insert at Beginning
```c 
#include<stdio.h>
#include<stdlib.h>
struct Node
{
    int data;
    struct Node *next;
};
struct Node *head = NULL;
void insertBegin(int value)
{
    struct Node *newNode;
    newNode = (struct Node *)malloc(sizeof(struct Node));
    newNode->data = value;
    newNode->next = head;
    head = newNode;
}
void display()
{
    struct Node *temp = head;
    while(temp != NULL)
    {
        printf("%d ", temp->data);
        temp = temp->next;
    }
}
int main()
{
    insertBegin(30);
    insertBegin(20);
    insertBegin(10);
    display();
    return 0;
}
```
Output:
```
10 20 30
```
## Deletion in Linked List
Deletion can be done:
* From beginning
* From end
* From specific position
## Deletion from Beginning
```
Before:
10 -> 20 -> 30

After deletion:
20 -> 30
```
## Example: Delete from Beginning
```c 
void deleteBegin()
{
    struct Node *temp;
    if(head == NULL)
    {
        printf("List Empty");
    }
    else
    {
        temp = head;
        head = head->next;
        free(temp);
    }
}
```
## Searching in Linked List
Search element node by node.
## Example
```c 
void search(int key)
{
    struct Node *temp = head;
    while(temp != NULL)
    {
        if(temp->data == key)
        {
            printf("Element Found");
            return;
        }
        temp = temp->next;
    }
    printf("Element Not Found");
}
```
## 2. Doubly Linked List
Each node contains:
* Previous pointer
* Data
* Next pointer
```
NULL <- [10] <-> [20] <-> [30] -> NULL
```
## Advantages of Doubly Linked List
* Traversal in both directions
* Easy deletion
## Disadvantages
* Extra memory for previous pointer
* More complex
## 3. Circular Linked List
Last node points back to first node.
```
10 -> 20 -> 30
^             |
|_____________|
```
## Advantages of Circular Linked List
* Continuous traversal possible
* Efficient for round-robin scheduling

## Advantages of Linked List
* Dynamic size
* Efficient insertion/deletion
* No memory wastage
* Flexible memory allocation
## Disadvantages of Linked List
* Extra memory for pointers
* Sequential access only
* Slower traversal than arrays
## Array vs Linked List
| Array                        | Linked List             |
| ---------------------------- | ----------------------- |
| Fixed size                   | Dynamic size            |
| Fast random access           | Sequential access       |
| Difficult insertion/deletion | Easy insertion/deletion |
| Contiguous memory            | Non-contiguous memory   |
## Time Complexity
| Operation              | Time Complexity |
| ---------------------- | --------------- |
| Traversal              | O(n)            |
| Insertion at beginning | O(1)            |
| Insertion at end       | O(n)            |
| Deletion               | O(n)            |
| Searching              | O(n)            |

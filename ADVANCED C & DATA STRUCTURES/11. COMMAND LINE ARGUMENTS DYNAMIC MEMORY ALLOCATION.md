# Allocation (Malloc, Calloc, Realloc)
## Dynamic Memory Allocation in C
Dynamic memory allocation is used to allocate memory at **runtime** from the **heap memory**.
It allows programs to create variables, arrays, and data structures dynamically.
###
Header file required:
```c
#include <stdlib.h>
```
## 1. malloc()
`malloc()` stands for **memory allocation**.
###
It allocates a block of memory of specified size and returns a pointer to the first byte.
### Syntax
```c
ptr = (datatype *)malloc(size_in_bytes);
```
### Example
```c
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    int n = 5;
    ptr = (int *)malloc(n * sizeof(int));
    if(ptr == NULL)
    {
        printf("Memory not allocated\n");
        return 0;
    }
    for(int i = 0; i < n; i++)
    {
        ptr[i] = i + 1;
    }
    for(int i = 0; i < n; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
1 2 3 4 5
```
### Important Points
- Memory contains **garbage values**
- Allocates **single block**
- Returns `NULL` if allocation fails
- Memory must be released using `free()`
## 2. calloc()
`calloc()` stands for **contiguous allocation**.
###
It allocates memory for multiple elements and initializes all bytes to `0`.
### Syntax
```c
ptr = (datatype *)calloc(number_of_elements, size_of_each_element);
```
### Example
```c
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    int n = 5;
    ptr = (int *)calloc(n, sizeof(int));
    if(ptr == NULL)
    {
        printf("Memory not allocated\n");
        return 0;
    }
    for(int i = 0; i < n; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
0 0 0 0 0
```
### Important Points
- Memory initialized with `0`
- Allocates contiguous memory
- Slightly slower than `malloc()`
- Used when initialization is required
## 3. realloc()
`realloc()` stands for **re-allocation**.
It changes the size of previously allocated memory block.
### Syntax
```c
ptr = (datatype *)realloc(ptr, new_size);
```
### Example
```c
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    int n = 3;
    ptr = (int *)malloc(n * sizeof(int));
    for(int i = 0; i < n; i++)
    {
        ptr[i] = i + 1;
    }
    n = 5;
    ptr = (int *)realloc(ptr, n * sizeof(int));
    for(int i = 3; i < n; i++)
    {
        ptr[i] = i + 1;
    }
    for(int i = 0; i < n; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
1 2 3 4 5
```
### Important Points
- Can increase or decrease memory size
- Existing data is preserved
- May move memory to another location
- If failed, returns `NULL`
## Difference Between malloc(), calloc(), realloc()
| Feature        | malloc()          | calloc()              | realloc()           |
| -------------- | ----------------- | --------------------- | ------------------- |
| Full Form      | Memory Allocation | Contiguous Allocation | Re-Allocation       |
| Initialization | Garbage values    | Zero initialized      | Keeps old data      |
| Arguments      | 1                 | 2                     | 2                   |
| Purpose        | Allocate memory   | Allocate + initialize | Resize memory       |
| Speed          | Faster            | Slightly slower       | Depends on resizing |
## free()
Used to release dynamically allocated memory.
### Syntax
```c
free(ptr);
```
### Example
```c
free(ptr);
ptr = NULL;
```
Setting pointer to `NULL` avoids dangling pointers.
## Real-Time Uses
- Dynamic arrays
- Linked lists
- Queues and stacks
- Trees and graphs
- Buffer allocation
- Multimedia streaming
- Network packet handling
- Device driver memory management

# De-Allocation (Free)
## De-Allocation in C using free()
`free()` is used to release dynamically allocated memory back to the operating system.

When memory is allocated using:
* `malloc()`
* `calloc()`
* `realloc()`
it remains occupied until `free()` is called.
## Syntax
```c 
free(ptr);
```
Where:
* `ptr` → pointer pointing to dynamically allocated memory
## Why free() is Important?
If memory is not released:
* Memory gets wasted
* Program memory usage increases
* Causes memory leaks
* System performance decreases
## Example using malloc() and free()
```c 
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)malloc(5 * sizeof(int));
    if(ptr == NULL)
    {
        printf("Memory allocation failed");
        return 0;
    }
    for(int i = 0; i < 5; i++)
    {
        ptr[i] = i + 1;
    }
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
1 2 3 4 5
```
## What Happens Internally?
Before `free()`:
```
ptr ---> Heap Memory
```
After `free(ptr)`:
```
Heap memory released
ptr becomes dangling pointer
```
The pointer still contains old address but memory is invalid.
## Dangling Pointer
A pointer pointing to freed memory is called a dangling pointer.
### Example
```c
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)malloc(sizeof(int));
    *ptr = 100;
    free(ptr);
    printf("%d", *ptr);
    return 0;
}
```
This causes undefined behavior.
## Safe Practice
After freeing memory:
```c 
free(ptr);
ptr = NULL;
```
This prevents accidental access.
## Freeing NULL Pointer
```c
free(NULL);
```
This is safe in C.

No error occurs.
## Double Free Problem
Freeing same memory twice causes undefined behavior.
### Wrong Example
```c
free(ptr);
free(ptr);
```
May crash the program.
## Correct Way
```c
free(ptr);
ptr = NULL;
```
## Example with calloc()
```c 
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)calloc(5, sizeof(int));
    if(ptr == NULL)
    {
        return 0;
    }
    free(ptr);
    ptr = NULL;
    return 0;
}
```
## Example with realloc()
```c
#include<stdio.h>
#include<stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)malloc(3 * sizeof(int));
    ptr = (int *)realloc(ptr, 5 * sizeof(int));
    free(ptr);
    return 0;
}
```
## Memory Leak
Memory allocated but not freed is called memory leak.
### Example
```c 
void fun()
{
    int *ptr;
    ptr = (int *)malloc(100 * sizeof(int));
}
```
Memory is lost after function ends.
## Real-Time Problems due to Memory Leaks
* System slowdown
* Embedded system crash
* Application freeze
* Server failure
* Out of memory error
## Real-Time Uses of free()
* Releasing buffers after usage
* Network packet cleanup
* Multimedia frame de-allocation
* Linked list node deletion
* Tree node deletion
* Dynamic array cleanup

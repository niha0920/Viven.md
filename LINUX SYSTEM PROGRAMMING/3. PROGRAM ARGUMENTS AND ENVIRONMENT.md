# Program Startup
**Program Startup** refers to what happens from the moment you execute a program until your `main()` function starts running.

When we run a C program in Linux:
```
./a.out
```
the program does **not directly start at `main()`**. The operating system and C runtime perform several initialization steps before control reaches `main()`.
## Steps in Program Startup
### 1. User Executes the Program
The shell (like Bash) runs the executable.

Example:
```
./program
```
The shell internally creates a new process using system calls like:
* `fork()` → creates a child process
* `exec()` family → loads the executable into memory
## 2. Kernel Loads the Executable
The Linux kernel:
* Loads the executable file into memory
* Creates a process image
* Sets up:
  * **Text segment** → program instructions
  * **Data segment** → initialized global/static variables
  * **BSS segment** → uninitialized global/static variables
  * **Heap** → dynamic memory (`malloc`)
  * **Stack** → function calls, local variables

Memory layout:
```
-----------------------
Stack
-----------------------
Heap
-----------------------
BSS Segment
-----------------------
Data Segment
-----------------------
Text Segment
-----------------------
```
## 3. Runtime Startup Code Executes
Before `main()` runs, special startup code from the C runtime library (CRT) executes.

In Linux using the GNU C Library (glibc), startup code initializes the environment.

Typical internal startup flow:
```
_start()
   ↓
__libc_start_main()
   ↓
main()
```
Where:
* `_start()` → actual entry point of program
* `__libc_start_main()` → initializes C runtime
* `main()` → user program begins

`_start()` is provided by runtime startup files (CRT files), not written by the programmer.
## 4. Initialization Performed Before `main()`
The runtime:
* Sets up command-line arguments (`argc`, `argv`)
* Sets environment variables (`envp`)
* Initializes standard I/O (`stdin`, `stdout`, `stderr`)
* Initializes shared libraries
* Executes constructors (for global initialization)

Example:
```c
int main(int argc, char *argv[])
{
    printf("Program started\n");
    return 0;
}
```
By the time `main()` starts:
```c
argc
argv
stdin
stdout
stderr
```
are already ready to use.
## 5. `main()` Executes
Control finally reaches:
```c
int main()
{
    printf("Hello Linux\n");
    return 0;
}
```
## 6. Program Termination
When `main()` ends:
```c
return 0;
```
control goes back to runtime library.

Flow:
```
main()
   ↓
exit()
   ↓
kernel cleanup
```
The runtime:
* Flushes buffered I/O
* Closes files
* Runs cleanup handlers
* Releases resources
## Internal Flow Diagram
```
Shell
 ↓
fork()
 ↓
exec()
 ↓
Kernel loads executable
 ↓
_start()
 ↓
__libc_start_main()
 ↓
main()
 ↓
exit()
 ↓
Kernel cleanup
```
### See Startup Symbol
You can check the actual entry point:
```
nm a.out | grep start
```
or
```
readelf -a a.out | grep Entry
```
Example output:
```
_start
```
This shows the real starting symbol of the program.

# Using argc/argv
In C, **command-line arguments** are passed to a program through `argc` and `argv`.

They are used to provide input to a program at execution time.

Syntax:
```c
int main(int argc, char *argv[])
{
    // code
}
```
or equivalently:
```c
int main(int argc, char **argv)
{
    // code
}
```
## Meaning of `argc` and `argv`
### `argc` (Argument Count)
`argc` stores the **number of command-line arguments** passed to the program.
* It is an integer
* Includes the program name itself

Example:
```
./a.out hello 10 linux
```
Arguments count:
```
./a.out
hello
10
linux
```
So:
```
argc = 4
```
## `argv` (Argument Vector)
`argv` is an array of pointers to strings.

It stores all command-line arguments.

Structure:
```
argv[0] → program name
argv[1] → first argument
argv[2] → second argument
...
argv[argc] → NULL
```
Example:

Command:
```
./a.out hello 10 linux
```
Memory representation:
```
argv[0] → "./a.out"
argv[1] → "hello"
argv[2] → "10"
argv[3] → "linux"
argv[4] → NULL
```
## Example Program
```c
#include <stdio.h>
int main(int argc, char *argv[])
{
    int i;
    printf("Number of arguments = %d\n", argc);
    for(i = 0; i < argc; i++)
    {
        printf("argv[%d] = %s\n", i, argv[i]);
    }
    return 0;
}
```
### Compilation
```
gcc args.c -o args
```
### Execution
```
./args apple mango 100 Linux
```
### Output
```
Number of arguments = 5
argv[0] = ./args
argv[1] = apple
argv[2] = mango
argv[3] = 100
argv[4] = Linux
```
## How `argc` and `argv` Work Internally
At program startup, the kernel places command-line arguments on the **stack** before calling `main()`.

Flow:
```
Shell
 ↓
exec()
 ↓
Kernel loads program
 ↓
argc, argv created
 ↓
main(argc, argv)
```
Example stack view:
```
argv[0] → "./args"
argv[1] → "apple"
argv[2] → "100"
argv[3] → NULL
```
Then:
```c
int main(int argc, char *argv[])
```
receives these values.
## Real-Time Uses of `argc/argv`
### 1. Passing filenames
```
./program input.txt
```
Program:
```c
FILE *fp = fopen(argv[1], "r");
```
## 2. Calculator program
```
./calc 10 20
```
Program:
```c
int a = atoi(argv[1]);
int b = atoi(argv[2]);
printf("%d\n", a + b);
```
## 3. Linux utilities
Example:
```
ls -l /home
```
Arguments:
```
argv[0] = "ls"
argv[1] = "-l"
argv[2] = "/home"
```
## Important Points
1. `argv[0]` always contains the program name.
2. `argc` includes the program name.
3. All arguments are strings.

Example:
```
./a.out 100
```
Here:
```c
argv[1]
```
contains:
```
"100"
```
not integer `100`.

Convert using:
```c
atoi()
```

# Handling Options with getopt()
`getopt()` is used to **parse command-line options (flags)** in Linux programs.

It helps process options like:
```
./prog -a -b file.txt
```
instead of manually checking `argv[]`.

It is commonly used in Linux commands such as:
```
ls -l
cp -r
grep -i file
tar -cvf
```
## Why `getopt()`?
Without `getopt()`, parsing options manually becomes difficult.

Example:
```
./program -a -b -f file.txt
```
You would have to check:
```c 
strcmp(argv[i], "-a")
strcmp(argv[i], "-b")
```
`getopt()` simplifies this process.
## Header File
```c 
#include <unistd.h>
```
## Syntax
```c
int getopt(int argc, char * const argv[],
           const char *optstring);
```
### Parameters
* `argc` → argument count
* `argv` → argument vector
* `optstring` → valid options
## Basic Example
Program:
```c 
#include <stdio.h>
#include <unistd.h>
int main(int argc, char *argv[])
{
    int opt;
    while((opt = getopt(argc, argv, "ab")) != -1)
    {
        switch(opt)
        {
            case 'a':
                printf("Option a selected\n");
                break;
            case 'b':
                printf("Option b selected\n");
                break;
            default:
                printf("Invalid option\n");
        }
    }
    return 0;
}
```
### Compile
```
gcc prog.c -o prog
```
### Run
```
./prog -a -b
```
### Output
```
Option a selected
Option b selected
```
## Understanding `optstring`
Example:
```c
"ab"
```
means:
* `-a` is valid
* `-b` is valid
## Option with Argument
Colon (`:`) means the option requires an argument.

Example:
```c 
"f:"
```
means:
```
./prog -f input.txt
```
`-f` requires a value.

Program:
```c 
#include <stdio.h>
#include <unistd.h>
int main(int argc, char *argv[])
{
    int opt;
    while((opt = getopt(argc, argv, "f:")) != -1)
    {
        switch(opt)
        {
            case 'f':
                printf("Filename = %s\n", optarg);
                break;
            default:
                printf("Usage error\n");
        }
    }
    return 0;
}
```
Run:
```
./prog -f data.txt
```
Output:
```
Filename = data.txt
```
Here:
```c 
optarg
```
contains:
```
"data.txt"
```
## Important Global Variables Used by `getopt()`
### 1. `optarg`
Stores option argument.

Example:
```
./prog -f test.txt
```
Then:
```c 
optarg = "test.txt"
```
## 2. `optind`
Stores next index in `argv`.

After option parsing:
```c 
argv[optind]
```
points to remaining arguments.

Example:
```
./prog -a file1 file2
```
After parsing:
```
optind → file1
```
Program:
```c 
for(int i = optind; i < argc; i++)
{
    printf("%s\n", argv[i]);
}
```
## 3. `opterr`
Controls automatic error messages.

Default:
```c 
opterr = 1
```
Disable:
```c 
opterr = 0;
```
## 4. `optopt`
Contains invalid option character.

Example:
```
./prog -x
```
Output:
```c 
Invalid option %c\n", optopt
```
## Example: Multiple Options
Program:
```c 
#include <stdio.h>
#include <unistd.h>
int main(int argc, char *argv[])
{
    int opt;
    while((opt = getopt(argc, argv, "af:")) != -1)
    {
        switch(opt)
        {
            case 'a':
                printf("a selected\n");
                break;
            case 'f':
                printf("File = %s\n", optarg);
                break;
            default:
                printf("Invalid option\n");
        }
    }
    return 0;
}
```
Run:
```
./prog -a -f input.txt
```
Output:
```
a selected
File = input.txt
```
## Internal Working
Example command:
```
./prog -a -f test.txt
```
Parsing flow:
```
getopt() → a
getopt() → f
optarg → test.txt
getopt() → -1
```
`-1` means parsing finished.
## Real-Time Example (like Linux command)
Command:
```
./copy -s source.txt -d dest.txt
```
Program:
```c 
while((opt = getopt(argc, argv, "s:d:")) != -1)
{
    switch(opt)
    {
        case 's':
            src = optarg;
            break;
        case 'd':
            dst = optarg;
            break;
    }
}
```

# Handling Options with getopt_long()
`getopt_long()` is an extension of `getopt()` that supports **long options**.

With `getopt()`, we use:
```
./prog -f input.txt
```
With `getopt_long()`:
```
./prog --file input.txt
```
It supports:
* Short options → `-f`
* Long options → `--file`

Many Linux commands use long options.

Example:
```
ls --help
gcc --version
tar --verbose
```
## Header Files
```c 
#include <stdio.h>
#include <unistd.h>
#include <getopt.h>
```
`getopt_long()` is declared in:
```c 
getopt.h
```
## Syntax
```c 
int getopt_long(int argc, char * const argv[],
                const char *shortopts,
                const struct option *longopts,
                int *longindex);
```
### Parameters
1. `argc` → argument count
2. `argv` → argument vector
3. `shortopts` → short option string (`"af:"`)
4. `longopts` → array of long options
5. `longindex` → index of matched long option
## Structure Used
`getopt_long()` uses:
```c 
struct option
{
    const char *name;
    int has_arg;
    int *flag;
    int val;
};
```
### Meaning of Fields
| Field     | Meaning                       |
| --------- | ----------------------------- |
| `name`    | Long option name              |
| `has_arg` | Whether option takes argument |
| `flag`    | Usually `NULL`                |
| `val`     | Returned value                |
## `has_arg` Values
* `no_argument` → no argument required
* `required_argument` → argument mandatory
* `optional_argument` → argument optional

Example:
```
--help
--file input.txt
```
## Example 1: Simple Long Option
Program:
```c 
#include <stdio.h>
#include <unistd.h>
#include <getopt.h>
int main(int argc, char *argv[])
{
    int opt;
    struct option long_options[] =
    {
        {"help", no_argument, 0, 'h'},
        {"version", no_argument, 0, 'v'},
        {0, 0, 0, 0}
    };
    while((opt = getopt_long(argc, argv,
                            "hv",
                            long_options,
                            NULL)) != -1)
    {
        switch(opt)
        {
            case 'h':
                printf("Help selected\n");
                break;
            case 'v':
                printf("Version selected\n");
                break;
            default:
                printf("Invalid option\n");
        }
    }
    return 0;
}
```
### Compile
```
gcc prog.c -o prog
```
### Run
```
./prog --help
```
Output:
```
Help selected
```
Run:
```
./prog --version
```
Output:
```
Version selected
```
## Example 2: Long Option with Argument
Program:
```c 
#include <stdio.h>
#include <unistd.h>
#include <getopt.h>
int main(int argc, char *argv[])
{
    int opt;
    struct option long_options[] =
    {
        {"file", required_argument, 0, 'f'},
        {0, 0, 0, 0}
    };
    while((opt = getopt_long(argc, argv,
                             "f:",
                             long_options,
                             NULL)) != -1)
    {
        switch(opt)
        {
            case 'f':
                printf("Filename = %s\n", optarg);
                break;
            default:
                printf("Invalid option\n");
        }
    }
    return 0;
}
```
### Run
```
./prog --file input.txt
```
Output:
```
Filename = input.txt
```
Here:
```c 
optarg
```
stores:
```
input.txt
```
## Supporting Both Short and Long Options
Program:
```c 
#include <stdio.h>
#include <unistd.h>
#include <getopt.h>
int main(int argc, char *argv[])
{
    int opt;
    struct option long_options[] =
    {
        {"help", no_argument, 0, 'h'},
        {"file", required_argument, 0, 'f'},
        {0,0,0,0}
    };
    while((opt = getopt_long(argc, argv,
                             "hf:",
                             long_options,
                             NULL)) != -1)
    {
        switch(opt)
        {
            case 'h':
                printf("Help selected\n");
                break;
            case 'f':
                printf("File = %s\n", optarg);
                break;
        }
    }
    return 0;
}
```
Run:
```
./prog -h
```
Output:
```
Help selected
```
Run:
```
./prog --file data.txt
```
Output:
```
File = data.txt
```
## `struct option` Breakdown
Example:
```c 
{"file", required_argument, 0, 'f'}
```
Meaning:
| Part                | Meaning           |
| ------------------- | ----------------- |
| `"file"`            | option name       |
| `required_argument` | argument required |
| `0`                 | flag unused       |
| `'f'`               | return character  |
Thus:
```
--file abc.txt
```
acts like:
```
-f abc.txt
```
## Internal Flow
Command:
```
./prog --file test.txt
```
Flow:
```
getopt_long()
      ↓
matches "file"
      ↓
returns 'f'
      ↓
optarg = test.txt
```
## Difference: `getopt()` vs `getopt_long()`
| Feature             | `getopt()` | `getopt_long()` |
| ------------------- | ---------: | --------------: |
| Short options       |        Yes |             Yes |
| Long options        |         No |             Yes |
| `--file` support    |         No |             Yes |
| Linux utility style |    Limited |          Better |
## Real-Time Example
Like a Linux utility:

Command:
```
./backup --source file.txt --destination /tmp
```
Program:
```c 
struct option long_options[] =
{
    {"source", required_argument, 0, 's'},
    {"destination", required_argument, 0, 'd'},
    {0,0,0,0}
};
```

# Environment
An **environment** is a collection of **name–value pairs (variables)** available to a process.

These variables provide information to programs such as:
* Username
* Home directory
* Shell
* Search path
* Terminal type

Example environment variables:
```
USER=niharika
HOME=/home/niharika
PATH=/usr/bin:/bin
SHELL=/bin/bash
PWD=/home/niharika
```
When a program starts, it inherits the environment from its parent process (usually the shell).
## What is an Environment Variable?
An environment variable is:
```
NAME=VALUE
```
Example:
```
PATH=/usr/bin:/bin
```
Here:
* `PATH` → variable name
* `/usr/bin:/bin` → value

Programs use these variables during execution.

Example:

The shell searches commands using `PATH`.
## How Environment Works
Flow:
```
Shell
 ↓
creates environment variables
 ↓
fork()
 ↓
exec()
 ↓
Child process inherits environment
```
Example:
```
export CITY=Hyderabad
./prog
```
The program can access:
```
CITY=Hyderabad
```
## Viewing Environment Variables
### Using shell command
```
env
```
or
```
printenv
```
Example output:
```
USER=niharika
HOME=/home/niharika
PATH=/usr/bin:/bin
LANG=en_US.UTF-8
```
## Accessing Environment in C
There are multiple ways.
### 1. Using `getenv()`
Header:
```c

```

```c
char *getenv(const char *name);
```
Example:
```c

int main()
{

    printf("PATH = %s\n", path);

```
### Compile
```
gcc env.c -o env
```
### Run
```

```
Output (example):
```

```
## Example: Read HOME Variable
```c
#include <stdio.h>

int main()
{
    printf("HOME = %s\n", getenv("HOME"));
    return 0;

```
Output:
```

```
## 2. Using Third Argument to `main()`
Linux can pass environment variables as a third argument:
```c
int main(int argc, char *argv[], char *envp[])
{
    int i;

```
Output:
```

PATH=/usr/bin:/bin
```
Here:
```
envp[i]
```
contains:
```
NAME=VALUE
```
strings.
## 3. Using Global Variable `environ`
Linux provides:
```c
extern char **environ;
```
Program:
```c

extern char **environ;

{
    int i = 0;

```

# Manipulating the Environment
**Manipulating the environment** means:
* Reading environment variables
* Adding new variables
* Modifying existing variables
* Deleting variables

Environment variables are stored as:
```
NAME=VALUE
```
Example:
```
USER=niharika
HOME=/home/niharika
PATH=/usr/bin:/bin
```
In C, the environment can be manipulated using functions from the standard library.
## Environment Manipulation Functions
| Function     | Purpose                   |
| ------------ | ------------------------- |
| `getenv()`   | Read environment variable |
| `setenv()`   | Create/modify variable    |
| `unsetenv()` | Delete variable           |
| `putenv()`   | Add/change variable       |
| `clearenv()` | Remove all variables      |
Header file:
```c
#include <stdlib.h>
```
## 1. Reading Environment Variable — `getenv()`
Syntax:
```c
char *getenv(const char *name);
```
Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    char *home;
    home = getenv("HOME");
    if(home != NULL)
        printf("HOME = %s\n", home);
    return 0;
}
```
Run:
```
./a.out
```
Output:
```
HOME = /home/user
```
### Important
If variable does not exist:
```c
getenv()
```
returns:
```
NULL
```
Example:
```c
printf("%s", getenv("XYZ"));
```
This may crash if `NULL` is printed directly.

Better:
```c
if(getenv("XYZ"))
```
## 2. Setting Environment Variable — `setenv()`
Used to create or modify variables.

Syntax:
```c
int setenv(const char *name,
           const char *value,
           int overwrite);
```
Parameters:
* `name` → variable name
* `value` → variable value
* `overwrite`
  * `1` → overwrite existing variable
  * `0` → don't overwrite

Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    setenv("CITY", "Hyderabad", 1);
    printf("%s\n", getenv("CITY"));
    return 0;
}
```
Output:
```
Hyderabad
```
## Overwrite Example
Program:
```c
setenv("CITY", "Delhi", 0);
```
If already exists:
```
CITY=Hyderabad
```
it will **not change**.

But:
```c
setenv("CITY", "Delhi", 1);
```
changes:
```
CITY=Delhi
```
## 3. Removing Variable — `unsetenv()`
Syntax:
```c
int unsetenv(const char *name);
```
Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    setenv("CITY", "Hyderabad", 1);
    printf("%s\n", getenv("CITY"));
    unsetenv("CITY");
    printf("%s\n", getenv("CITY"));
    return 0;
}
```
Output:
```
Hyderabad
(null)
```
After removal:
```c
getenv("CITY")
```
returns:
```
NULL
```
## 4. Using `putenv()`
Syntax:
```c
int putenv(char *string);
```
Format:
```
NAME=VALUE
```
Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    putenv("LANGUAGE=C");
    printf("%s\n", getenv("LANGUAGE"));
    return 0;
}
```
Output:
```
C
```
### Important Difference: `putenv()` vs `setenv()`
`putenv()` uses:
```
NAME=VALUE
```
as a single string.

Example:
```c
putenv("CITY=Hyderabad");
```
whereas:
```c
setenv("CITY", "Hyderabad", 1);
```
takes separate name/value.
## 5. Removing All Environment Variables — `clearenv()`
Syntax:
```c
int clearenv(void);
```
Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    clearenv();
    printf("%s\n", getenv("PATH"));
    return 0;
}
```
Output:
```
(null)
```
All variables are removed.
## Using `environ`
Environment internally exists as:
```c
extern char **environ;
```
Example:
```c
#include <stdio.h>
extern char **environ;
int main()
{
    int i = 0;
    while(environ[i] != NULL)
    {
        printf("%s\n", environ[i]);
        i++;
    }
    return 0;
}
```
This prints complete environment.

Memory representation:
```
environ[0] → USER=niharika
environ[1] → HOME=/home/niharika
environ[2] → PATH=/usr/bin
environ[n] → NULL
```

Suppose program depends on:
```
DEBUG=1
```

```c
#include <stdio.h>

int main()

```

```
export DEBUG=1

```

```
Debug mode enabled
```

# Program Exit
**Program Exit** is the process of terminating a program and returning control to the operating system.

A program exits when:
* `main()` finishes
* `return` is executed from `main()`
* `exit()` is called
* `_exit()` is called
* `abort()` is called (abnormal termination)
## Ways a Program Can Exit
```
1. return from main()
2. exit()
3. _exit()
4. abort()
```
## 1. Returning from `main()`
Simplest method.

Example:
```c
#include <stdio.h>
int main()
{
    printf("Program ended\n");
    return 0;
}
```
Flow:
```
main()
   ↓
return 0
   ↓
exit(0)
   ↓
kernel cleanup
```
Important:
```c
return 0;
```
inside `main()` is equivalent to:
```c
exit(0);
```
The runtime library internally calls `exit()`.
## 2. Using `exit()`
Header:
```c
#include <stdlib.h>
```
Syntax:
```c
void exit(int status);
```
Used to terminate program immediately.

Example:
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    printf("Before exit\n");
    exit(0);
    printf("After exit\n");
}
```
Output:
```
Before exit
```
`exit()` stops execution immediately.
## Exit Status
Example:
```c
exit(0);
```
Status:
* `0` → success
* non-zero → failure

Example:
```c
exit(1);
```
Shell can read exit status:
```
echo $?
```
Example:
```
./a.out
echo $?
```
Output:
```
0
```
## Standard Exit Macros
Available in:
```c
#include <stdlib.h>
```
Use:
```c
EXIT_SUCCESS
EXIT_FAILURE
```
Example:
```c
exit(EXIT_SUCCESS);
```
or
```c
exit(EXIT_FAILURE);
```
Equivalent to:
```
0
1
```
(but more portable)
## What `exit()` Does Internally
Before termination:
1. Flushes standard I/O buffers
2. Closes open files
3. Executes functions registered with `atexit()`
4. Releases resources
5. Returns status to OS

Flow:
```
exit()
   ↓
flush stdio buffers
   ↓
run atexit handlers
   ↓
close files
   ↓
kernel cleanup
```
## 3. Using `_exit()`
Header:
```c
#include <unistd.h>
```
Syntax:
```c
void _exit(int status);
```
`_exit()` terminates process **immediately**.

It **does not**:
* flush stdio buffers
* call `atexit()` handlers

Example:
```c
#include <stdio.h>
#include <unistd.h>
int main()
{
    printf("Hello");
    _exit(0);
}
```
Output:
```
```
(no output)

Why?

Because:
```c
printf()
```
buffer was not flushed.

Compare:
```c
exit(0);
```
Output:
```
Hello
```
## `exit()` vs `_exit()`
| Feature                 | `exit()` | `_exit()` |
| ----------------------- | -------: | --------: |
| Flush stdio             |      Yes |        No |
| Run `atexit()` handlers |      Yes |        No |
| Close files gracefully  |      Yes |        No |
| Immediate termination   |       No |       Yes |
Real-time use:

After `fork()` child failure:
```c
fork()
```
child commonly uses:
```c
_exit()
```
to avoid duplicate buffer flushing.
## 4. Using `abort()`
Header:
```c
#include <stdlib.h>
```
Syntax:
```c
void abort(void);
```
Terminates program abnormally.

Generates signal:
```
SIGABRT
```
Example:
```c
#include <stdlib.h>
int main()
{
    abort();
}
```
Output:
```
Aborted (core dumped)
```
Used when fatal errors occur.
## Registering Exit Functions — `atexit()`
Syntax:
```c
int atexit(void (*function)(void));
```
Runs function automatically when program exits normally.

Example:
```c
#include <stdio.h>
#include <stdlib.h>
void cleanup()
{
    printf("Cleanup function called\n");
}
int main()
{
    atexit(cleanup);
    printf("Main running\n");
    return 0;
}
```
Output:
```
Main running
Cleanup function called
```
Flow:
```
main()
   ↓
return
   ↓
exit()
   ↓
atexit() function runs
```
## Internal Flow of Program Exit
```
main()
   ↓
return / exit()
   ↓
flush buffers
   ↓
run atexit()
   ↓
close files
   ↓
kernel cleanup
```
## Real-Time Example
Suppose a log file must be closed properly:
```c
void cleanup()
{
    fclose(fp);
}
```
Register:
```c
atexit(cleanup);
```
Even if program exits early:
```c
exit(0);
```
cleanup still runs.

# Registering Exit Handlers
**Exit handlers** are functions that execute automatically when a program terminates **normally**.

They are used for:
* Releasing resources
* Closing files
* Saving logs
* Cleanup operations

In C, exit handlers are registered using:
```c
atexit()
```
Header file:
```c
#include <stdlib.h>
```
## Why Register Exit Handlers?
Suppose a program opens a file:
```c
FILE *fp = fopen("data.txt", "w");
```
Before the program exits, we may want to:
* Close file
* Save data
* Print cleanup message

Instead of calling cleanup manually everywhere, we register a cleanup function.
## `atexit()` Function
Syntax:
```c
int atexit(void (*function)(void));
```
Parameter:
```
function → function pointer
```
Return value:
* `0` → success
* non-zero → failure
## Basic Example
Program:
```c
#include <stdio.h>
#include <stdlib.h>
void cleanup()
{
    printf("Cleanup function executed\n");
}
int main()
{
    atexit(cleanup);
    printf("Main function running\n");
    return 0;
}
```
Output:
```
Main function running
Cleanup function executed
```
Flow:
```
main()
   ↓
return
   ↓
exit()
   ↓
cleanup()
   ↓
program terminates
```
## Important Point
When:
```c
return 0;
```
is executed inside `main()`:

Internally:
```c
exit(0);
```
is called.

Therefore:
```c
atexit()
```
functions execute.
## Multiple Exit Handlers
We can register multiple functions.

Example:
```c
#include <stdio.h>
#include <stdlib.h>
void fun1()
{
    printf("fun1 executed\n");
}
void fun2()
{
    printf("fun2 executed\n");
}
void fun3()
{
    printf("fun3 executed\n");
}
int main()
{
    atexit(fun1);
    atexit(fun2);
    atexit(fun3);
    printf("Main done\n");
    return 0;
}
```
Output:
```
Main done
fun3 executed
fun2 executed
fun1 executed
```
### Important
Execution order is:
```
LIFO (Last In First Out)
```
Meaning:
```
Last registered
        ↓
First executed
```
Stack behavior:
```
fun3
fun2
fun1
```
## Internal Working
Example:
```c
atexit(fun1);
atexit(fun2);
atexit(fun3);
```
Internal stack:
```
TOP → fun3
       fun2
       fun1
```
On exit:
```
fun3()
fun2()
fun1()
```
## Exit Handlers and `exit()`
Example:
```c
#include <stdio.h>
#include <stdlib.h>
void cleanup()
{
    printf("cleanup executed\n");
}
int main()
{
    atexit(cleanup);
    printf("Before exit\n");
    exit(0);
}
```
Output:
```
Before exit
cleanup executed
```
Because:
```c
exit()
```
runs registered handlers.
## `atexit()` and `_exit()`
Example:
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
void cleanup()
{
    printf("cleanup executed\n");
}
int main()
{
    atexit(cleanup);
    _exit(0);
}
```
Output:
```
```
No output.

Reason:
```c
_exit()
```
does **not** execute:
* exit handlers
* stdio flush
## `abort()` and Exit Handlers
Example:
```c
#include <stdlib.h>
void cleanup()
{
    printf("cleanup\n");
}
int main()
{
    atexit(cleanup);
    abort();
}
```
Output:
```
Aborted (core dumped)
```
Handler does not run.

Because:
```c
abort()
```
causes abnormal termination.
## Real-Time Example
Log cleanup:
```c
#include <stdio.h>
#include <stdlib.h>
FILE *fp;
void close_log()
{
    fclose(fp);
    printf("Log file closed\n");
}
int main()
{
    fp = fopen("log.txt", "w");
    atexit(close_log);
    fprintf(fp, "Application started\n");
    return 0;
}
```
Even if `main()` exits normally:
```
log file closed safely
```
## Limitations
Exit handlers must have format:
```c
void function(void)
```
Example:
- Correct
```c
void cleanup(void)
{
}
```
- Wrong
```c
void cleanup(int x)
{
}
```
## Internal Flow
```
main()
   ↓
atexit(cleanup)
   ↓
return / exit()
   ↓
registered handlers run
   ↓
files closed
   ↓
kernel cleanup
```

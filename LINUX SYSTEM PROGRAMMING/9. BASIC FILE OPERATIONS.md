# Stream vs: System Calls
In Linux programming, **Streams** and **System Calls** are two different ways to perform input/output (I/O). The main difference is the abstraction level.
| Feature       | Stream I/O                                                    | System Calls I/O                            |
| ------------- | ------------------------------------------------------------- | ------------------------------------------- |
| Level         | High-level I/O                                                | Low-level I/O                               |
| Library       | Uses C standard library (`stdio.h`)                           | Uses OS kernel interface (`unistd.h`)       |
| Examples      | `fopen()`, `fread()`, `fwrite()`, `fprintf()`, `fclose()`     | `open()`, `read()`, `write()`, `close()`    |
| Data Handling | Buffered I/O                                                  | Unbuffered I/O                              |
| Speed         | Usually faster for frequent small operations due to buffering | May be slower for small repeated operations |
| Ease of Use   | Easier and more convenient                                    | More control, slightly complex              |
| File Access   | Uses `FILE *` stream                                          | Uses file descriptor (`int fd`)             |
| Portability   | More portable                                                 | More OS-specific                            |
## 1. Stream I/O (High-Level I/O)
A **stream** is an abstraction provided by the C Standard Library to make file handling easier.
* Uses `FILE *`
* Defined in `stdio.h`
* Automatically manages buffering

Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "w");
    fprintf(fp, "Hello Linux Programming");
    fclose(fp);
    return 0;
}
```
**How it works internally:**

`fprintf()` → library buffer → eventually calls `write()` system call.

So stream functions internally depend on system calls.
### 2. System Calls (Low-Level I/O)
A **system call** directly interacts with the Linux kernel.
* Uses file descriptors (`fd`)
* Defined in `unistd.h` and `fcntl.h`
* No automatic buffering

Example:
```c
#include <unistd.h>
#include <fcntl.h>
int main() {
    int fd = open("test.txt", O_WRONLY | O_CREAT, 0644);
    write(fd, "Hello Linux Programming", 23);
    close(fd);
    return 0;
}
```
Here:
* `open()` returns a file descriptor
* `write()` directly requests the kernel to write data
* `close()` releases the file
### Relationship Between Streams and System Calls
A stream internally uses a file descriptor.

For example:
```c
FILE *fp = fopen("file.txt", "r");
```
Internally:
```c
fd = open("file.txt", O_RDONLY);
```
Then stream functions like `fread()` use buffering and eventually call `read()`.
### Diagram
```
Application
    ↓
Stream Functions (stdio)
(fopen, fprintf, fread)
    ↓
System Calls
(open, write, read)
    ↓
Linux Kernel
    ↓
Hardware / File System
```
### When to Use What?
**Use Streams when:**
* General file handling
* Text processing
* Simpler code
* Buffered I/O is beneficial

**Use System Calls when:**
* System programming
* Device drivers, pipes, sockets
* Need precise control over I/O
* Working with processes and IPC

# Opening/Closing Streams
In C, a **stream** is opened using `fopen()` and closed using `fclose()`. These functions are part of the C Standard I/O library (`stdio.h`).
## 1. Opening a Stream — `fopen()`
The `fopen()` function opens a file and associates it with a stream.
### Syntax
```c
FILE *fopen(const char *filename, const char *mode);
```
### Parameters
* `filename` → Name/path of the file
* `mode` → Specifies how the file is opened
### Return Value
* Returns a `FILE *` pointer on success
* Returns `NULL` on failure
### Example
```c
#include <stdio.h>
int main() {
    FILE *fp;
    fp = fopen("sample.txt", "w");
    if (fp == NULL) {
        printf("Error opening file\n");
        return 1;
    }
    printf("File opened successfully\n");
    fclose(fp);
    return 0;
}
```
## File Opening Modes
| Mode   | Meaning                                                     |
| ------ | ----------------------------------------------------------- |
| `"r"`  | Open for reading (file must exist)                          |
| `"w"`  | Open for writing (creates file, truncates existing content) |
| `"a"`  | Open for appending (creates file if not exists)             |
| `"r+"` | Read and write                                              |
| `"w+"` | Read and write, truncates file                              |
| `"a+"` | Read and append                                             |
| `"rb"` | Read binary file                                            |
| `"wb"` | Write binary file                                           |
| `"ab"` | Append binary file                                          |
### Examples
#### Read mode
```c
FILE *fp = fopen("data.txt", "r");
```
Opens file only for reading.
#### Write mode
```c
FILE *fp = fopen("data.txt", "w");
```
Creates file if absent; erases previous contents if present.
#### Append mode
```c
FILE *fp = fopen("data.txt", "a");
```
Adds new data at end of file.
## 2. Closing a Stream — `fclose()`
After file operations, the stream should be closed using `fclose()`.
### Syntax
```c
int fclose(FILE *stream);
```
### Return Value
* `0` → Success
* `EOF` → Error
### Example
```c
#include <stdio.h>
int main() {
    FILE *fp;
    fp = fopen("test.txt", "r");
    if (fp == NULL) {
        printf("Cannot open file\n");
        return 1;
    }
    printf("File opened\n");
    fclose(fp);
    printf("File closed\n");
    return 0;
}
```
## Why `fclose()` Is Important
When a stream is closed:
1. Buffered data is written to the file (flushes buffer)
2. Memory/resources are released
3. File descriptor is closed internally

Internally:
```
fclose(fp)
     ↓
fflush(fp)
     ↓
close(fd)
```
So `fclose()` eventually calls the low-level `close()` system call.
## Flow of Opening and Closing Streams
```
fopen()
   ↓
FILE * stream created
   ↓
Read/Write operations
(fprintf, fscanf, fread, fwrite)
   ↓
fclose()
   ↓
Buffer flushed
   ↓
File descriptor closed
```
### Important Notes
* Always check `fopen()` return value for `NULL`.
* Always close files using `fclose()`.
* Forgetting `fclose()` may cause data loss (buffer not written).
* A stream should not be used after closing.
### Example (Complete Program)
```c
#include <stdio.h>
int main() {
    FILE *fp;
    fp = fopen("message.txt", "w");
    if (fp == NULL) {
        printf("File open failed\n");
        return 1;
    }
    fprintf(fp, "Linux System Programming");
    fclose(fp);
    printf("Data written and file closed\n");
    return 0;
}
```
**Output:**
```
Data written and file closed
```
**Contents of `message.txt`:**
```
Linux System Programming
```

# Stream Input/Output Functions
**Stream I/O functions** are high-level input/output functions provided by the C Standard Library (`stdio.h`) to perform operations on streams (`FILE *`).

These functions allow reading from and writing to files, keyboard, screen, and other I/O devices.
## Categories of Stream I/O Functions
1. Character-at-a-time I/O
2. Line-at-a-time I/O
3. Formatted I/O
4. Block/Binary I/O
```
Stream (FILE *)
      ↓
Input / Output Functions
 ┌────────┬─────────┬──────────┬──────────┐
 Character   Line      Formatted   Block
```
## 1. Character-at-a-Time I/O
These functions read/write **one character at a time**.
| Function  | Purpose             |
| --------- | ------------------- |
| `fgetc()` | Read one character  |
| `fputc()` | Write one character |
| `getc()`  | Read one character  |
| `putc()`  | Write one character |
### `fgetc()`
Reads one character from a stream.
#### Syntax
```c
int fgetc(FILE *stream);
```
Returns:
* Character read → success
* `EOF` → end of file or error

Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    char ch;
    while ((ch = fgetc(fp)) != EOF) {
        printf("%c", ch);
    }
    fclose(fp);
    return 0;
}
```
## `fputc()`
Writes one character to a stream.
### Syntax
```c
int fputc(int ch, FILE *stream);
```
Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("file.txt", "w");
    fputc('A', fp);
    fputc('B', fp);
    fputc('C', fp);
    fclose(fp);
    return 0;
}
```
File contents:
```
ABC
```
## 2. Line-at-a-Time I/O
Used to read/write a complete line (string).
| Function  | Purpose    |
| --------- | ---------- |
| `fgets()` | Read line  |
| `fputs()` | Write line |
### `fgets()`
Reads a string from a stream.
#### Syntax
```c
char *fgets(char *str, int n, FILE *stream);
```
* Reads up to `n-1` characters
* Stops at newline (`\n`) or EOF

Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    char str[100];
    while (fgets(str, sizeof(str), fp)) {
        printf("%s", str);
    }
    fclose(fp);
    return 0;
}
```
## `fputs()`
Writes a string to a stream.
### Syntax
```c
int fputs(const char *str, FILE *stream);
```
Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "w");
    fputs("Linux\n", fp);
    fputs("System Programming\n", fp);
    fclose(fp);
    return 0;
}
```
## 3. Formatted I/O
Used for formatted reading and writing.
| Function    | Purpose          |
| ----------- | ---------------- |
| `fprintf()` | Formatted output |
| `fscanf()`  | Formatted input  |
### `fprintf()`
Works like `printf()` but writes to a file.
#### Syntax
```c
int fprintf(FILE *stream,
            const char *format, ...);
```
Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("student.txt", "w");
    int id = 101;
    float marks = 89.5;
    fprintf(fp,
            "ID=%d Marks=%.2f",
            id,
            marks);
    fclose(fp);
    return 0;
}
```
File contents:
```
ID=101 Marks=89.50
```
## `fscanf()`
Works like `scanf()` but reads from file.
### Syntax
```c
int fscanf(FILE *stream,
           const char *format, ...);
```
Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("student.txt", "r");
    int id;
    float marks;
    fscanf(fp,
           "ID=%d Marks=%f",
           &id,
           &marks);
    printf("ID=%d\n", id);
    printf("Marks=%f\n", marks);
    fclose(fp);
    return 0;
}
```
## 4. Block/Binary I/O
Used for binary files and structures.
| Function   | Purpose             |
| ---------- | ------------------- |
| `fread()`  | Read block of data  |
| `fwrite()` | Write block of data |
### `fwrite()`
Writes binary data.
#### Syntax
```c
size_t fwrite(const void *ptr,
              size_t size,
              size_t count,
              FILE *stream);
```
Example:
```c
#include <stdio.h>
struct Student {
    int id;
    float marks;
};
int main() {
    FILE *fp = fopen("data.dat", "wb");
    struct Student s = {1, 85.5};
    fwrite(&s,
           sizeof(s),
           1,
           fp);
    fclose(fp);
    return 0;
}
```
## `fread()`
Reads binary data.
### Syntax
```c
size_t fread(void *ptr,
             size_t size,
             size_t count,
             FILE *stream);
```
Example:
```c
#include <stdio.h>
struct Student {
    int id;
    float marks;
};
int main() {
    FILE *fp = fopen("data.dat", "rb");
    struct Student s;
    fread(&s,
          sizeof(s),
          1,
          fp);
    printf("ID=%d\n", s.id);
    printf("Marks=%f\n",
           s.marks);
    fclose(fp);
    return 0;
}
```
## Summary Table
| Category         | Input Function | Output Function |
| ---------------- | -------------- | --------------- |
| Character I/O    | `fgetc()`      | `fputc()`       |
| Line I/O         | `fgets()`      | `fputs()`       |
| Formatted I/O    | `fscanf()`     | `fprintf()`     |
| Block/Binary I/O | `fread()`      | `fwrite()`      |
### Internal Working
```
Application
    ↓
Stream I/O Functions
(fprintf, fgets, fwrite)
    ↓
Buffer (stdio)
    ↓
System Calls
(read, write)
    ↓
Kernel
```
**Key idea:** Stream I/O functions are buffered, easier to use, and internally rely on low-level system calls like `read()` and `write()`.

# Stream Status / Errors
When working with streams (`FILE *`), errors may occur during file operations such as reading, writing, or opening files.

The C Standard I/O library (`stdio.h`) provides functions to:
* Detect **End of File (EOF)**
* Detect **stream errors**
* Clear stream status flags
## Stream Status Indicators
Each stream maintains internal status flags:
1. **EOF flag** → End of file reached
2. **Error flag** → I/O error occurred
```
Stream (FILE *)
      ↓
 ┌──────────────┐
 │ EOF Flag     │
 │ Error Flag   │
 └──────────────┘
```
## 1. `feof()` — End-of-File Check
Checks whether the **EOF flag** is set for a stream.
### Syntax
```c
int feof(FILE *stream);
```
### Return Value
* Non-zero → EOF reached
* `0` → EOF not reached
### Example
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    char ch;
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch);
    }
    if (feof(fp)) {
        printf("\nEnd of file reached\n");
    }
    fclose(fp);
    return 0;
}
```
### Important Point
`feof()` becomes true **only after attempting to read past EOF**.

Wrong approach:
```c
while (!feof(fp)) {
    ch = fgetc(fp);
    printf("%c", ch);
}
```
Why wrong?

Because EOF is detected **after** a failed read, this can process invalid data.

Correct approach:
```c
while ((ch = fgetc(fp)) != EOF) {
    putchar(ch);
}
```
## 2. `ferror()` — Error Check
Checks whether an error occurred on the stream.
### Syntax
```c
int ferror(FILE *stream);
```
### Return Value
* Non-zero → Error occurred
* `0` → No error
### Example
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("data.txt", "r");
    if (fp == NULL) {
        printf("Cannot open file\n");
        return 1;
    }
    char ch;
    while ((ch = fgetc(fp)) != EOF) {
        putchar(ch);
    }
    if (ferror(fp)) {
        printf("\nError reading file\n");
    }
    fclose(fp);
    return 0;
}
```
## 3. `clearerr()` — Clear Stream Status Flags
Clears both:
* EOF flag
* Error flag
### Syntax
```c
void clearerr(FILE *stream);
```
### Example
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    char ch;
    while ((ch = fgetc(fp)) != EOF);
    if (feof(fp))
        printf("EOF reached\n");
    clearerr(fp);
    if (!feof(fp))
        printf("EOF cleared\n");
    fclose(fp);
    return 0;
}
```
## 4. `perror()` — Print Error Message
Prints a descriptive error message using `errno`.
### Syntax
```c
void perror(const char *msg);
```
Example:
```c
#include <stdio.h>
int main() {
    FILE *fp = fopen("abc.txt", "r");
    if (fp == NULL) {
        perror("File open failed");
        return 1;
    }
    fclose(fp);
    return 0;
}
```
Possible output:
```
File open failed: No such file or directory
```
## 5. `strerror()` — Get Error String
Returns error message as a string.
### Syntax
```c
char *strerror(int errnum);
```
Example:
```c
#include <stdio.h>
#include <errno.h>
#include <string.h>
int main() {
    FILE *fp = fopen("abc.txt", "r");
    if (fp == NULL) {
        printf("%s\n",
               strerror(errno));
    }
    return 0;
}
```
Output:
```
No such file or directory
```
## Stream Status Flow
```
File Read/Write
      ↓
Problem?
 ┌────────────┬─────────────┐
 EOF reached  Error occurred
      ↓              ↓
   feof()         ferror()
      ↓              ↓
    clearerr() clears both
```
## Summary Table
| Function     | Purpose                    |
| ------------ | -------------------------- |
| `feof()`     | Check end-of-file          |
| `ferror()`   | Check stream error         |
| `clearerr()` | Clear EOF/error flags      |
| `perror()`   | Print system error message |
| `strerror()` | Get error message string   |

# Stream File Positioning
**Stream file positioning** means moving or determining the current position of a file pointer inside a stream (`FILE *`).

It allows:
* Reading/writing from a specific location
* Skipping data
* Random file access
* Finding current position

The C Standard I/O library (`stdio.h`) provides functions for stream positioning.
## Stream Positioning Functions
| Function    | Purpose                   |
| ----------- | ------------------------- |
| `ftell()`   | Get current file position |
| `fseek()`   | Move file position        |
| `rewind()`  | Move to beginning of file |
| `fgetpos()` | Get current position      |
| `fsetpos()` | Set position              |
## File Position Concept
When a file is opened, the stream maintains a **file position indicator**.

Example file:
```
HELLO
01234
```
Suppose current pointer is at character `L`:
```
HELLO
   ^
Current Position = 2
```
## 1. `ftell()` — Get Current Position
Returns the current file position.
### Syntax
```c 
long ftell(FILE *stream);
```
### Return Value
* Current position (in bytes)
* `-1L` on error
### Example
```c 
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    printf("Position = %ld\n",
           ftell(fp));
    fgetc(fp);
    fgetc(fp);
    printf("Position = %ld\n",
           ftell(fp));
    fclose(fp);
    return 0;
}
```
Example output:
```
Position = 0
Position = 2
```
## 2. `fseek()` — Move File Position
Moves the file position indicator.
### Syntax
```c 
int fseek(FILE *stream,
          long offset,
          int origin);
```
### Parameters
* `offset` → Number of bytes to move
* `origin` → Starting point
### Origins
| Constant   | Meaning           |
| ---------- | ----------------- |
| `SEEK_SET` | Beginning of file |
| `SEEK_CUR` | Current position  |
| `SEEK_END` | End of file       |
### Return Value
* `0` → Success
* Non-zero → Error
### Example 1: Move to Specific Position
```c 
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    fseek(fp, 3, SEEK_SET);
    printf("%c\n", fgetc(fp));
    fclose(fp);
    return 0;
}
```
If file contains:
```
HELLO
```
Output:
```
L
```
(Reads character at index 3)
## Example 2: Move Relative to Current Position
```c 
fseek(fp, 2, SEEK_CUR);
```
Moves 2 bytes forward from current location.
## Example 3: Move Relative to End
```c
fseek(fp, -1, SEEK_END);
```
Moves one byte before file end.
## 3. `rewind()` — Go to Beginning
Moves file position to the start of the file.
### Syntax
```c 
void rewind(FILE *stream);
```
Equivalent to:
```c 
fseek(stream, 0, SEEK_SET);
```
Also clears error and EOF flags.
### Example
```c 
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    fgetc(fp);
    fgetc(fp);
    printf("Position = %ld\n",
           ftell(fp));
    rewind(fp);
    printf("Position = %ld\n",
           ftell(fp));
    fclose(fp);
    return 0;
}
```
Output:
```
Position = 2
Position = 0
```
## 4. `fgetpos()` — Get Position
Stores current stream position.
### Syntax
```c 
int fgetpos(FILE *stream,
            fpos_t *pos);
```
### Example
```c 
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    fpos_t pos;
    fgetc(fp);
    fgetc(fp);
    fgetpos(fp, &pos);
    fclose(fp);
    return 0;
}
```
## 5. `fsetpos()` — Restore Position
Sets stream position using saved value.
### Syntax
```c 
int fsetpos(FILE *stream,
            const fpos_t *pos);
```
### Example
```c 
#include <stdio.h>
int main() {
    FILE *fp = fopen("test.txt", "r");
    fpos_t pos;
    fgetc(fp);
    fgetpos(fp, &pos);
    fgetc(fp);
    fgetc(fp);
    fsetpos(fp, &pos);
    printf("%c\n", fgetc(fp));
    fclose(fp);
    return 0;
}
```
## Positioning Diagram
```
File: HELLO

Index: 0 1 2 3 4
       H E L L O
         ^
      Current Position

fseek(fp, 2, SEEK_SET)
            ↓
       H E L L O
           ^
```
## Summary Table
| Function    | Purpose              |
| ----------- | -------------------- |
| `ftell()`   | Get current position |
| `fseek()`   | Move file position   |
| `rewind()`  | Move to start        |
| `fgetpos()` | Save position        |
| `fsetpos()` | Restore position     |

# Stream Buffering
**Stream buffering** is a mechanism where data is temporarily stored in memory (**buffer**) before being read from or written to a file/device.

Instead of immediately calling the kernel for every I/O operation, the C Standard Library (`stdio.h`) stores data in a buffer and performs I/O in larger chunks.

This improves performance because **system calls are expensive**.
## Without Buffering vs With Buffering
```
Without Buffering:
Program → write() → Kernel → File
Program → write() → Kernel → File
Program → write() → Kernel → File

(Too many system calls)
```
```
With Buffering:
Program → Buffer (memory)
                ↓
          write() once
                ↓
              File
```
## Why Buffering Is Needed
Suppose you write characters one by one:
```c
fputc('A', fp);
fputc('B', fp);
fputc('C', fp);
```
Without buffering:
```
write('A')
write('B')
write('C')
```
3 system calls.

With buffering:
```
Buffer = ABC
write("ABC")
```
Only **one system call** may occur.
### Benefit
* Faster I/O
* Fewer kernel calls
* Better performance
## Types of Stream Buffering
There are **three buffering modes**.
| Mode           | Meaning                          |
| -------------- | -------------------------------- |
| Full buffering | Data stored until buffer full    |
| Line buffering | Flush when newline (`\n`) occurs |
| No buffering   | Immediate I/O                    |
### 1. Fully Buffered (`_IOFBF`)
Data is written/read only when:
* Buffer becomes full
* `fflush()` is called
* File is closed (`fclose()`)

Example:
```
Buffer: ABCDEFGHIJ
(Buffer full)
      ↓
Write to file
```
Common for **regular files**.
## 2. Line Buffered (`_IOLBF`)
Buffer flushes when:
* Newline (`\n`) appears
* Buffer full
* `fflush()` or `fclose()`

Example:
```c
printf("Hello\n");
```
After newline:
```
Buffer flushed immediately
```
Common for terminal output (`stdout`).
## 3. Unbuffered (`_IONBF`)
Every I/O operation immediately accesses kernel.

Example:
```
fputc('A')
    ↓
write()
```
No temporary buffer.

Common for:
* `stderr`
* Critical logging
## Default Buffering Behavior
| Stream       | Default Buffering        |
| ------------ | ------------------------ |
| `stdin`      | Line buffered            |
| `stdout`     | Line buffered (terminal) |
| `stderr`     | Unbuffered               |
| Regular file | Fully buffered           |
## `fflush()` — Flush Buffer
Forces buffered output to be written immediately.
### Syntax
```c
int fflush(FILE *stream);
```
Example:
```c
#include <stdio.h>
int main() {
    printf("Hello");
    fflush(stdout);
    return 0;
}
```
Without `fflush()`, output may wait in buffer.
## `setbuf()` — Set Buffering
Controls stream buffering.
### Syntax
```c
void setbuf(FILE *stream,
            char *buffer);
```
### Example
Disable buffering:
```c
#include <stdio.h>
int main() {
```

* Sorting large files

Suppose a program processes huge data.

```
Program
   ↓
```

# Opening / Closing File Descriptors
In Linux system programming, files are accessed using **file descriptors (FDs)** through **system calls**.

A **file descriptor** is a small integer used by the kernel to identify an open file, device, pipe, or socket.

File descriptor I/O is **low-level I/O** and is different from stream I/O (`FILE *`).
## What Is a File Descriptor?
When a file is opened, the kernel returns an integer:
```
File Descriptor (FD)
        ↓
     Integer Handle
```
Example:
```
0 → Standard Input  (stdin)
1 → Standard Output (stdout)
2 → Standard Error  (stderr)
3 → First opened file
4 → Second opened file
```
## Opening a File — `open()`
The `open()` system call opens a file and returns a file descriptor.
### Syntax
```c
#include <fcntl.h>
int open(const char *pathname,
         int flags,
         mode_t mode);
```
> `mode` is required only when creating a file (`O_CREAT`).
### Return Value
* Non-negative integer → success (FD)
* `-1` → error
## File Open Flags
### Access Modes
| Flag       | Meaning      |
| ---------- | ------------ |
| `O_RDONLY` | Read only    |
| `O_WRONLY` | Write only   |
| `O_RDWR`   | Read + write |
### Common Additional Flags
| Flag       | Meaning                                  |
| ---------- | ---------------------------------------- |
| `O_CREAT`  | Create file if not exists                |
| `O_APPEND` | Append at file end                       |
| `O_TRUNC`  | Clear file contents                      |
| `O_EXCL`   | Used with `O_CREAT`; fail if file exists |
## Example 1: Open Existing File
```c 
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    fd = open("test.txt",
              O_RDONLY);
    if (fd == -1) {
        perror("open");
        return 1;
    }
    printf("FD = %d\n", fd);
    close(fd);
    return 0;
}
```
## Example 2: Create File
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    fd = open("file.txt",
              O_WRONLY | O_CREAT,
              0644);
    if (fd == -1) {
        perror("open");
        return 1;
    }
    close(fd);
    return 0;
}
```
### File Permission `0644`
```
Owner : rw-
Group : r--
Others: r--
```
(Octal permissions)
## Closing a File Descriptor — `close()`
After finishing file operations, the FD must be closed.
### Syntax
```c 
#include <unistd.h>
int close(int fd);
```
### Return Value
* `0` → success
* `-1` → error
## Example
```c 
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    fd = open("test.txt",
              O_RDONLY);
    if (fd == -1) {
        perror("open");
        return 1;
    }
    printf("File opened\n");
    close(fd);
    printf("File closed\n");
    return 0;
}
```
Output:
```
File opened
File closed
```
## Internal Working
```
open("test.txt")
        ↓
Kernel opens file
        ↓
Returns file descriptor
(e.g., fd = 3)
        ↓
Program uses fd
        ↓
close(fd)
        ↓
Kernel releases resource
```

| `0` | Standard Input  |
| `1` | Standard Output |


Writes to standard output.

| Stream                 | File Descriptor      |
| ---------------------- | -------------------- |

# File Descriptor I/O
**File Descriptor I/O** refers to low-level input/output operations performed using **file descriptors (FDs)** and **system calls**.

Unlike stream I/O (`FILE *`), file descriptor I/O directly interacts with the Linux kernel and is generally **unbuffered**.

Main system calls used:
| Function  | Purpose    |
| --------- | ---------- |
| `read()`  | Read data  |
| `write()` | Write data |
| `open()`  | Open file  |
| `close()` | Close file |
## What Is File Descriptor I/O?
When a file is opened using `open()`, the kernel returns an integer called a **file descriptor**.

Example:
```
fd = open("test.txt", O_RDONLY);
```
Suppose:
```
fd = 3
```
Then:
```
read(fd, ...)
write(fd, ...)
close(fd)
```
use that descriptor.
## 1. `read()` — Read Data
Reads data from a file descriptor.
### Syntax
```c 
#include <unistd.h>
ssize_t read(int fd,
             void *buffer,
             size_t count);
```
### Parameters
* `fd` → file descriptor
* `buffer` → memory to store data
* `count` → number of bytes to read
### Return Value
* Positive number → bytes read
* `0` → End of file (EOF)
* `-1` → Error
### Example
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    char buffer[100];
    fd = open("test.txt",
              O_RDONLY);
    if (fd == -1) {
        perror("open");
        return 1;
    }
    int n = read(fd,
                 buffer,
                 sizeof(buffer));
    buffer[n] = '\0';
    printf("%s", buffer);
    close(fd);
    return 0;
}
```
## 2. `write()` — Write Data
Writes data to a file descriptor.
### Syntax
```c 
#include <unistd.h>
ssize_t write(int fd,
              const void *buffer,
              size_t count);
```
### Parameters
* `fd` → file descriptor
* `buffer` → data to write
* `count` → bytes to write
### Return Value
* Positive number → bytes written
* `-1` → error
### Example
```c 
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    fd = open("test.txt",
              O_WRONLY | O_CREAT,
              0644);
    write(fd,
          "Linux Programming\n",
          18);
    close(fd);
    return 0;
}
```
## Example: Copy File Using `read()` and `write()`
```c 
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int src, dest;
    char buffer[100];
    int bytes;
    src = open("source.txt",
               O_RDONLY);
    dest = open("dest.txt",
                O_WRONLY | O_CREAT | O_TRUNC,
                0644);
    while ((bytes = read(src,
                          buffer,
                          sizeof(buffer))) > 0) {
        write(dest,
              buffer,
              bytes);
    }
    close(src);
    close(dest);
    return 0;
}
```
## `read()` Internal Flow
```
Program
   ↓
read(fd, buffer, n)
   ↓
Kernel
   ↓
File System
   ↓
Data copied to buffer
```
## `write()` Internal Flow
```
Program
   ↓
write(fd, data, n)
   ↓
Kernel
   ↓
File System
   ↓
Data stored
```
## Standard File Descriptors
| FD  | Meaning         |
| --- | --------------- |
| `0` | Standard Input  |
| `1` | Standard Output |
| `2` | Standard Error  |
Example:
```c 
#include <unistd.h>
int main() {
    write(1,
          "Hello World\n",
          12);
    return 0;
}
```
Output:
```
Hello World
```

| Stream I/O             | File Descriptor I/O  |
| ---------------------- | -------------------- |
| `fread()` / `fwrite()` | `read()` / `write()` |
| Buffered               | Unbuffered           |
| Uses `FILE *`          | Uses `int fd`        |
| High-level             | Low-level            |
| Easier                 | More control         |

# Repositioning File Descriptors
**Repositioning a file descriptor** means changing the current file offset (position) associated with an open file descriptor.

This is used for:
* Random access to files
* Skipping bytes
* Reading/writing from specific locations
* Finding current position

In low-level Linux I/O, this is done using the `lseek()` system call.
## File Offset Concept
When a file is opened, the kernel maintains a **file offset** (current position).

Example file:
```
HELLO
01234
```
Suppose current position is:
```
HELLO
  ^
Offset = 1
```
If we read one byte:
```
HELLO
   ^
Offset = 2
```
The file offset moves automatically.
## `lseek()` — Reposition File Offset
Moves the file descriptor position.
### Syntax
```c
#include <unistd.h>
off_t lseek(int fd,
            off_t offset,
            int whence);
```
### Parameters
* `fd` → file descriptor
* `offset` → number of bytes to move
* `whence` → starting reference point
### `whence` Values
| Constant   | Meaning           |
| ---------- | ----------------- |
| `SEEK_SET` | Beginning of file |
| `SEEK_CUR` | Current position  |
| `SEEK_END` | End of file       |
### Return Value
* New file position → success
* `-1` → error
## Example 1: Move to Specific Position
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    char ch;
    fd = open("test.txt",
              O_RDONLY);
    lseek(fd, 2, SEEK_SET);
    read(fd, &ch, 1);
    printf("%c\n", ch);
    close(fd);
    return 0;
}
```
If file contains:
```
HELLO
```
Output:
```
L
```
Explanation:
```
Index: 0 1 2 3 4
       H E L L O
           ^
```
## Example 2: Move Relative to Current Position
```c
lseek(fd, 3, SEEK_CUR);
```
Moves 3 bytes forward from current position.

Example:
```
Current → position 2
Move +3
New position = 5
```
## Example 3: Move Relative to End
```c
lseek(fd, -1, SEEK_END);
```
Moves to one byte before file end.

Example:
```
HELLO
    ^
```
Reading now gives:
```
O
```
## Example 4: Find File Size
A common trick:
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    off_t size;
    fd = open("test.txt",
              O_RDONLY);
    size = lseek(fd,
                 0,
                 SEEK_END);
    printf("File Size = %ld bytes\n",
           size);
    close(fd);
    return 0;
}
```
Explanation:
```
Move to end
      ↓
Returned offset = file size
```
## Random Access Example
Read specific byte positions.
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    char ch;
    fd = open("test.txt",
              O_RDONLY);
    lseek(fd, 4, SEEK_SET);
    read(fd, &ch, 1);
    printf("%c\n", ch);
    close(fd);
    return 0;
}
```
## Internal Working
```
open()
   ↓
Kernel maintains file offset
   ↓
read()/write()
   ↓
Offset changes automatically
   ↓
lseek()
   ↓
Offset repositioned
```
## `fseek()` vs `lseek()`
| Stream I/O | File Descriptor I/O      |
| ---------- | ------------------------ |
| `fseek()`  | `lseek()`                |
| `ftell()`  | `lseek(fd, 0, SEEK_CUR)` |



# Stream / File Descriptor Conversions
In Linux programming, sometimes we need to convert between:
* **Stream (`FILE *`)** → File descriptor (`int fd`)
* **File descriptor (`int fd`)** → Stream (`FILE *`)

This is useful when mixing:
* High-level stream I/O (`fopen()`, `fprintf()`, `fscanf()`)
* Low-level system call I/O (`open()`, `read()`, `write()`)

The standard library provides:
| Function   | Purpose                          |
| ---------- | -------------------------------- |
| `fileno()` | Convert stream → file descriptor |
| `fdopen()` | Convert file descriptor → stream |
## Why Conversion Is Needed
Sometimes a program opens a file using stream I/O but later needs a system call.

Example:
```
FILE * stream
      ↓
Need write(), read(), lseek()
      ↓
Convert to file descriptor
```
Or:
```
open()
   ↓
fd returned
   ↓
Need fprintf(), fgets()
   ↓
Convert to FILE *
```
## 1. `fileno()` — Stream to File Descriptor
Returns the file descriptor associated with a stream.
### Syntax
```c
#include <stdio.h>
int fileno(FILE *stream);
```
### Return Value
* File descriptor → success
* `-1` → error
### Example
```c 
#include <stdio.h>
#include <unistd.h>
int main() {
    FILE *fp;
    int fd;
    fp = fopen("test.txt", "w");
    if (fp == NULL) {
        perror("fopen");
        return 1;
    }
    fd = fileno(fp);
    printf("FD = %d\n", fd);
    write(fd,
          "Hello\n",
          6);
    fclose(fp);
    return 0;
}
```
### Internal Working
```
fopen()
   ↓
FILE * stream created
   ↓
Underlying file descriptor exists
   ↓
fileno(fp)
   ↓
fd returned
```
Example:
```
FILE *fp
    ↓
fd = 3
```
## 2. `fdopen()` — File Descriptor to Stream
Associates an existing file descriptor with a stream.
### Syntax
```c 
#include <stdio.h>
FILE *fdopen(int fd,
             const char *mode);
```
### Parameters
* `fd` → existing file descriptor
* `mode` → stream mode (`"r"`, `"w"`, `"a"`, `"r+"`, etc.)
### Return Value
* `FILE *` → success
* `NULL` → failure
### Example
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    FILE *fp;
    fd = open("test.txt",
              O_WRONLY | O_CREAT,
              0644);
    if (fd == -1) {
        perror("open");
        return 1;
    }
    fp = fdopen(fd, "w");
    if (fp == NULL) {
        perror("fdopen");
        close(fd);
        return 1;
    }
    fprintf(fp,
            "Linux Programming\n");
    fclose(fp);
    return 0;
}
```
### Internal Working
```
open()
   ↓
fd obtained
   ↓
fdopen(fd, "w")
   ↓
FILE * stream created
   ↓
Use fprintf(), fgets(), etc.
```
## Important Rule About `fdopen()`
After converting:
```c 
FILE *fp = fdopen(fd, "w");
```
Do **not** call:
```c
close(fd);
```
and
```c
fclose(fp);
```
both.

Why?

Because `fclose(fp)` automatically closes the underlying file descriptor.

Wrong:
```c 
close(fd);
fclose(fp);
```
May cause:
```
Double close error
```
Correct:
```c 
fclose(fp);
```
## Example: Mixing Stream + System Call I/O
```c 
#include <stdio.h>
#include <unistd.h>
int main() {
    FILE *fp;
    int fd;
    fp = fopen("demo.txt", "w");
    fd = fileno(fp);
    write(fd,
          "Hello ",
          6);
    fprintf(fp,
            "World");
    fclose(fp);
    return 0;
}
```
File contents:
```
Hello World
```
## Conversion Flow
```
Stream → FD
FILE *
   ↓
fileno()
   ↓
int fd
```
```
FD → Stream
int fd
   ↓
fdopen()
   ↓
FILE *
```
## Summary Table
| Conversion      | Function   |
| --------------- | ---------- |
| `FILE *` → `fd` | `fileno()` |
| `fd` → `FILE *` | `fdopen()` |
## Stream vs File Descriptor Relationship
```
FILE * (Stream)
       ↓
Uses internal file descriptor
       ↓
Kernel file table
```

# cat using ANSI I/O
The `cat` command displays file contents on the terminal.

Using **ANSI I/O (stream I/O)**, we implement it with stream functions such as:
* `fopen()`
* `fgetc()` / `fgets()`
* `fputc()` / `fputs()`
* `fclose()`
## Algorithm
```
Start
  ↓
Open file using fopen()
  ↓
Read character from file
(fgetc)
  ↓
Write character to stdout
(fputc)
  ↓
EOF reached?
 ├── No → continue
 └── Yes
  ↓
Close file
  ↓
End
```
## Program: `cat` Using ANSI I/O
```c 
#include <stdio.h>
#include <stdlib.h>
int main(int argc, char *argv[])
{
    FILE *fp;
    int ch;
    /* Check command-line argument */
    if (argc != 2)
    {
        printf("Usage: %s <filename>\n",
               argv[0]);
        return 1;
    }
    /* Open file */
    fp = fopen(argv[1], "r");
    if (fp == NULL)
    {
        perror("fopen");
        return 1;
    }
    /* Read and display file */
    while ((ch = fgetc(fp)) != EOF)
    {
        fputc(ch, stdout);
    }
    /* Check for read error */
    if (ferror(fp))
    {
        perror("Read error");
    }
    /* Close file */
    fclose(fp);
    return 0;
}
```
## Explanation
### 1. Open File
```c 
fp = fopen(argv[1], "r");
```
Opens file in **read mode**.

Example:
```
./cat file.txt
```
If `file.txt` exists:
```
FILE *fp created
```
## 2. Read Character-by-Character
```c 
while ((ch = fgetc(fp)) != EOF)
```
Reads one character at a time until:
```
EOF
```
is reached.
## 3. Print to Screen
```c
fputc(ch, stdout);
```
Writes character to standard output.

Equivalent:
```c 
printf("%c", ch);
```
but `fputc()` is more appropriate for character I/O.
## 4. Error Checking
```c 
if (ferror(fp))
```
Checks stream errors after reading.
## 5. Close File
```c 
fclose(fp);
```
Flushes buffers and closes stream.
## Example
Suppose `sample.txt` contains:
```
Linux
System Programming
```
Run:
```
./cat sample.txt
```
Output:
```
Linux
System Programming
```
## Internal Working
```
fopen()
   ↓
FILE * stream
   ↓
fgetc()
(buffered read)
   ↓
fputc(stdout)
(buffered output)
   ↓
fclose()
```
Internally:
```
fgetc()
   ↓
read() system call

fputc()
   ↓
write() system call
```
## Version Using `fgets()` + `fputs()`
More efficient (line-by-line):
```c 
#include <stdio.h>
int main(int argc, char *argv[])
{
    FILE *fp;
    char buffer[1024];
    if (argc != 2)
    {
        printf("Usage: %s <file>\n",
               argv[0]);
        return 1;
    }
    fp = fopen(argv[1], "r");
    if (fp == NULL)
    {
        perror("fopen");
        return 1;
    }
    while (fgets(buffer,
                 sizeof(buffer),
                 fp) != NULL)
    {
        fputs(buffer, stdout);
    }
    fclose(fp);
    return 0;
}
```
### Why Better?
Instead of:
```
1 character
1 function call
```
it processes:
```
Entire line
1 function call
```
Improves efficiency.
## Summary
| Method        | Functions Used        |
| ------------- | --------------------- |
| Character I/O | `fgetc()` + `fputc()` |
| Line I/O      | `fgets()` + `fputs()` |

# cat using POSIX I/O
The `cat` command displays file contents on the terminal.

Using **POSIX I/O (low-level I/O)**, we implement it using:
* `open()`
* `read()`
* `write()`
* `close()`

These functions work directly with **file descriptors** and are defined in `unistd.h` and `fcntl.h`.
## Algorithm
```
Start
  ↓
Open file using open()
  ↓
Read data using read()
  ↓
Write data to stdout using write()
  ↓
EOF reached?
 ├── No → continue
 └── Yes
  ↓
Close file
  ↓
End
```
## Program: `cat` Using POSIX I/O
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
#include <stdlib.h>
#define BUFFER_SIZE 1024
int main(int argc, char *argv[])
{
    int fd;
    ssize_t bytes_read;
    char buffer[BUFFER_SIZE];
    /* Check command-line argument */
    if (argc != 2)
    {
        printf("Usage: %s <filename>\n",
               argv[0]);
        return 1;
    }
    /* Open file */
    fd = open(argv[1], O_RDONLY);
    if (fd == -1)
    {
        perror("open");
        return 1;
    }
    /* Read and display file */
    while ((bytes_read =
            read(fd,
                 buffer,
                 sizeof(buffer))) > 0)
    {
        write(STDOUT_FILENO,
              buffer,
              bytes_read);
    }
    /* Check read error */
    if (bytes_read == -1)
    {
        perror("read");
    }
    /* Close file */
    close(fd);
    return 0;
}
```
## Explanation
### 1. Open File
```c
fd = open(argv[1], O_RDONLY);
```
Opens the file in **read-only mode**.

Returns:
* File descriptor (`fd >= 0`) → success
* `-1` → failure

Example:
```
fd = 3
```
## 2. Read Data
```c
read(fd,
     buffer,
     sizeof(buffer))
```
Reads bytes from file.

Returns:
* Positive number → bytes read
* `0` → EOF
* `-1` → error

Example:
```
File content:
HELLO
```
After reading:
```
buffer = "HELLO"
bytes_read = 5
```
## 3. Write to Standard Output
```c
write(STDOUT_FILENO,
      buffer,
      bytes_read);
```
Equivalent to:
```c
write(1, buffer, bytes_read);
```
Because:
```
STDOUT_FILENO = 1
```
Writes data to terminal.
## 4. EOF Handling
Loop:
```c
while ((bytes_read = read(...)) > 0)
```
continues until:
```
read() returns 0
```
meaning EOF.
## 5. Close File
```c
close(fd);
```
Releases kernel resources.
## Example
Suppose `sample.txt` contains:
```
Linux
System Programming
```
Run:
```
./cat sample.txt
```
Output:
```
Linux
System Programming
```
## Internal Working
```
open()
   ↓
Kernel returns fd
   ↓
read(fd, buffer, n)
   ↓
Kernel copies data
   ↓
write(stdout, buffer, n)
   ↓
Terminal output
   ↓
close(fd)
```
## Character-by-Character Version
Less efficient but educational:
```c
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main(int argc, char *argv[])
{
    int fd;
    char ch;
    if (argc != 2)
    {
        printf("Usage: %s <file>\n",
               argv[0]);
        return 1;
    }
    fd = open(argv[1], O_RDONLY);
    if (fd == -1)
    {
        perror("open");
        return 1;
    }
    while (read(fd, &ch, 1) > 0)
    {
        write(STDOUT_FILENO,
              &ch,
              1);
    }
    close(fd);
    return 0;
}
```
### Why Inefficient?
```
1 byte read
1 system call

1 byte write
1 system call
```
Too many kernel calls.

Buffered block I/O is better:
```
1024 bytes read
1024 bytes write
```
Fewer system calls → faster execution.
## ANSI I/O vs POSIX I/O for `cat`
| ANSI I/O              | POSIX I/O       |
| --------------------- | --------------- |
| `fopen()`             | `open()`        |
| `fgetc()` / `fgets()` | `read()`        |
| `fputc()` / `fputs()` | `write()`       |
| `fclose()`            | `close()`       |
| Buffered              | Unbuffered      |
| `FILE *`              | File descriptor |

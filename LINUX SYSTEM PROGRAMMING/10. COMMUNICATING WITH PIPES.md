# Introduction to Pipes
A **pipe** is a communication mechanism in operating systems (especially in Linux/UNIX) that allows **data to flow from one process to another**. It provides a way for processes to exchange information using a **FIFO (First In First Out)** method.

Think of a pipe like a water pipe:
* One process **writes** data into the pipe.
* Another process **reads** data from the pipe.
* Data comes out in the same order it was inserted (**FIFO**).
## Why Pipes Are Used
Pipes are mainly used for:
1. **Inter-Process Communication (IPC)** – communication between processes.
2. **Data sharing** between parent and child processes.
3. **Connecting commands** in Linux shell.

Example in Linux shell:
```
ls | wc -l
```
Here:
* `ls` lists files.
* `|` is the pipe operator.
* `wc -l` counts the number of lines.

Flow of execution:
```
ls ----> pipe ----> wc -l
```
The output of `ls` becomes the input of `wc -l`.
### Characteristics of Pipes
* **Unidirectional communication** → data flows in one direction only.
* **FIFO order** → first data written is first read.
* Used between **related processes** (usually parent-child).
* Implemented by the kernel using a buffer.
### Types of Pipes
#### 1. Anonymous Pipe (Unnamed Pipe)
* Created using `pipe()` system call.
* Used between related processes (parent-child).
* Exists only while processes are running.

Function syntax:
```c
#include <unistd.h>
int pipe(int fd[2]);
```
`fd[0]` → read end

`fd[1]` → write end

If successful:
```
returns 0
```
Otherwise:
```
returns -1
```
### Working of an Anonymous Pipe
```
Parent Process
      |
   pipe()
      |
   fork()
   /     \
Child    Parent
(read)   (write)
```
Example:
```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>
int main() {
    int fd[2];
    pid_t pid;
    char write_msg[] = "Hello Child";
    char read_msg[50];
    pipe(fd);
    pid = fork();
    if(pid > 0) {
        // Parent process
        close(fd[0]); // close read end
        write(fd[1], write_msg, strlen(write_msg)+1);
    }
    else {
        // Child process
        close(fd[1]); // close write end
        read(fd[0], read_msg, sizeof(read_msg));
        printf("Child received: %s\n", read_msg);
    }
    return 0;
}
```
### Explanation
1. `pipe(fd)` creates a pipe.
2. `fork()` creates child process.
3. Parent writes data using `write()`.
4. Child reads data using `read()`.
5. Unused ends of pipe are closed using `close()`.

Output:
```
Child received: Hello Child
```
## 2. Named Pipe (FIFO)
A named pipe allows communication between **unrelated processes**.

It is created using:
```c
mkfifo()
```
Example command:
```
mkfifo mypipe
```
Now one process can write:
```
echo "Hello" > mypipe
```
Another process can read:
```
cat < mypipe
```
### Advantages of Pipes
* Simple IPC mechanism.
* Efficient data transfer.
* Useful for chaining commands.
* Parent-child communication is easy.
### Limitations of Pipes
* Ordinary pipes are **one-way communication only**.
* Anonymous pipes work only between related processes.
* Limited buffer size.

# Standard I/O: `popen()` / `pclose()`
The functions `popen()` and `pclose()` are part of the **Standard I/O library** and are used to **execute a shell command and communicate with it through a pipe**.

They combine:
* **Pipe creation** (`pipe()`)
* **Process creation** (`fork()`)
* **Command execution** (`exec()`)

into a single interface.
## Purpose
`popen()` allows a program to:
* Execute another command/process.
* Read output from the command.
* Send input to the command.

It returns a **FILE pointer (`FILE *`)** so we can use standard I/O functions like:
```c
fgets()
fprintf()
fscanf()
fgetc()
fputs()
```
## `popen()` Syntax
```c
#include <stdio.h>
FILE *popen(const char *command, const char *type);
```
### Parameters
#### `command`
Shell command to execute.

Example:
```c
"ls -l"
"date"
"sort"
```
#### `type`
Specifies the communication mode:
| Mode  | Meaning                  |
| ----- | ------------------------ |
| `"r"` | Read from command output |
| `"w"` | Write to command input   |
### Return Value
* Returns `FILE *` on success.
* Returns `NULL` on failure.
## `pclose()` Syntax
```c
int pclose(FILE *stream);
```
### Purpose
Used to:
1. Close the pipe.
2. Wait for the child process to terminate.
3. Return child exit status.

Return:
```
Exit status of command
```
## How `popen()` Works
When:
```c
fp = popen("date", "r");
```
internally:
1. A pipe is created.
2. `fork()` creates child process.
3. Child executes shell command (`date`).
4. Parent gets stream (`FILE *`).

Flow:
```
Program ----pipe----> Command
```
For `"r"` mode:
```
Command output ---> Program
```
For `"w"` mode:
```
Program input ---> Command
```
## Example 1: Reading Command Output (`"r"` mode)
Program executes `date` command and reads output.
```c
#include <stdio.h>
int main() {
    FILE *fp;
    char buffer[100];
    fp = popen("date", "r");
    if(fp == NULL) {
        printf("popen failed\n");
        return 1;
    }
    fgets(buffer, sizeof(buffer), fp);
    printf("Current Date: %s", buffer);
    pclose(fp);
    return 0;
}
```
### Output (example)
```
Current Date: Sun May 24 11:20:15 IST 2026
```
### Explanation
* `popen("date","r")` runs `date`.
* Program reads command output using `fgets()`.
* `pclose()` closes pipe and waits for command completion.
## Example 2: Reading `ls` Output
```c
#include <stdio.h>
int main() {
    FILE *fp;
    char line[256];
    fp = popen("ls", "r");
    if(fp == NULL) {
        perror("popen");
        return 1;
    }
    while(fgets(line, sizeof(line), fp) != NULL) {
        printf("%s", line);
    }
    pclose(fp);
    return 0;
}
```
### Flow
```
ls command ---> pipe ---> program
```
The program reads filenames line by line.
## Example 3: Writing to Command (`"w"` mode)
Send input to `sort` command.
```c
#include <stdio.h>
int main() {
    FILE *fp;
    fp = popen("sort", "w");
    if(fp == NULL) {
        perror("popen");
        return 1;
    }
    fprintf(fp, "banana\n");
    fprintf(fp, "apple\n");
    fprintf(fp, "orange\n");
    pclose(fp);
    return 0;
}
```
### What Happens?
Program writes:
```
banana
apple
orange
```
to `sort`.

The `sort` command sorts them and prints:
```
apple
banana
orange
```
## Internal View
### `"r"` Mode
```
        Child Process
      (exec command)
            |
         writes output
            |
         PIPE
            |
       Parent Process
         reads data
```
### `"w"` Mode
```
Parent Process
 writes data
      |
     PIPE
      |
Child Process
(exec command)
 reads input
```
## Difference Between `pipe()` and `popen()`
| `pipe()`                              | `popen()`                           |
| ------------------------------------- | ----------------------------------- |
| Low-level system call                 | High-level standard I/O function    |
| Uses file descriptors                 | Uses `FILE *` stream                |
| Requires `fork()` + `exec()` manually | Automatically creates child process |
| More control                          | Easier to use                       |
## Advantages of `popen()`
* Simple to execute commands.
* Standard I/O stream support.
* Easier than `pipe()` + `fork()` + `exec()`.
* Useful for reading shell command output.
## Limitations
* Mostly unidirectional communication.
* Less control than manual `pipe()` implementation.
* Uses shell internally (security risk if command input is unsafe).

# System Call: `pipe()`
The `pipe()` system call is used to create a **pipe for inter-process communication (IPC)** between processes.

A pipe allows one process to **send data** and another process to **receive data**.

Communication happens in **FIFO order (First In First Out)**.
## Definition
A pipe is a **unidirectional communication channel**.

It has:
* **One write end** → used to send data
* **One read end** → used to receive data

Data written first is read first.

Flow:
```
Writer Process -----> PIPE -----> Reader Process
```
## Syntax of `pipe()`
```c
#include <unistd.h>
int pipe(int fd[2]);
```
### Parameter
`fd[2]` is an integer array containing two file descriptors.

After successful execution:
| Descriptor | Purpose   |
| ---------- | --------- |
| `fd[0]`    | Read end  |
| `fd[1]`    | Write end |
## Return Value
| Value | Meaning |
| ----- | ------- |
| `0`   | Success |
| `-1`  | Failure |
Example:
```c
if(pipe(fd) == -1)
{
    perror("pipe");
}
```
## How `pipe()` Works
Suppose:
```c
int fd[2];
pipe(fd);
```
Kernel creates:
```
fd[1] (write end) ----> [ KERNEL PIPE BUFFER ] ----> fd[0] (read end)
```
* Data written to `fd[1]`
* Data read from `fd[0]`

FIFO behavior:
```
Write: A B C
Read : A B C
```
## Why `fork()` Is Used with `pipe()`
Normally `pipe()` is used with `fork()` because parent and child inherit file descriptors.

Example flow:
```
Parent
   |
 pipe()
   |
 fork()
 /     \
Parent  Child
write    read
```
The parent writes into pipe, child reads from it.
## Example 1: Parent Writes, Child Reads
```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>
int main()
{
    int fd[2];
    pid_t pid;
    char msg[] = "Hello Child";
    char buffer[100];
    pipe(fd);
    pid = fork();
    if(pid > 0)
    {
        /* Parent process */
        close(fd[0]);        // close read end
        write(fd[1], msg, strlen(msg)+1);
    }
    else
    {
        /* Child process */
        close(fd[1]);        // close write end
        read(fd[0], buffer, sizeof(buffer));
        printf("Child received: %s\n", buffer);
    }
    return 0;
}
```
Output:
```
Child received: Hello Child
```
## Explanation
### Step 1: Create pipe
```c
pipe(fd);
```
Creates:
```
fd[0] -> read
fd[1] -> write
```
### Step 2: Create child
```c
fork();
```
Both parent and child inherit descriptors.
### Step 3: Parent writes
```c
write(fd[1], msg, strlen(msg)+1);
```
Message enters pipe.
### Step 4: Child reads
```c
read(fd[0], buffer, sizeof(buffer));
```
Child receives message.
## Why `close()` Is Necessary
Unused pipe ends should be closed.

Parent:
```c
close(fd[0]);
```
because it only writes.

Child:
```c
close(fd[1]);
```
because it only reads.

Without closing:
* Resources are wasted
* Read may block unexpectedly
* EOF detection becomes difficult

Correct structure:
```
Parent (write only)
close(read end)

Child (read only)
close(write end)
```
## Example 2: Child Writes, Parent Reads
```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>
int main()
{
    int fd[2];
    pid_t pid;
    char buffer[100];
    pipe(fd);
    pid = fork();
    if(pid == 0)
    {
        close(fd[0]);
        write(fd[1], "Hello Parent", 13);
    }
    else
    {
        close(fd[1]);
        read(fd[0], buffer, sizeof(buffer));
        printf("Parent received: %s\n", buffer);
    }
    return 0;
}
```
Output:
```
Parent received: Hello Parent
```
## Internal Representation
```
          PIPE
    ----------------
fd[1] --> BUFFER --> fd[0]
(write)            (read)
```
## Important System Calls Used with `pipe()`
| Function  | Purpose              |
| --------- | -------------------- |
| `pipe()`  | Create pipe          |
| `fork()`  | Create child process |
| `read()`  | Read from pipe       |
| `write()` | Write into pipe      |
| `close()` | Close unused end     |
## Characteristics of `pipe()`
* Unidirectional communication
* FIFO mechanism
* Kernel-managed buffer
* Usually parent-child communication
* Temporary (exists while processes run)
## Advantages
* Simple IPC mechanism
* Fast communication
* Efficient for related processes
## Limitations
* One-way communication only
* Works best with related processes
* Limited buffer size
## Difference Between `pipe()` and `popen()`
| `pipe()`                 | `popen()`                   |
| ------------------------ | --------------------------- |
| Low-level system call    | High-level library function |
| Uses file descriptors    | Uses `FILE *`               |
| Requires manual `fork()` | Automatic process creation  |
| More control             | Easier to use               |

# Using `pipe()`
The `pipe()` system call is used for **Inter-Process Communication (IPC)** between processes, usually between a **parent and child process**.

It creates a **communication channel** through which one process writes data and another process reads it.

`pipe()` is generally used together with:
* `fork()`
* `read()`
* `write()`
* `close()`
## Syntax
```c
#include <unistd.h>
int pipe(int fd[2]);
```
### Parameter
```c
int fd[2];
```
After `pipe()` executes:
| File Descriptor | Purpose   |
| --------------- | --------- |
| `fd[0]`         | Read end  |
| `fd[1]`         | Write end |
## Return Value
| Return | Meaning |
| ------ | ------- |
| `0`    | Success |
| `-1`   | Failure |
Example:
```c
int fd[2];
if(pipe(fd) == -1)
{
    perror("pipe");
    return 1;
}
```
## Steps for Using `pipe()`
Typical usage of `pipe()` follows these steps:
### Step 1: Create pipe
```c
int fd[2];
pipe(fd);
```
Kernel creates:
```
fd[1] ----> PIPE BUFFER ----> fd[0]
(write)                     (read)
```
## Step 2: Create child process
```c
fork();
```
After `fork()`:
* Parent and child both inherit pipe descriptors.
```
Parent Process
     |
   fork()
     |
  Child Process
```
## Step 3: Close unused pipe ends
Parent writing:
```c
close(fd[0]);
```
Child reading:
```c
close(fd[1]);
```
This avoids:
* resource leaks
* blocking problems
* incorrect EOF behavior
## Step 4: Write into pipe
Using `write()`:
```c
write(fd[1], msg, strlen(msg)+1);
```
## Step 5: Read from pipe
Using `read()`:
```c
read(fd[0], buffer, sizeof(buffer));
```
## Example 1: Parent Writes, Child Reads
```c
#include <stdio.h>
#include <unistd.h>
#include <string.h>
int main()
{
    int fd[2];
    pid_t pid;
    char msg[] = "Hello Child";
    char buffer[100];
    if(pipe(fd) == -1)
    {
        perror("pipe");
        return 1;
    }
    pid = fork();
    if(pid > 0)
    {
        /* Parent process */
        close(fd[0]);     // close read end
        write(fd[1], msg, strlen(msg)+1);
        close(fd[1]);
    }
    else
    {
        /* Child process */
        close(fd[1]);     // close write end
        read(fd[0], buffer, sizeof(buffer));
        printf("Child received: %s\n", buffer);
        close(fd[0]);
    }
    return 0;
}
```
### Output
```
Child received: Hello Child
```
## Working of Example
### Pipe Creation
```
fd[1] ----> PIPE ----> fd[0]
 write                 read
```
### After `fork()`
```
            Parent
               |
             fork()
            /      \
      Parent        Child
       write         read
```
### Data Flow
```
Parent
write(fd[1])
      |
      V
   PIPE BUFFER
      |
      V
Child
read(fd[0])
```
## Example 2: Child Writes, Parent Read
```c

{

```



```
FIFO (First In First Out)
```

Kernel buffer representation:
```
          PIPE BUFFER
--------------------------------

--------------------------------

Parent writes -----> Child reads
```
## Important Rules While Using `pipe()`
### 1. Always close unused ends
Parent:
```c
close(fd[0]);
```
Child:
```c
close(fd[1]);
```
## 2. Use `fork()` after `pipe()`
Correct:
```c
pipe(fd);
fork();
```
Wrong:
```c

```

```c
if(pipe(fd) == -1)
{
    perror("pipe");
}
```
## 4. Pipe is unidirectional
One pipe:
```
Parent ----> Child
```
For two-way communication:
```
Parent <----> Child
```
two pipes are needed.
## Example: Two-Way Communication
```
Pipe1: Parent ---> Child
Pipe2: Child ---> Parent
```

## Advantages
* Fast IPC
* Simple communication
* Efficient parent-child interaction
## Limitations

Using `pipe()` involves:
1. Create pipe

# Named Pipes
A **Named Pipe**, also called a **FIFO (First In First Out)**, is a special type of pipe used for **Inter-Process Communication (IPC)** between **unrelated processes**.

Unlike ordinary (anonymous) pipes:
* Anonymous pipes are mainly used between **parent and child processes**.
* Named pipes allow **independent/unrelated processes** to communicate.

The data transfer still follows:

# Using Named Pipes
A **Named Pipe (FIFO)** is used for **Inter-Process Communication (IPC)** between processes, especially **unrelated processes**.

Unlike anonymous pipes (`pipe()`), named pipes have a **name in the filesystem**, allowing different programs to communicate.

Usage flow:
```
Writer Process ----> Named Pipe (FIFO) ----> Reader Process
```
Named pipes are created using:
```c
mkfifo()
```
and accessed like normal files using:
```c
open()
read()
write()
close()
```
## Steps for Using Named Pipes
### Step 1: Create FIFO
Use `mkfifo()`.

Syntax:
```c
#include <sys/types.h>
#include <sys/stat.h>
int mkfifo(const char *pathname, mode_t mode);
```
Example:
```c
mkfifo("mypipe", 0666);
```
Here:
* `"mypipe"` → FIFO name
* `0666` → read/write permissions
## Step 2: Open FIFO
Open like a file.

For writing:
```c
open("mypipe", O_WRONLY);
```
For reading:
```c
open("mypipe", O_RDONLY);
```
## Step 3: Read or Write Data
Write:
```c
write(fd, msg, strlen(msg)+1);
```
Read:
```c
read(fd, buffer, sizeof(buffer));
```
## Step 4: Close FIFO
```c
close(fd);
```
## Example 1: Simple Writer Program
This program writes data into a named pipe.
```c
#include <stdio.h>
#include <unistd.h>
#include <fcntl.h>
#include <sys/stat.h>
#include <string.h>
int main()
{
    int fd;
    char msg[] = "Hello Reader";
```

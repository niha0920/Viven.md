# What a Process is 
A **process** is a program that is currently being executed by the operating system.

In simple terms:
> **Program = passive (stored on disk)**

> **Process = active (running in memory)**

For example:
* A C program stored as `hello.c` or compiled executable `a.out` is just a **program**.
* When you run it (`./a.out`), the operating system loads it into memory and executes it — now it becomes a **process**.
## Characteristics of a Process
A process contains:
1. **Program Code (Text Segment)**

   The actual instructions to execute.
2. **Data Section**

   Global and static variables.
3. **Heap**

   Dynamically allocated memory (`malloc()` in C).
4. **Stack**

   Function calls, local variables, return addresses.
5. **Program Counter**

   Keeps track of the next instruction to execute.
6. **CPU Registers**

   Store temporary execution data.
### Process Memory Layout
```
+------------------+
|     Stack        | ← Local variables, function calls
+------------------+
|      Heap        | ← Dynamic memory (malloc)
+------------------+
|   Data Segment   | ← Global/static variables
+------------------+
|   Text Segment   | ← Program instructions
+------------------+
```
### Example
Consider this C program:
```c
#include <stdio.h>
int main() {
    printf("Hello World\n");
    return 0;
}
```
* Before execution → it is a **program**.
* After running → OS creates a **process** for it.
### Process States
A process generally moves through these states:
```
New → Ready → Running → Waiting/Blocked → Ready → Terminated
```
* **New** → Process is being created
* **Ready** → Waiting for CPU
* **Running** → Executing on CPU
* **Waiting/Blocked** → Waiting for I/O/event
* **Terminated** → Finished execution
### In Linux
Every process has a unique identifier called **PID (Process ID)**.

You can see running processes using:
```
ps
```
or
```
top
```
Example:
```
$ ps
PID   TTY     TIME   CMD
1234  pts/0   00:00  bash
5678  pts/0   00:00  a.out
```
Here `5678` is the PID of the running process.

# Process Relationships
In an operating system, processes can have relationships with other processes, mainly through **creation and hierarchy**.

The most important relationship is:
> **Parent Process ↔ Child Process**

When one process creates another process, the creator is called the **parent process**, and the created process is called the **child process**.

In Linux, this is commonly done using the `fork()` system call.
## Parent and Child Process
```
Parent Process
      |
   fork()
      |
      ├── Child Process 1
      ├── Child Process 2
      └── Child Process 3
```
Example:
* A shell (`bash`) runs a command.
* The shell creates a new process to execute the command.

Example hierarchy:
```
init/systemd (PID 1)
        |
      bash
        |
      gcc
```
Here:
* `systemd` is parent of `bash`
* `bash` is parent of `gcc`

You can view process hierarchy in Linux using:
```
pstree
```
Example output:
```
systemd─┬─bash───vim
        └─sshd───bash
```
## Types of Process Relationships
### 1. Parent Process
A process that creates another process.

Example:
* Shell creates a program.
```c
pid_t pid = fork();
```
The process calling `fork()` becomes the parent.
### 2. Child Process
The newly created process.

After `fork()`:
* Parent and child execute independently.
* Child gets a copy of the parent’s memory space.
## Sibling Processes
Processes created by the same parent are called **siblings**.
```
        Parent
       /   |   \
    Child1 Child2 Child3
```
Here, Child1, Child2, and Child3 are sibling processes.
## Orphan Process
If a parent terminates before the child finishes, the child becomes an **orphan process**.

Then the OS assigns it to the process with PID 1 (usually `systemd` or `init`).
```
Parent (dies)
      X
      |
    Child  → adopted by PID 1
```
Example:
* Parent exits.
* Child still running.

The child continues execution.
## Zombie Process
A zombie process is a process that has completed execution but still has an entry in the process table because the parent has not collected its exit status.
```
Parent running
      |
   Child exits
      |
 Zombie (waiting for parent to read status)
```
This happens when the parent does not call functions like:
```c
wait();
```
or
```c
waitpid();
```
Zombie consumes almost no memory but occupies a process table entry.
## Process Tree in Linux
Linux processes are arranged in a hierarchical tree.

At the top:

systemd (or historically `init`) → PID 1

Example:
```
systemd (PID 1)
 ├── sshd
 │    └── bash
 │         └── vim
 └── chrome
```
## Important System Calls in Process Relationships
| System Call     | Purpose                |
| --------------- | ---------------------- |
| `fork()`        | Create child process   |
| `exec()` family | Replace process image  |
| `wait()`        | Parent waits for child |
| `getpid()`      | Get process ID         |
| `getppid()`     | Get parent process ID  |
## Example Program
```c
#include <stdio.h>
#include <unistd.h>
int main() {
    pid_t pid = fork();
    if(pid == 0) {
        printf("Child Process\n");
    } else {
        printf("Parent Process\n");
    }
    return 0;
}
```
Output:
```
Parent Process
Child Process
```
After `fork()`:
* Parent and child run concurrently.
* Both continue from the next instruction after `fork()`.

# Create a Child Process
In Linux, a child process is created using the `fork()` system call.
## What is `fork()`?
`fork()` creates a **new process** (child process) by duplicating the calling process (parent process).

Syntax:
```c
pid_t fork(void);
```
After `fork()`:
* A **child process** is created.
* Parent and child execute **independently**.
* Both continue execution from the statement immediately after `fork()`.
### Return Value of `fork()`
| Return Value | Meaning                                         |
| ------------ | ----------------------------------------------- |
| `0`          | Executing in child process                      |
| `> 0`        | Executing in parent process (returns child PID) |
| `< 0`        | Error in process creation                       |
## Process Creation Flow
```
Parent Process
      |
    fork()
      |
 ┌────┴────┐
 |         |
Parent    Child
```
## Example Program: Create a Child Process
```c
#include <stdio.h>
#include <unistd.h>
int main() {
    pid_t pid;
    pid = fork();
    if(pid < 0) {
        printf("Fork failed\n");
    }
    else if(pid == 0) {
        printf("Child Process\n");
        printf("PID = %d\n", getpid());
        printf("Parent PID = %d\n", getppid());
    }
    else {
        printf("Parent Process\n");
        printf("PID = %d\n", getpid());
        printf("Child PID = %d\n", pid);
    }
    return 0;
}
```
### Sample Output
```
Parent Process
PID = 1200
Child PID = 1201

Child Process
PID = 1201
Parent PID = 1200
```
### Explanation
1. Parent process starts execution.
2. `fork()` is called.
3. OS creates a child process.
4. Execution splits into two flows:
* **Parent process** (`pid > 0`)
* **Child process** (`pid == 0`)

Both execute concurrently.
### How `fork()` Works
```
Before fork()

Parent Process
     |
   fork()

After fork()

Parent Process -------->
Child Process  -------->
```
The child gets a copy of:
* Program code
* Variables
* File descriptors
* Environment

But parent and child have **different PIDs**.
## Example with Variable Copy
```c
#include <stdio.h>
#include <unistd.h>
int main() {
    int x = 10;
    pid_t pid = fork();
    if(pid == 0) {
        x = 20;
        printf("Child x = %d\n", x);
    } else {
        printf("Parent x = %d\n", x);
    }
    return 0;
}
```
Possible output:
```
Parent x = 10
Child x = 20
```
Explanation:
* Child receives a copy of `x`.
* Changing `x` in child does not affect parent.
### Compile and Run
```
gcc child.c -o child
./child
```
### Related System Calls
| System Call | Purpose                 |
| ----------- | ----------------------- |
| `fork()`    | Create child process    |
| `getpid()`  | Get current process ID  |
| `getppid()` | Get parent process ID   |
| `wait()`    | Parent waits for child  |
| `exec()`    | Execute another program |

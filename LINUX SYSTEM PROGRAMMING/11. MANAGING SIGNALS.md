# What Signals Are
In Linux system programming, **signals** are software-generated notifications sent to a process to indicate that a specific event has occurred. They are a form of **inter-process communication (IPC)** used by the operating system or other processes to interrupt a program and notify it about an event.

Think of a signal as a **software interrupt** that tells a process: *“Something important happened—respond to it.”*
## Why Signals Are Used
Signals are used to:
* Notify a process about errors or exceptional conditions
* Control process execution (stop, continue, terminate)
* Enable communication between processes
* Handle asynchronous events
### How Signals Work
1. A signal is generated by:
   * The kernel (OS)
   * Another process
   * The process itself
2. The signal is delivered to the target process.
3. The process responds in one of these ways:
   * **Default action** → predefined action by OS
   * **Ignore the signal**
   * **Catch the signal** using a signal handler function
### Common Signals in Linux
| Signal    | Meaning                                    | Default Action          |
| --------- | ------------------------------------------ | ----------------------- |
| `SIGINT`  | Interrupt from keyboard (`Ctrl+C`)         | Terminate process       |
| `SIGKILL` | Forcefully kill a process                  | Terminate immediately   |
| `SIGTERM` | Request process termination                | Terminate               |
| `SIGSTOP` | Stop (pause) a process                     | Stop execution          |
| `SIGCONT` | Continue a stopped process                 | Resume execution        |
| `SIGSEGV` | Invalid memory access (Segmentation fault) | Terminate               |
| `SIGALRM` | Timer alarm signal                         | Terminate               |
| `SIGCHLD` | Child process status changed               | Ignore/default handling |
### Example
If a running C program is executing and you press **Ctrl + C**, the terminal sends `SIGINT` to that process, causing it to terminate unless the program handles the signal.

Example C program handling a signal:
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("Signal %d received\n", sig);
}
int main()
{
    signal(SIGINT, handler);
    while (1)
    {
        printf("Running...\n");
        sleep(1);
    }
    return 0;
}
```
**Output behavior:**
* Program keeps running.
* Press `Ctrl+C`
* Instead of terminating immediately, the signal handler runs and prints:
```
Signal 2 received
```

# Blocking/Checking Signals
In Linux system programming, **blocking signals** means temporarily preventing a process from receiving certain signals. The signal is not lost; it becomes **pending** and is delivered when the signal is unblocked.
## Why Block Signals?
Signals are blocked to:
* Protect critical sections of code
* Avoid interruption during sensitive operations
* Prevent race conditions in signal handling

For example, if a process is updating shared data, interruption by a signal may cause inconsistent results. Blocking prevents this.
### Signal Mask
A process maintains a **signal mask**, which is a set of blocked signals.

To manipulate the signal mask, Linux provides:
* `sigprocmask()` → block/unblock signals
* `sigpending()` → check pending signals
* `sigismember()` → check if a signal belongs to a set
## 1. Blocking Signals using `sigprocmask()`
Function prototype:
```c 
#include <signal.h>
int sigprocmask(int how,
                const sigset_t *set,
                sigset_t *oldset);
```
### Parameters
* `how` → action to perform:
  * `SIG_BLOCK` → add signals to blocked set
  * `SIG_UNBLOCK` → remove signals from blocked set
  * `SIG_SETMASK` → replace current mask
* `set` → signal set to manipulate
* `oldset` → stores previous signal mask
### Steps for Blocking a Signal
1. Create signal set (`sigset_t`)
2. Initialize using `sigemptyset()`
3. Add signal using `sigaddset()`
4. Block signal using `sigprocmask()`

Example:
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    // Block SIGINT
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("SIGINT blocked for 10 seconds.\n");
    sleep(10);
    printf("Unblocking SIGINT...\n");
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    while (1);
    return 0;
}
```
### Working
* Press `Ctrl+C` during the first 10 seconds.
* `SIGINT` is blocked, so process will not terminate.
* Signal becomes **pending**.
* After unblocking, pending `SIGINT` is delivered immediately.
## 2. Checking Pending Signals using `sigpending()`
Prototype:
```c 
#include <signal.h>
int sigpending(sigset_t *set);
```
This function returns the set of **pending blocked signals**.

Example:
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set, pending;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("Press Ctrl+C now\n");
    sleep(5);
    sigpending(&pending);
    if (sigismember(&pending, SIGINT))
        printf("SIGINT is pending\n");
    else
        printf("SIGINT not pending\n");
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    return 0;
}
```
### Working
* `SIGINT` is blocked.
* User presses `Ctrl+C`.
* Signal cannot be delivered immediately → becomes pending.
* `sigpending()` checks pending signals.
* `sigismember()` confirms whether `SIGINT` exists in pending set.
## 3. Checking a Signal using `sigismember()`
Prototype:
```c 
int sigismember(const sigset_t *set, int signo);
```
Returns:
* `1` → signal present in set
* `0` → not present
* `-1` → error

Example:
```c
if(sigismember(&pending, SIGINT))
{
    printf("SIGINT pending\n");
}
```
## Flow of Blocking and Checking Signals
```
Create signal set
       ↓
Initialize (sigemptyset)
       ↓
Add signal (sigaddset)
       ↓
Block signal (sigprocmask)
       ↓
Signal arrives → becomes pending
       ↓
Check pending (sigpending)
       ↓
Check membership (sigismember)
       ↓
Unblock signal (sigprocmask)
       ↓
Pending signal delivered
```

# Working with Signal Sets
A **signal set** is a collection of signals represented using the `sigset_t` data type. Signal sets are used to **block, unblock, add, remove, or test signals** in Linux signal handling.

Instead of dealing with one signal at a time, signal sets allow you to manage **multiple signals together**.
## Why Signal Sets Are Needed
Signal sets are used for:
* Blocking multiple signals simultaneously
* Managing pending signals
* Installing signal masks
* Testing whether a signal exists in a set

The data type used is:
```c
sigset_t
```
## Common Functions for Signal Sets
| Function        | Purpose                        |
| --------------- | ------------------------------ |
| `sigemptyset()` | Initialize an empty signal set |
| `sigfillset()`  | Fill set with all signals      |
| `sigaddset()`   | Add a signal to set            |
| `sigdelset()`   | Remove a signal from set       |
| `sigismember()` | Check if signal exists in set  |
## 1. `sigemptyset()` – Create an Empty Set
This function initializes a signal set with **no signals**.

Prototype:
```c
#include <signal.h>
int sigemptyset(sigset_t *set);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    printf("Empty signal set created\n");
    return 0;
}
```
### Working
Initially, the signal set contains nothing.
```
{ }
```
## 2. `sigfillset()` – Fill Set with All Signals
This function adds **all possible signals** to the set.

Prototype:
```c
#include <signal.h>
int sigfillset(sigset_t *set);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigfillset(&set);
    printf("Set filled with signals\n");
    return 0;
}
```
### Working
Conceptually:
```
{SIGINT, SIGTERM, SIGALRM, SIGUSR1, ...}
```
Note:

`SIGKILL` and `SIGSTOP` still **cannot be blocked or handled**, even if present conceptually in a full set.
## 3. `sigaddset()` – Add Signal to a Set
Adds a specific signal into a signal set.

Prototype:
```c
#include <signal.h>
int sigaddset(sigset_t *set, int signo);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigaddset(&set, SIGTERM);
    printf("Signals added\n");
    return 0;
}
```
### Working
After adding:
```
{SIGINT, SIGTERM}
```
## 4. `sigdelset()` – Remove Signal from Set
Removes a signal from a signal set.

Prototype:
```c
#include <signal.h>
int sigdelset(sigset_t *set, int signo);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigfillset(&set);
    sigdelset(&set, SIGINT);
    printf("SIGINT removed from set\n");
    return 0;
}
```
### Working
Before:
```
{SIGINT, SIGTERM, SIGUSR1, ...}
```
After:
```
{SIGTERM, SIGUSR1, ...}
```
## 5. `sigismember()` – Check Signal Presence
Checks whether a signal exists in a signal set.

Prototype:
```c
#include <signal.h>
int sigismember(const sigset_t *set, int signo);
```
Returns:
* `1` → signal exists in set
* `0` → signal not present
* `-1` → error

Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    if (sigismember(&set, SIGINT))
        printf("SIGINT present\n");
    else
        printf("SIGINT absent\n");
    return 0;
}
```
Output:
```
SIGINT present
```
## Example: Working with Multiple Signal Set Operations
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigaddset(&set, SIGTERM);
    printf("Added SIGINT and SIGTERM\n");
    sigdelset(&set, SIGTERM);
    if(sigismember(&set, SIGTERM))
        printf("SIGTERM present\n");
    else
        printf("SIGTERM removed\n");
    return 0;
}
```
Output:
```
Added SIGINT and SIGTERM
SIGTERM removed
```
### Flow of Signal Set Operations
```
Create sigset_t
       ↓
sigemptyset() / sigfillset()
       ↓
sigaddset()
       ↓
sigdelset()
       ↓
sigismember()
       ↓
Use set with sigprocmask()
```

# Example of Blocking Signals
This example demonstrates how to **block a signal** using `sigprocmask()`. Here, `SIGINT` (generated by **Ctrl+C**) is blocked temporarily.
## Program
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set;
    /* Initialize empty signal set */
    sigemptyset(&set);
    /* Add SIGINT to signal set */
    sigaddset(&set, SIGINT);
    /* Block SIGINT */
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("SIGINT blocked for 10 seconds.\n");
    printf("Press Ctrl+C now...\n");
    sleep(10);
    printf("Unblocking SIGINT...\n");
    /* Unblock SIGINT */
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    while (1)
    {
        printf("Program running...\n");
        sleep(1);
    }
    return 0;
}
```
## How the Program Works
### Step 1: Create a Signal Set
```c
sigset_t set;
```
A variable of type `sigset_t` is created to hold signals.
## Step 2: Initialize Empty Set
```c
sigemptyset(&set);
```
Initially, the set contains no signals.

Conceptually:
```
{ }
```
## Step 3: Add `SIGINT`
```c
sigaddset(&set, SIGINT);
```
`SIGINT` is added to the signal set.

Conceptually:
```
{SIGINT}
```
## Step 4: Block the Signal
```c
sigprocmask(SIG_BLOCK, &set, NULL);
```
This blocks `SIGINT`.

Now if the user presses **Ctrl+C**, the signal is **not immediately delivered**.

Instead, it becomes **pending**.

---

### Step 5: Wait for 10 Seconds

```c
sleep(10);
```

During this period:

* `SIGINT` is blocked
* Pressing `Ctrl+C` does not terminate the process
* Signal is stored as pending

---

### Step 6: Unblock Signal

```c
sigprocmask(SIG_UNBLOCK, &set, NULL);
```

When unblocked:

* Pending `SIGINT` is delivered immediately
* Default action executes (program terminates)

---

## Execution Example

Program starts:

```text
SIGINT blocked for 10 seconds.
Press Ctrl+C now...
```

User presses **Ctrl+C** during sleep.

Nothing happens immediately.

After 10 seconds:

```text
Unblocking SIGINT...
```

Then pending signal executes:

```text
^C
```

Program terminates.

---

## Process Flow

```text
Create signal set
        ↓
sigemptyset()
        ↓
sigaddset(SIGINT)
        ↓
sigprocmask(SIG_BLOCK)
        ↓
Ctrl+C pressed
        ↓
SIGINT becomes pending
        ↓
sleep(10)
        ↓
sigprocmask(SIG_UNBLOCK)
        ↓
Pending SIGINT delivered
        ↓
Program terminates
```

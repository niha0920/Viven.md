# What Signals Are
In Linux system programming, **signals** are software-generated notifications sent to a process to indicate that a specific event has occurred. They are a form of **inter-process communication (IPC)** used by the operating system or other processes to interrupt a program and notify it about an event.

Think of a signal as a **software interrupt** that tells a process: *“Something important happened—respond to it.”*
## Why Signals Are Used
Signals are used to:
* Notify a process about errors or exceptional conditions
* Control process execution (stop, continue, terminate)
* Enable communication between processes
* Handle asynchronous events
### How Signals Work
1. A signal is generated by:
   * The kernel (OS)
   * Another process
   * The process itself
2. The signal is delivered to the target process.
3. The process responds in one of these ways:
   * **Default action** → predefined action by OS
   * **Ignore the signal**
   * **Catch the signal** using a signal handler function
### Common Signals in Linux
| Signal    | Meaning                                    | Default Action          |
| --------- | ------------------------------------------ | ----------------------- |
| `SIGINT`  | Interrupt from keyboard (`Ctrl+C`)         | Terminate process       |
| `SIGKILL` | Forcefully kill a process                  | Terminate immediately   |
| `SIGTERM` | Request process termination                | Terminate               |
| `SIGSTOP` | Stop (pause) a process                     | Stop execution          |
| `SIGCONT` | Continue a stopped process                 | Resume execution        |
| `SIGSEGV` | Invalid memory access (Segmentation fault) | Terminate               |
| `SIGALRM` | Timer alarm signal                         | Terminate               |
| `SIGCHLD` | Child process status changed               | Ignore/default handling |
### Example
If a running C program is executing and you press **Ctrl + C**, the terminal sends `SIGINT` to that process, causing it to terminate unless the program handles the signal.

Example C program handling a signal:
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("Signal %d received\n", sig);
}
int main()
{
    signal(SIGINT, handler);
    while (1)
    {
        printf("Running...\n");
        sleep(1);
    }
    return 0;
}
```
**Output behavior:**
* Program keeps running.
* Press `Ctrl+C`
* Instead of terminating immediately, the signal handler runs and prints:
```
Signal 2 received
```

# Blocking/Checking Signals
In Linux system programming, **blocking signals** means temporarily preventing a process from receiving certain signals. The signal is not lost; it becomes **pending** and is delivered when the signal is unblocked.
## Why Block Signals?
Signals are blocked to:
* Protect critical sections of code
* Avoid interruption during sensitive operations
* Prevent race conditions in signal handling

For example, if a process is updating shared data, interruption by a signal may cause inconsistent results. Blocking prevents this.
### Signal Mask
A process maintains a **signal mask**, which is a set of blocked signals.

To manipulate the signal mask, Linux provides:
* `sigprocmask()` → block/unblock signals
* `sigpending()` → check pending signals
* `sigismember()` → check if a signal belongs to a set
## 1. Blocking Signals using `sigprocmask()`
Function prototype:
```c 
#include <signal.h>
int sigprocmask(int how,
                const sigset_t *set,
                sigset_t *oldset);
```
### Parameters
* `how` → action to perform:
  * `SIG_BLOCK` → add signals to blocked set
  * `SIG_UNBLOCK` → remove signals from blocked set
  * `SIG_SETMASK` → replace current mask
* `set` → signal set to manipulate
* `oldset` → stores previous signal mask
### Steps for Blocking a Signal
1. Create signal set (`sigset_t`)
2. Initialize using `sigemptyset()`
3. Add signal using `sigaddset()`
4. Block signal using `sigprocmask()`

Example:
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    // Block SIGINT
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("SIGINT blocked for 10 seconds.\n");
    sleep(10);
    printf("Unblocking SIGINT...\n");
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    while (1);
    return 0;
}
```
### Working
* Press `Ctrl+C` during the first 10 seconds.
* `SIGINT` is blocked, so process will not terminate.
* Signal becomes **pending**.
* After unblocking, pending `SIGINT` is delivered immediately.
## 2. Checking Pending Signals using `sigpending()`
Prototype:
```c 
#include <signal.h>
int sigpending(sigset_t *set);
```
This function returns the set of **pending blocked signals**.

Example:
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set, pending;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("Press Ctrl+C now\n");
    sleep(5);
    sigpending(&pending);
    if (sigismember(&pending, SIGINT))
        printf("SIGINT is pending\n");
    else
        printf("SIGINT not pending\n");
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    return 0;
}
```
### Working
* `SIGINT` is blocked.
* User presses `Ctrl+C`.
* Signal cannot be delivered immediately → becomes pending.
* `sigpending()` checks pending signals.
* `sigismember()` confirms whether `SIGINT` exists in pending set.
## 3. Checking a Signal using `sigismember()`
Prototype:
```c 
int sigismember(const sigset_t *set, int signo);
```
Returns:
* `1` → signal present in set
* `0` → not present
* `-1` → error

Example:
```c
if(sigismember(&pending, SIGINT))
{
    printf("SIGINT pending\n");
}
```
## Flow of Blocking and Checking Signals
```
Create signal set
       ↓
Initialize (sigemptyset)
       ↓
Add signal (sigaddset)
       ↓
Block signal (sigprocmask)
       ↓
Signal arrives → becomes pending
       ↓
Check pending (sigpending)
       ↓
Check membership (sigismember)
       ↓
Unblock signal (sigprocmask)
       ↓
Pending signal delivered
```

# Working with Signal Sets
A **signal set** is a collection of signals represented using the `sigset_t` data type. Signal sets are used to **block, unblock, add, remove, or test signals** in Linux signal handling.

Instead of dealing with one signal at a time, signal sets allow you to manage **multiple signals together**.
## Why Signal Sets Are Needed
Signal sets are used for:
* Blocking multiple signals simultaneously
* Managing pending signals
* Installing signal masks
* Testing whether a signal exists in a set

The data type used is:
```c
sigset_t
```
## Common Functions for Signal Sets
| Function        | Purpose                        |
| --------------- | ------------------------------ |
| `sigemptyset()` | Initialize an empty signal set |
| `sigfillset()`  | Fill set with all signals      |
| `sigaddset()`   | Add a signal to set            |
| `sigdelset()`   | Remove a signal from set       |
| `sigismember()` | Check if signal exists in set  |
## 1. `sigemptyset()` – Create an Empty Set
This function initializes a signal set with **no signals**.

Prototype:
```c
#include <signal.h>
int sigemptyset(sigset_t *set);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    printf("Empty signal set created\n");
    return 0;
}
```
### Working
Initially, the signal set contains nothing.
```
{ }
```
## 2. `sigfillset()` – Fill Set with All Signals
This function adds **all possible signals** to the set.

Prototype:
```c
#include <signal.h>
int sigfillset(sigset_t *set);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigfillset(&set);
    printf("Set filled with signals\n");
    return 0;
}
```
### Working
Conceptually:
```
{SIGINT, SIGTERM, SIGALRM, SIGUSR1, ...}
```
Note:

`SIGKILL` and `SIGSTOP` still **cannot be blocked or handled**, even if present conceptually in a full set.
## 3. `sigaddset()` – Add Signal to a Set
Adds a specific signal into a signal set.

Prototype:
```c
#include <signal.h>
int sigaddset(sigset_t *set, int signo);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigaddset(&set, SIGTERM);
    printf("Signals added\n");
    return 0;
}
```
### Working
After adding:
```
{SIGINT, SIGTERM}
```
## 4. `sigdelset()` – Remove Signal from Set
Removes a signal from a signal set.

Prototype:
```c
#include <signal.h>
int sigdelset(sigset_t *set, int signo);
```
Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigfillset(&set);
    sigdelset(&set, SIGINT);
    printf("SIGINT removed from set\n");
    return 0;
}
```
### Working
Before:
```
{SIGINT, SIGTERM, SIGUSR1, ...}
```
After:
```
{SIGTERM, SIGUSR1, ...}
```
## 5. `sigismember()` – Check Signal Presence
Checks whether a signal exists in a signal set.

Prototype:
```c
#include <signal.h>
int sigismember(const sigset_t *set, int signo);
```
Returns:
* `1` → signal exists in set
* `0` → signal not present
* `-1` → error

Example:
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    if (sigismember(&set, SIGINT))
        printf("SIGINT present\n");
    else
        printf("SIGINT absent\n");
    return 0;
}
```
Output:
```
SIGINT present
```
## Example: Working with Multiple Signal Set Operations
```c
#include <stdio.h>
#include <signal.h>
int main()
{
    sigset_t set;
    sigemptyset(&set);
    sigaddset(&set, SIGINT);
    sigaddset(&set, SIGTERM);
    printf("Added SIGINT and SIGTERM\n");
    sigdelset(&set, SIGTERM);
    if(sigismember(&set, SIGTERM))
        printf("SIGTERM present\n");
    else
        printf("SIGTERM removed\n");
    return 0;
}
```
Output:
```
Added SIGINT and SIGTERM
SIGTERM removed
```
### Flow of Signal Set Operations
```
Create sigset_t
       ↓
sigemptyset() / sigfillset()
       ↓
sigaddset()
       ↓
sigdelset()
       ↓
sigismember()
       ↓
Use set with sigprocmask()
```

# Example of Blocking Signals
This example demonstrates how to **block a signal** using `sigprocmask()`. Here, `SIGINT` (generated by **Ctrl+C**) is blocked temporarily.
## Program
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    sigset_t set;
    /* Initialize empty signal set */
    sigemptyset(&set);
    /* Add SIGINT to signal set */
    sigaddset(&set, SIGINT);
    /* Block SIGINT */
    sigprocmask(SIG_BLOCK, &set, NULL);
    printf("SIGINT blocked for 10 seconds.\n");
    printf("Press Ctrl+C now...\n");
    sleep(10);
    printf("Unblocking SIGINT...\n");
    /* Unblock SIGINT */
    sigprocmask(SIG_UNBLOCK, &set, NULL);
    while (1)
    {
        printf("Program running...\n");
        sleep(1);
    }
    return 0;
}
```
## How the Program Works
### Step 1: Create a Signal Set
```c
sigset_t set;
```
A variable of type `sigset_t` is created to hold signals.
## Step 2: Initialize Empty Set
```c
sigemptyset(&set);
```
Initially, the set contains no signals.

Conceptually:
```
{ }
```
## Step 3: Add `SIGINT`
```c
sigaddset(&set, SIGINT);
```
`SIGINT` is added to the signal set.

Conceptually:
```
{SIGINT}
```
## Step 4: Block the Signal
```c
sigprocmask(SIG_BLOCK, &set, NULL);
```
This blocks `SIGINT`.

Now if the user presses **Ctrl+C**, the signal is **not immediately delivered**.

Instead, it becomes **pending**.
## Step 5: Wait for 10 Seconds
```c
sleep(10);
```
During this period:
* `SIGINT` is blocked
* Pressing `Ctrl+C` does not terminate the process
* Signal is stored as pending
## Step 6: Unblock Signal
```c
sigprocmask(SIG_UNBLOCK, &set, NULL);
```
When unblocked:
* Pending `SIGINT` is delivered immediately
* Default action executes (program terminates)
## Execution Example
Program starts:
```
SIGINT blocked for 10 seconds.
Press Ctrl+C now...
```
User presses **Ctrl+C** during sleep.

Nothing happens immediately.

After 10 seconds:
```
Unblocking SIGINT...
```
Then pending signal executes:
```
^C
```
Program terminates.
## Process Flow
```
Create signal set
        ↓
sigemptyset()
        ↓
sigaddset(SIGINT)
        ↓
sigprocmask(SIG_BLOCK)
        ↓
Ctrl+C pressed
        ↓
SIGINT becomes pending
        ↓
sleep(10)
        ↓
sigprocmask(SIG_UNBLOCK)
        ↓
Pending SIGINT delivered
        ↓
Program terminates
```

# Handling Signals with `sigaction()`
`sigaction()` is the **recommended and reliable way** to handle signals in Linux. It allows a process to specify what action should occur when a signal is received.

It is more powerful and portable than `signal()` because it provides:
* Better control over signal handling
* Reliable behavior across systems
* Support for signal masking and flags
## Syntax
```c
#include <signal.h>
int sigaction(int signum,
              const struct sigaction *act,
              struct sigaction *oldact);
```
### Parameters
* `signum` → signal number to handle (e.g., `SIGINT`)
* `act` → structure describing the new action
* `oldact` → stores previous signal action (optional)
### Return Value
* `0` → success
* `-1` → failure
## Structure of `sigaction`
```c
struct sigaction
{
    void (*sa_handler)(int);
    sigset_t sa_mask;
    int sa_flags;
};
```
### Members
#### 1. `sa_handler`
Specifies the function to execute when a signal arrives.

Example:
```c
void handler(int sig)
{
    printf("Signal %d received\n", sig);
}
```
## 2. `sa_mask`
A set of signals to block while the handler executes.

Example:
```c
sigemptyset(&act.sa_mask);
sigaddset(&act.sa_mask, SIGTERM);
```
Meaning:
* While handling one signal, `SIGTERM` will be temporarily blocked.
## 3. `sa_flags`
Controls signal handling behavior.

Common flags:
| Flag           | Meaning                                          |
| -------------- | ------------------------------------------------ |
| `SA_RESTART`   | Restart interrupted system calls                 |
| `SA_SIGINFO`   | Use extended signal handler                      |
| `SA_NODEFER`   | Do not automatically block signal during handler |
| `SA_RESETHAND` | Reset to default action after handling           |
## Basic Example: Handling `SIGINT`
This program catches **Ctrl+C (`SIGINT`)** using `sigaction()`.
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void sigint_handler(int sig)
{
    printf("\nSIGINT received (signal %d)\n", sig);
}
int main()
{
    struct sigaction act;
    /* Assign signal handler */
    act.sa_handler = sigint_handler;
    /* No extra signals blocked */
    sigemptyset(&act.sa_mask);
    /* No special flags */
    act.sa_flags = 0;
    /* Register handler */
    sigaction(SIGINT, &act, NULL);
    while (1)
    {
        printf("Running...\n");
        sleep(1);
    }
    return 0;
}
```
### Working
1. Program runs continuously.
2. User presses **Ctrl+C**.
3. OS sends `SIGINT`.
4. `sigaction()` invokes `sigint_handler()` instead of terminating.

Output:
```
Running...
Running...
^C
SIGINT received (signal 2)
Running...
Running...
```
The process continues running because the default action was replaced.
## Example with `SA_RESTART`
Sometimes a signal interrupts system calls such as `read()` or `sleep()`.

Using `SA_RESTART` automatically restarts interrupted calls.
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("Signal handled\n");
}
int main()
{
    struct sigaction act;
    act.sa_handler = handler;
    sigemptyset(&act.sa_mask);
    act.sa_flags = SA_RESTART;
    sigaction(SIGINT, &act, NULL);
    while (1)
    {
        sleep(5);
        printf("Still running\n");
    }
    return 0;
}
```
Without `SA_RESTART`, some system calls may fail with `EINTR`.
## Example: Blocking Signals During Handler
You can block signals while a handler executes.
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("Handling SIGINT...\n");
    sleep(5);
    printf("Done\n");
}
int main()
{
    struct sigaction act;
    act.sa_handler = handler;
    sigemptyset(&act.sa_mask);
    /* Block SIGTERM while handler runs */
    sigaddset(&act.sa_mask, SIGTERM);
    act.sa_flags = 0;
    sigaction(SIGINT, &act, NULL);
    while (1);
    return 0;
}
```
Meaning:
* While `SIGINT` handler runs,
* `SIGTERM` is temporarily blocked.
## Flow of `sigaction()`
```
Program starts
       ↓
Create sigaction structure
       ↓
Assign handler (sa_handler)
       ↓
Set signal mask (sa_mask)
       ↓
Set flags (sa_flags)
       ↓
Register using sigaction()
       ↓
Signal occurs
       ↓
Handler function executes
       ↓
Program continues or exits
```
### `signal()` vs `sigaction()`
| Feature        |    `signal()` | `sigaction()` |
| -------------- | ------------: | ------------: |
| Reliability    | Less reliable | More reliable |
| Signal masking |            No |           Yes |
| Flags support  |            No |           Yes |
| Recommended    |            No |           Yes |

# `sigaction()` Example
The following example demonstrates how to handle a signal using `sigaction()`. Here, the program catches `SIGINT` (generated by **Ctrl+C**) and executes a custom signal handler instead of terminating.
## Program
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void signal_handler(int sig)
{
    printf("\nSignal %d received\n", sig);
}
int main()
{
    struct sigaction act;
    /* Assign signal handler */
    act.sa_handler = signal_handler;
    /* Initialize empty signal mask */
    sigemptyset(&act.sa_mask);
    /* No special flags */
    act.sa_flags = 0;
    /* Register SIGINT handler */
    sigaction(SIGINT, &act, NULL);
    while (1)
    {
        printf("Program running...\n");
        sleep(1);
    }
    return 0;
}
```
## Explanation of the Program
### Step 1: Define Signal Handler
```c 
void signal_handler(int sig)
{
    printf("\nSignal %d received\n", sig);
}
```
This function executes whenever `SIGINT` is received.
* `sig` contains the signal number.
* For `SIGINT`, the value is usually `2`.

If the user presses **Ctrl+C**, this handler runs.
## Step 2: Create `sigaction` Structure
```c 
struct sigaction act;
```
This structure stores signal handling information.

It contains:
* `sa_handler` → handler function
* `sa_mask` → blocked signals during handling
* `sa_flags` → behavior options
## Step 3: Set Signal Handler
```c 
act.sa_handler = signal_handler;
```
Associates the signal with the custom handler.

Meaning:
```
SIGINT → signal_handler()
```
## Step 4: Initialize Signal Mask
```c 
sigemptyset(&act.sa_mask);
```
No additional signals are blocked during execution of handler.

Conceptually:
```
{ }
```
## Step 5: Set Flags
```c 
act.sa_flags = 0;
```
No special behavior is enabled.
## Step 6: Register the Signal Action
```c 
sigaction(SIGINT, &act, NULL);
```
This tells Linux:
> “When `SIGINT` occurs, call `signal_handler()`.”
## Step 7: Infinite Loop
```c 
while (1)
{
    printf("Program running...\n");
    sleep(1);
}
```
Program continues running forever.

When **Ctrl+C** is pressed:
* Default termination is prevented
* Custom handler executes
## Execution Example
Program starts:
```
Program running...
Program running...
Program running...
```
User presses **Ctrl+C**
```
^C
Signal 2 received
```
Program continues:
```
Program running...
Program running...
```
## Program Flow
```
Program starts
        ↓
Create sigaction structure
        ↓
Assign handler function
        ↓
Initialize signal mask
        ↓
Register with sigaction()
        ↓
Program executes
        ↓
User presses Ctrl+C
        ↓
SIGINT generated
        ↓
signal_handler() executes
        ↓
Program continues
```
### Why Use `sigaction()` Instead of `signal()`?
* More reliable
* Supports signal masking (`sa_mask`)
* Supports flags (`sa_flags`)
* Recommended for Linux system programming

In short: **`sigaction()` lets a program intercept signals and execute a custom function when the signal occurs.**

# Handling Signals with `signal()`
`signal()` is a function used to **catch and handle signals** in a process. It allows a program to specify a **signal handler function** that executes whenever a particular signal is received.

It is a simple way to handle signals, although `sigaction()` is generally preferred for robust Linux programming.
## Syntax
```c 
#include <signal.h>
void (*signal(int signum,
              void (*handler)(int)))(int);
```
### Simplified Form
```c 
signal(signal_number, handler_function);
```
### Parameters
* `signum` → signal to handle (e.g., `SIGINT`)
* `handler` → function to execute when signal occurs

The handler can be:
* A custom function
* `SIG_IGN` → ignore signal
* `SIG_DFL` → default behavior
## How `signal()` Works
1. Register a signal handler using `signal()`
2. Signal occurs
3. OS interrupts normal execution
4. Registered handler executes
5. Program resumes (or terminates depending on logic)

Flow:
```
Program starts
       ↓
Register handler using signal()
       ↓
Signal occurs
       ↓
Handler function executes
       ↓
Program resumes
```
## Example 1: Handling `SIGINT` (`Ctrl+C`)
This program catches `SIGINT` and prints a message instead of terminating.
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("\nSignal %d received\n", sig);
}
int main()
{
    signal(SIGINT, handler);
    while (1)
    {
        printf("Running...\n");
        sleep(1);
    }
    return 0;
}
```
### Working
* Program runs continuously.
* User presses **Ctrl+C**.
* Terminal sends `SIGINT`.
* Handler function executes.

Output:
```
Running...
Running...
^C
Signal 2 received
Running...
```
Instead of terminating, the program continues running.
## Example 2: Ignoring a Signal
Signals can also be ignored.
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    signal(SIGINT, SIG_IGN);
    while (1)
    {
        printf("Ctrl+C ignored\n");
        sleep(1);
    }
    return 0;
}
```
### Working
* Pressing **Ctrl+C** does nothing.
* `SIGINT` is ignored.

Output:
```
Ctrl+C ignored
Ctrl+C ignored
```
## Example 3: Restore Default Behavior
You can restore default handling.
```c 
signal(SIGINT, SIG_DFL);
```
Meaning:
* `SIGINT` behaves normally again.
* `Ctrl+C` terminates the process.
## Signal Handler Function
A signal handler must have this form:
```c 
void handler(int sig)
{
    /* code */
}
```
Example:
```c 
void handler(int sig)
{
    printf("Caught signal %d\n", sig);
}
```
The parameter `sig` contains the signal number received.
## Return Value of `signal()`
Returns:
* Previous signal handler → success
* `SIG_ERR` → failure

Example:
```c 
if (signal(SIGINT, handler) == SIG_ERR)
{
    printf("Error installing signal handler\n");
}
```
## Common Signals Used with `signal()`
| Signal    | Meaning               |
| --------- | --------------------- |
| `SIGINT`  | Interrupt (`Ctrl+C`)  |
| `SIGTERM` | Termination request   |
| `SIGALRM` | Alarm timer           |
| `SIGUSR1` | User-defined signal 1 |
| `SIGUSR2` | User-defined signal 2 |
## `signal()` vs `sigaction()`
| Feature             |     `signal()` |            `sigaction()` |
| ------------------- | -------------: | -----------------------: |
| Simplicity          |           Easy |            More detailed |
| Reliability         |          Lower |                   Higher |
| Signal mask support |             No |                      Yes |
| Flags support       |             No |                      Yes |
| Recommended         | Basic learning | Production/Linux systems |

# Sending Signals
Signals can be sent from one process to another (or to itself) to notify an event or control execution. In Linux, signals are commonly sent using functions like `kill()`, `raise()`, and `alarm()`.

Typical uses:
* Terminate a process
* Pause or resume execution
* Notify another process
* Trigger custom signal handlers
## 1. Sending Signals using `kill()`
Despite its name, `kill()` is used to **send any signal**, not just terminate a process.
### Syntax
```c 
#include <signal.h>
int kill(pid_t pid, int sig);
```
### Parameters
* `pid` → target process ID
* `sig` → signal to send
### Return Value
* `0` → success
* `-1` → failure
### PID Meaning
| PID Value | Meaning                                     |
| --------- | ------------------------------------------- |
| `> 0`     | Send signal to specific process             |
| `0`       | Send to all processes in same process group |
| `< -1`    | Send to process group                       |
| `-1`      | Send to all permitted processes             |
## Example: Send `SIGTERM` to a Process
```c 
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
int main()
{
    pid_t pid;
    printf("Enter PID: ");
    scanf("%d", &pid);
    kill(pid, SIGTERM);
    printf("SIGTERM sent\n");
    return 0;
}
```
### Working
* User enters a process ID.
* `kill()` sends `SIGTERM`.
* Target process terminates (unless handled/ignored).
## Example: Send `SIGKILL`
```c
kill(pid, SIGKILL);
```
Meaning:
* Forcefully terminate process.
* Cannot be caught or ignored.
## 2. Sending Signals to Self using `raise()`
`raise()` sends a signal to the **calling process itself**.
### Syntax
```c 
#include <signal.h>
int raise(int sig);
```
### Example
```c 
#include <stdio.h>
#include <signal.h>
void handler(int sig)
{
    printf("Signal %d received\n", sig);
}
int main()
{
    signal(SIGINT, handler);
    printf("Sending SIGINT to self...\n");
    raise(SIGINT);
    return 0;
}
```
Output:
```
Sending SIGINT to self...
Signal 2 received
```
### Working
* `raise(SIGINT)` sends `SIGINT` to current process.
* Registered handler executes.

Equivalent idea:
```
Current process → send signal to itself
```
## 3. Sending Timer Signals using `alarm()`
`alarm()` sends `SIGALRM` after a specified number of seconds.
### Syntax
```c 
#include <unistd.h>
unsigned int alarm(unsigned int seconds);
```
### Example
```c
#include <stdio.h>
#include <signal.h>
#include <unistd.h>
void handler(int sig)
{
    printf("Alarm signal received!\n");
}
int main()
{
    signal(SIGALRM, handler);
    alarm(5);
    printf("Waiting for alarm...\n");
    pause();
    return 0;
}
```
Output (after 5 seconds):
```
Waiting for alarm...
Alarm signal received!
```
### Working
* `alarm(5)` sets timer.
* After 5 seconds, kernel sends `SIGALRM`.
* Handler executes.
## Example: Parent Sending Signal to Child
```c 
#include <stdio.h>
#include <unistd.h>
#include <signal.h>
void handler(int sig)
{
    printf("Child received signal\n");
}
int main()
{
    pid_t pid = fork();
    if (pid == 0)
    {
        signal(SIGUSR1, handler);
        while (1);
    }
    else
    {
        sleep(2);
        kill(pid, SIGUSR1);
    }
    return 0;
}
```
### Working
1. Parent creates child.
2. Child waits.
3. Parent sends `SIGUSR1`.
4. Child handler executes.

Output:
```
Child received signal
```
## Flow of Sending Signals
```
Signal generated
       ↓
kill() / raise() / alarm()
       ↓
Kernel delivers signal
       ↓
Target process receives signal
       ↓
Handler executes or default action occurs
```
### Common Signal Sending Functions
| Function  | Purpose                        |
| --------- | ------------------------------ |
| `kill()`  | Send signal to another process |
| `raise()` | Send signal to self            |
| `alarm()` | Send timed `SIGALRM`           |

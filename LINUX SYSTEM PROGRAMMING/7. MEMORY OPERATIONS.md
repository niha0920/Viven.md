# Allocating/Freeing Memory
Memory allocation means reserving memory during program execution (**runtime**) instead of compile time. In C, this is called **dynamic memory allocation** and is done using functions from the standard library (`stdlib.h`).
## Why Dynamic Memory Allocation?
Sometimes the amount of memory needed is not known in advance.

Example:
* Reading a file of unknown size
* Creating dynamic data structures like linked lists, trees, queues
* Allocating memory based on user input

Instead of fixed-size arrays:
```c
int arr[100];
```
You can allocate memory dynamically:
```c
int *arr;
arr = (int *)malloc(100 * sizeof(int));
```
## Memory Allocation Functions
The main functions are:
1. `malloc()`
2. `calloc()`
3. `realloc()`
4. `free()`
## 1. `malloc()` – Memory Allocation
`malloc()` allocates a block of memory.
### Syntax
```c
ptr = (type *)malloc(size);
```
* `size` → number of bytes to allocate
* Returns pointer to allocated memory
* Returns `NULL` if allocation fails
* Memory contains **garbage values** (not initialized)
### Example
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)malloc(5 * sizeof(int));
    if(ptr == NULL)
    {
        printf("Memory allocation failed\n");
        return 1;
    }
    for(int i = 0; i < 5; i++)
    {
        ptr[i] = i + 1;
    }
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
1 2 3 4 5
```
### Working
* `malloc(5 * sizeof(int))` allocates memory for 5 integers
* Pointer stores starting address
* Memory is freed using `free()`
## 2. `calloc()` – Contiguous Allocation
`calloc()` allocates memory for multiple elements and initializes them to **zero**.
### Syntax
```c
ptr = (type *)calloc(number_of_elements, size_of_each_element);
```
### Example
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)calloc(5, sizeof(int));
    if(ptr == NULL)
    {
        printf("Allocation failed\n");
        return 1;
    }
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
0 0 0 0 0
```
### Difference between `malloc()` and `calloc()`
| `malloc()`          |                `calloc()` |
| ------------------- | ------------------------: |
| Allocates one block | Allocates multiple blocks |
| Garbage values      |       Initializes to zero |
| Faster              |           Slightly slower |
## 3. `realloc()` – Resize Memory
Used to resize previously allocated memory.
### Syntax
```c
ptr = realloc(ptr, new_size);
```
Can:
* Increase memory size
* Decrease memory size
### Example
```c
#include <stdio.h>
#include <stdlib.h>
int main()
{
    int *ptr;
    ptr = (int *)malloc(3 * sizeof(int));
    if(ptr == NULL)
    {
        return 1;
    }
    for(int i = 0; i < 3; i++)
    {
        ptr[i] = i + 1;
    }
    ptr = realloc(ptr, 5 * sizeof(int));
    for(int i = 3; i < 5; i++)
    {
        ptr[i] = i + 1;
    }
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", ptr[i]);
    }
    free(ptr);
    return 0;
}
```
### Output
```
1 2 3 4 5
```
### Important Note
Safer way to use `realloc()`:
```c
int *temp = realloc(ptr, new_size);
if(temp != NULL)
{
    ptr = temp;
}
```
Why?

If `realloc()` fails, it returns `NULL`. Direct assignment can lose the original pointer and cause a memory leak.
## 4. `free()` – Free Allocated Memory
`free()` releases dynamically allocated memory back to the system.
### Syntax
```c
free(ptr);
```
### Example
```c
int *ptr = malloc(10 * sizeof(int));
free(ptr);
ptr = NULL;
```
Setting pointer to `NULL` after freeing helps avoid **dangling pointers**.
## Memory Layout of a C Program
A program memory is divided into sections:
```
-------------------
Stack
-------------------
Heap   ← malloc/calloc/realloc
-------------------
Data Segment
-------------------
Code Segment
-------------------
```
Dynamic memory allocation happens in the **Heap**.
## Common Problems in Dynamic Memory
### 1. Memory Leak
Occurs when allocated memory is never freed.
```c
int *ptr = malloc(sizeof(int));
ptr = NULL;   // lost reference
```
Problem: memory cannot be accessed or freed.

Correct:
```c
free(ptr);
ptr = NULL;
```
## 2. Dangling Pointer
Using memory after freeing it.
```c
int *ptr = malloc(sizeof(int));
free(ptr);
*ptr = 10;   // dangerous
```
Fix:
```c
free(ptr);
ptr = NULL;
```
## 3. Double Free
Freeing same memory twice.
```c
free(ptr);
free(ptr);   // error
```
Fix:
```c
free(ptr);
ptr = NULL;
```

# Memory Alignment
**Memory alignment** means storing data at memory addresses that are suitable for the CPU architecture so memory access becomes **faster and efficient**.

The processor usually reads data in chunks (word boundaries). If data is placed at properly aligned addresses, access is faster.

Example:
* `char` → 1 byte alignment
* `short` → 2 byte alignment
* `int` → 4 byte alignment
* `double` → 8 byte alignment (depends on system)
## Why Alignment is Needed?
CPU accesses memory efficiently when data is stored at addresses divisible by its size/alignment.

Example:

An `int` (4 bytes) is better stored at addresses:
```
1000 ✓
1004 ✓
1008 ✓
1012 ✓
```
Bad alignment:
```
1001 ✗
1002 ✗
1003 ✗
```
Misaligned memory access may:
* Slow performance
* Need multiple memory operations
* Cause hardware exceptions on some systems
## Example of Alignment
Consider:
```c
struct Example
{
    char c;
    int x;
};
```
Memory layout:
```
Address      Data
1000         c (1 byte)
1001-1003    padding
1004-1007    x (4 bytes)
```
Although:
```
char = 1 byte
int = 4 bytes
Total = 5 bytes
```
Actual size:
```c
sizeof(struct Example) = 8
```
Why?

Compiler inserts **padding bytes** to align `int` properly.
## Structure Padding
Padding = extra unused bytes inserted by compiler for alignment.

Example:
```c
#include <stdio.h>
struct A
{
    char a;
    int b;
    char c;
};
int main()
{
    printf("%zu\n", sizeof(struct A));
    return 0;
}
```
Expected size:
```
1 + 4 + 1 = 6 bytes
```
Actual layout:
```
a      → 1 byte
padding→ 3 bytes
b      → 4 bytes
c      → 1 byte
padding→ 3 bytes
```
Total:
```
12 bytes
```
Why last padding?

Because structure size is generally aligned to the **largest member alignment** (here `int = 4`).
## Better Structure Design (Reduce Padding)
Bad ordering:
```c
struct A
{
    char a;
    int b;
    char c;
};
```
Better ordering:
```c
struct A
{
    int b;
    char a;
    char c;
};
```
Now layout:
```
b → 4 bytes
a → 1 byte
c → 1 byte
padding → 2 bytes
```
Total:
```c
sizeof(struct A) = 8
```
Less wasted memory.
## Example Program
```c
#include <stdio.h>
struct A
{
    char a;
    int b;
    char c;
};
struct B
{
    int b;
    char a;
    char c;
};
int main()
{
    printf("Size of A = %zu\n", sizeof(struct A));
    printf("Size of B = %zu\n", sizeof(struct B));
    return 0;
}
```
Output:
```
Size of A = 12
Size of B = 8
```
## Packed Structures
Sometimes we remove padding.

Using compiler directive:
```c
#pragma pack(1)
struct A
{
    char a;
    int b;
    char c;
};
```
Or GCC attribute:
```c
struct A
{
    char a;
    int b;
    char c;
} __attribute__((packed));
```
Now:
```
Size = 6 bytes
```
But:
- Packed structures may reduce performance because CPU accesses misaligned memory.

Use only when necessary:
* Network packets
* Hardware registers
* Binary file formats
## Alignment Rule in Structures
1. Each member aligned according to its type.
2. Compiler adds padding between members if required.
3. Structure size becomes multiple of largest alignment requirement.
## Real Example
```c
struct Test
{
    char c;
    double d;
    int i;
};
```
Memory layout (typical 64-bit system):
```
c        → 1 byte
padding  → 7 bytes
d        → 8 bytes
i        → 4 bytes
padding  → 4 bytes
```
Total:
```
24 bytes
```
## Checking Alignment
C provides alignment operator:
```c
#include <stdio.h>
#include <stdalign.h>
int main()
{
    printf("%zu\n", alignof(int));
    printf("%zu\n", alignof(double));
}
```
Possible output:
```
4
8
```
## Key Terms
* **Alignment** → placing data at proper memory boundary
* **Padding** → extra bytes inserted
* **Packed structure** → structure without padding
* **Misaligned access** → slower access

# Locked Memory
**Locked memory** means memory pages are prevented from being swapped out to disk by the operating system.

Normally, Linux can move memory pages between RAM and swap space to manage memory efficiently. With **memory locking**, selected memory remains in RAM.

This is useful when:
* Security-sensitive data must not be written to disk
* Real-time applications need predictable timing
* Low-latency systems cannot tolerate page faults

Examples:
* Passwords / cryptographic keys
* Embedded and real-time systems
* High-performance applications
## Why Lock Memory?
Normally:
```
Program Memory (RAM)
       ↓
Linux may swap pages to disk
```
With locked memory:
```
Program Memory (RAM)
       ↓
Cannot be swapped
```
Benefits:
1. **Avoid swapping sensitive information**
2. **Reduce latency** caused by page faults
3. **Predictable execution** in real-time systems
## Linux Functions for Locked Memory
Linux provides:
1. `mlock()`
2. `munlock()`
3. `mlockall()`
4. `munlockall()`

These are declared in:
```c
#include <sys/mman.h>
```
## 1. `mlock()` – Lock Memory Region
Locks a specific memory region in RAM.
### Syntax
```c
int mlock(const void *addr, size_t len);
```
Parameters:
* `addr` → starting address
* `len` → number of bytes

Return:
* `0` → success
* `-1` → failure (`errno` set)
### Example
```c
#include <stdio.h>
#include <stdlib.h>
#include <sys/mman.h>
int main()
{
    int *ptr = malloc(sizeof(int) * 10);
    if(ptr == NULL)
    {
        return 1;
    }
    if(mlock(ptr, sizeof(int) * 10) == -1)
    {
        perror("mlock");
        free(ptr);
        return 1;
    }
    printf("Memory locked\n");
    munlock(ptr, sizeof(int) * 10);
    free(ptr);
    return 0;
}
```
## 2. `munlock()` – Unlock Memory
Unlocks previously locked memory.
### Syntax
```c
int munlock(const void *addr, size_t len);
```
Example:
```c
munlock(ptr, sizeof(int) * 10);
```
After unlocking:
* Memory becomes eligible for swapping again.
## 3. `mlockall()` – Lock Entire Process Memory
Locks all current and/or future pages of a process.
### Syntax
```c
int mlockall(int flags);
```
Flags:
```c
MCL_CURRENT
```
Lock all currently mapped pages.
```c
MCL_FUTURE
```
Lock future allocations too.

Example:
```c
mlockall(MCL_CURRENT | MCL_FUTURE);
```
Meaning:
* Current memory locked
* Future allocations also locked
### Example Program
```c
#include <stdio.h>
#include <sys/mman.h>
int main()
{
    if(mlockall(MCL_CURRENT | MCL_FUTURE) == -1)
    {
        perror("mlockall");
        return 1;
    }
    printf("Process memory locked\n");
    munlockall();
    return 0;
}
```
## 4. `munlockall()` – Unlock Everything
Unlocks all memory pages of process.
### Syntax
```c
int munlockall(void);
```
Example:
```c
munlockall();
```
## How Locked Memory Works
```
Without mlock():
RAM  <----> SWAP
 pages can move
With mlock():
RAM only
(no swapping)
```
## Common Errors
### Permission Issue
Sometimes:
```
mlock: Cannot allocate memory
```
Reason:

Linux limits lockable memory.

Check limit:
```
ulimit -l
```
Increase limit:
```
ulimit -l unlimited
```
Or configure system limits.
## Important Notes
### 1. Memory Locking is Limited
Linux limits amount of lockable memory to prevent abuse.

Check:
```
ulimit -l
```
## 2. Locked Memory Uses RAM
Too much locking can reduce system performance because memory cannot be swapped.
## 3. Real-Time Systems Use It
Real-time programs often use:
```c
mlockall(MCL_CURRENT | MCL_FUTURE);
```
to avoid page-fault delays.
## Security Example
Sensitive data:
```c
char password[100];
```
If swapped to disk:
* attacker may recover it later.

Lock memory:
```c
mlock(password, sizeof(password));
```
so password remains in RAM.

# Memory Copy/Initialization
Memory copy and initialization are common low-level operations in C used to manipulate blocks of memory.

These functions are provided in:
```c
#include <string.h>
```
Common functions:
1. `memcpy()` → copy memory
2. `memmove()` → safe memory copy for overlapping memory
3. `memset()` → initialize memory
4. `memcmp()` → compare memory blocks
## 1. `memcpy()` – Copy Memory
`memcpy()` copies bytes from one memory location to another.
### Syntax
```c
void *memcpy(void *dest, const void *src, size_t n);
```
Parameters:
* `dest` → destination memory
* `src` → source memory
* `n` → number of bytes to copy

Returns:
* Pointer to destination
### Example
```c
#include <stdio.h>
#include <string.h>
int main()
{
    int src[5] = {1, 2, 3, 4, 5};
    int dest[5];
    memcpy(dest, src, sizeof(src));
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", dest[i]);
    }
    return 0;
}
```
Output:
```
1 2 3 4 5
```
### How it Works
```
Before copy:
src  → [1 2 3 4 5]
dest → [ ? ? ? ? ? ]
After memcpy():
dest → [1 2 3 4 5]
```
- `memcpy()` does **not** handle overlapping memory safely.
## 2. `memmove()` – Safe Memory Copy
`memmove()` works like `memcpy()` but safely handles overlapping memory regions.
### Syntax
```c
void *memmove(void *dest, const void *src, size_t n);
```
### Example
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char str[] = "HELLO";
    memmove(str + 2, str, 3);
    printf("%s\n", str);
    return 0;
}
```
Output:
```
HEHEL
```
Why `memmove()`?

Because source and destination overlap.

Bad with `memcpy()`:
```c
memcpy(str + 2, str, 3);
```
Result → undefined behavior.
## `memcpy()` vs `memmove()`
| `memcpy()`                    |     `memmove()` |
| ----------------------------- | --------------: |
| Faster                        | Slightly slower |
| No overlap handling           |    Overlap-safe |
| Undefined behavior on overlap |            Safe |
## 3. `memset()` – Memory Initialization
`memset()` fills memory with a byte value.
### Syntax
```c
void *memset(void *ptr, int value, size_t n);
```
Parameters:
* `ptr` → memory block
* `value` → byte value to set
* `n` → bytes to fill
### Example: Initialize to Zero
```c
#include <stdio.h>
#include <string.h>
int main()
{
    int arr[5];
    memset(arr, 0, sizeof(arr));
    for(int i = 0; i < 5; i++)
    {
        printf("%d ", arr[i]);
    }
    return 0;
}
```
Output:
```
0 0 0 0 0
```
## Example: Fill Character Array
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char str[10];
    memset(str, 'A', 5);
    str[5] = '\0';
    printf("%s\n", str);
    return 0;
}
```
Output:
```
AAAAA
```
## Important Note on `memset()`
`memset()` works **byte by byte**.

Example:
```c
int arr[5];
memset(arr, 1, sizeof(arr));
```
This **does not** make all elements `1`.

Reason:

`memset()` fills bytes:
```
0x01 0x01 0x01 0x01
```
An integer becomes:
```
16843009
```
Instead use loop:
```c
for(int i = 0; i < 5; i++)
{
    arr[i] = 1;
}
```
## 4. `memcmp()` – Compare Memory
Compares memory blocks byte by byte.
### Syntax
```c
int memcmp(const void *ptr1, const void *ptr2, size_t n);
```
Returns:
* `0` → equal
* `<0` → first block smaller
* `>0` → first block larger
### Example
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char s1[] = "ABC";
    char s2[] = "ABC";
    if(memcmp(s1, s2, 3) == 0)
    {
        printf("Equal\n");
    }
    return 0;
}
```
Output:
```
Equal
```
## Memory Representation
```
Array:
[10][20][30][40]
memcpy()
copy bytes
memset()
fill bytes
memcmp()
compare bytes
```
## Real-Time / Embedded Usage
Common usage:
Initialize structure:
```c
struct Data d;
memset(&d, 0, sizeof(d));
```
Copy packet:
```c
memcpy(dest_buffer, src_buffer, length);
```
Move overlapping buffer:
```c
memmove(buffer + 1, buffer, size);
```

# Memory Comparison/Search
Memory comparison and search functions are used to **compare, locate, or scan raw memory blocks byte-by-byte**.

These functions are available in:
```c
#include <string.h>
```
Common functions:
1. `memcmp()` → compare memory blocks
2. `memchr()` → search a byte in memory
3. `memrchr()` → reverse search (GNU extension)
## 1. `memcmp()` – Compare Memory Blocks
`memcmp()` compares two memory regions byte by byte.
### Syntax
```c
int memcmp(const void *ptr1, const void *ptr2, size_t n);
```
Parameters:
* `ptr1` → first memory block
* `ptr2` → second memory block
* `n` → number of bytes to compare

Returns:
* `0` → memory blocks are equal
* `< 0` → first differing byte in `ptr1` is smaller
* `> 0` → first differing byte in `ptr1` is larger
### Example
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char a[] = "HELLO";
    char b[] = "HELLO";
    if(memcmp(a, b, 5) == 0)
    {
        printf("Equal\n");
    }
    return 0;
}
```
Output:
```
Equal
```
## Example: Different Memory
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char a[] = "HELLO";
    char b[] = "HELMO";
    int result = memcmp(a, b, 5);
    printf("%d\n", result);
    return 0;
}
```
Explanation:
```
HELLO
HELMO
   ^
```
Comparison stops at first mismatch.
## `memcmp()` vs `strcmp()`
| `memcmp()`            |       `strcmp()` |
| --------------------- | ---------------: |
| Compares raw memory   | Compares strings |
| Uses byte count (`n`) |  Stops at `'\0'` |
| Works for any data    |     Only strings |
Example:
```c
int arr1[] = {1,2,3};
int arr2[] = {1,2,3};
memcmp(arr1, arr2, sizeof(arr1));
```
Useful for structures, arrays, buffers.
## 2. `memchr()` – Search Byte in Memory
Searches for a byte value in memory.
### Syntax
```c
void *memchr(const void *ptr, int value, size_t n);
```
Parameters:
* `ptr` → memory block
* `value` → byte to search
* `n` → bytes to scan

Returns:
* Pointer to first occurrence
* `NULL` if not found
### Example
```c
#include <stdio.h>
#include <string.h>
int main()
{
    char str[] = "LINUX";
    char *p = memchr(str, 'N', strlen(str));
    if(p != NULL)
    {
        printf("Found: %c\n", *p);
    }
    return 0;
}
```
Output:
```
Found: N
```
## Memory Representation
```
L  I  N  U  X
      ↑
   memchr()
```
## Example Using Integers
Since memory is byte-oriented:
```c
#include <stdio.h>
#include <string.h>
int main()
{
    unsigned char arr[] = {10,20,30,40};
    unsigned char *p =
        memchr(arr, 30, sizeof(arr));
    if(p)
    {
        printf("Found %d\n", *p);
    }
    return 0;
}
```
Output:
```
Found 30
```
## 3. `memrchr()` – Reverse Memory Search (GNU)
Searches from end to beginning.
### Syntax
```c
void *memrchr(const void *ptr,
              int value,
              size_t n);
```
Returns:
* Last occurrence
* `NULL` if not found
### Example
```c
#define _GNU_SOURCE
#include <stdio.h>
#include <string.h>
int main()
{
    char str[] = "BANANA";
    char *p = memrchr(str, 'A', strlen(str));
    if(p)
    {
        printf("Last A at index %ld\n",
               p - str);
    }
    return 0;
}
```
Output:
```
Last A at index 5
```
## Difference Between Search Functions
| Function    |               Purpose |
| ----------- | --------------------: |
| `memchr()`  | Find first occurrence |
| `memrchr()` |  Find last occurrence |
| `memcmp()`  |        Compare memory |
## Important Notes
### 1. `memcmp()` compares bytes
Example:
```c
int a = 1000;
int b = 1000;
memcmp(&a, &b, sizeof(int));
```
Compares raw memory representation.
## 2. `memchr()` searches bytes
Searching in an `int` array can be tricky because:
```
1000 = multiple bytes
```
It searches byte values, not integer values.
## 3. Structure Comparison Warning
Avoid:
```c
struct Data a, b;
memcmp(&a, &b, sizeof(a));
```
Why?

Because padding bytes may differ due to **memory alignment**, even when members are logically equal.

Better:
```c
a.id == b.id &&
a.age == b.age
```
## Practical Use Cases
* Packet comparison
* Binary protocol parsing
* Searching raw buffers
* File/binary data matching
* Embedded systems memory scanning

# Introducing Threaded Programming
## What is a Thread?
A **thread** is the smallest unit of execution inside a process. A process may contain one or more threads that execute concurrently.
* A **process** has its own memory space and resources.
* A **thread** shares the same memory and resources of the process with other threads.

For example, in a web browser:
* One thread handles the user interface.
* Another thread downloads files.
* Another thread renders web pages.

Thus, multiple tasks can run simultaneously within the same process.
### Why Use Threaded Programming?
Threaded programming is used to improve:
1. **Concurrency** – Multiple tasks execute at the same time.
2. **Responsiveness** – Programs remain responsive even during lengthy operations.
3. **Performance** – Multiple CPU cores can execute threads in parallel.
4. **Resource Sharing** – Threads share process memory and data efficiently.

Example:

A media player can:
* Play audio/video
* Read user input
* Load subtitles

all at the same time using separate threads.
### Process vs Thread
| Process                       | Thread                                   |
| ----------------------------- | ---------------------------------------- |
| Independent execution unit    | Smallest execution unit inside a process |
| Has separate memory           | Shares process memory                    |
| Creation is expensive         | Creation is lightweight                  |
| Communication is slower (IPC) | Communication is faster (shared memory)  |
### Types of Thread Models
#### 1. Single-Threaded Program
Only one thread executes.

Example:
* A calculator application performing one task at a time.

Execution flow:
```
Start → Execute Task → End
```
#### 2. Multi-Threaded Program
Multiple threads execute concurrently.

Example:
* Web browser
* Chat application
* Download manager

Execution flow:
```
Main Process
   ├── Thread 1
   ├── Thread 2
   └── Thread 3
```
### Advantages of Threaded Programming
1. **Improved Responsiveness**
   * User interface remains active while background work continues.
2. **Better CPU Utilization**
   * Multi-core processors execute threads simultaneously.
3. **Efficient Communication**
   * Threads share memory, making data exchange faster.
4. **Economy**
   * Creating threads requires fewer resources than processes.
5. **Scalability**
   * Applications scale well on multicore systems.
### Challenges in Threaded Programming
1. **Synchronization Problems**
   * Multiple threads accessing shared data can cause inconsistency.
2. **Race Conditions**
   * Unexpected results when threads access shared resources simultaneously.
3. **Deadlocks**
   * Threads wait indefinitely for each other.
4. **Debugging Difficulty**
   * Concurrent execution makes bugs harder to detect.
### Thread Lifecycle
A thread generally moves through these states:
```
New → Runnable → Running → Waiting/Blocked → Terminated
```
### Threading in Linux
In Linux, threads are commonly created using the POSIX Threads library:

POSIX Threads (**Pthreads**)

Important functions:
```c
pthread_create()   // create a thread
pthread_join()     // wait for thread completion
pthread_exit()     // terminate thread
pthread_self()     // get thread ID
```
### Simple Example of Threaded Programming
```c
#include <stdio.h>
#include <pthread.h>
void *message(void *arg)
{
    printf("Hello from Thread!\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, message, NULL);
    pthread_join(tid, NULL);
    printf("Hello from Main Thread!\n");
    return 0;
}
```
**Output:**
```
Hello from Thread!
Hello from Main Thread!
```
### Explanation
* `pthread_create()` creates a new thread.
* Both main thread and child thread run concurrently.
* `pthread_join()` waits for thread completion.
* Shared process resources are used by both threads.

# Building Threaded Programs
In Linux, threaded programs are generally built using the **POSIX Threads (Pthreads)** library.

POSIX Threads provides APIs to create, manage, synchronize, and terminate threads.
## Steps to Build a Threaded Program
The basic steps are:
1. Include the pthread header file
2. Create thread function(s)
3. Create thread(s) using `pthread_create()`
4. Synchronize using `pthread_join()` (optional but common)
5. Compile with pthread support
## 1. Include Required Header File
To use threads, include:
```c
#include <pthread.h>
```
This header contains thread-related function declarations.

Example:
```c
#include <stdio.h>
#include <pthread.h>
```
## 2. Create a Thread Function
A thread executes a function called a **thread routine**.

Syntax:
```c
void *thread_function(void *arg)
{
    /* thread code */
    return NULL;
}
```
Explanation:
* Return type: `void *`
* Argument: `void *arg`
* Allows passing any type of data to a thread.

Example:
```c
void *display(void *arg)
{
    printf("Thread Executing\n");
    return NULL;
}
```
## 3. Create a Thread using `pthread_create()`
Syntax:
```c
pthread_create(thread, attr, function, arg);
```
Prototype:
```c
int pthread_create(
        pthread_t *thread,
        const pthread_attr_t *attr,
        void *(*start_routine)(void *),
        void *arg
);
```
Parameters:
1. `thread`
   * Stores thread ID.
2. `attr`
   * Thread attributes.
   * Usually `NULL` (default settings).
3. `start_routine`
   * Function executed by thread.
4. `arg`
   * Data passed to thread.

Example:
```c
pthread_t tid;
pthread_create(&tid, NULL, display, NULL);
```
This creates one thread executing `display()`.
## 4. Wait for Thread Completion using `pthread_join()`
The main thread often waits for child thread completion.

Syntax:
```c
pthread_join(thread, retval);
```
Example:
```c
pthread_join(tid, NULL);
```
Meaning:
* Wait until thread `tid` finishes execution.

Without `pthread_join()`, the main program may terminate before the thread completes.
## 5. Complete Program Example
```c
#include <stdio.h>
#include <pthread.h>
void *thread_func(void *arg)
{
    printf("Hello from Thread\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, thread_func, NULL);
    pthread_join(tid, NULL);
    printf("Hello from Main Thread\n");
    return 0;
}
```
## Compilation of Threaded Program
To compile a threaded program:
```
gcc program.c -pthread
```
or
```
gcc program.c -lpthread
```
Recommended:
```
gcc program.c -pthread
```
Reason:
* Enables thread-safe compilation flags.
* Links pthread library automatically.

Example:
```
gcc thread.c -pthread -o thread
```
Run:
```
./thread
```
Output:
```
Hello from Thread
Hello from Main Thread
```
## Internal Working
Execution flow:
```
Main Thread
    |
    |---- pthread_create()
            |
            |---- Child Thread Executes
            |
    |---- pthread_join()
            |
Main Continues
```
## Multiple Threads Example
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Running\n");
    return NULL;
}
int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, task, NULL);
    pthread_create(&t2, NULL, task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    return 0;
}
```
Possible Output:
```
Thread Running
Thread Running
```
(Order may vary because threads execute concurrently.)
## Important Thread Functions
| Function           | Purpose         |
| ------------------ | --------------- |
| `pthread_create()` | Create thread   |
| `pthread_join()`   | Wait for thread |
| `pthread_exit()`   | Exit thread     |
| `pthread_self()`   | Get thread ID   |
| `pthread_cancel()` | Cancel thread   |
## Advantages of Threaded Programs
* Better responsiveness
* Concurrent task execution
* Improved CPU utilization
* Faster execution on multicore processors
* Efficient resource sharing
## Common Mistakes
### 1. Forgetting `pthread_join()`
Problem:
```c
pthread_create(&tid, NULL, func, NULL);
return 0;
```
Program may exit before thread finishes.

Fix:
```c
pthread_join(tid, NULL);
```
### 2. Missing `-pthread`
Problem:
```
gcc program.c
```
Error:
```
undefined reference to pthread_create
```
Fix:
```
gcc program.c -pthread
```

# Creating Threads
In Linux, threads are commonly created using the **POSIX Threads (Pthreads)** library.

POSIX Threads provides the `pthread_create()` function to create new threads.
## What is Thread Creation?
Thread creation means starting a new thread inside a process so that multiple tasks can execute concurrently.

When a thread is created:
* A new execution path begins.
* It runs independently of the main thread.
* It shares the same process resources (memory, files, variables).

Example:
```
Process
   |
   ├── Main Thread
   ├── Thread 1
   └── Thread 2
```
## Function Used: `pthread_create()`
### Syntax
```c 
pthread_create(thread, attr, start_routine, arg);
```
### Prototype
```c
int pthread_create(
    pthread_t *thread,
    const pthread_attr_t *attr,
    void *(*start_routine)(void *),
    void *arg
);
```
### Header File
```c 
#include <pthread.h>
```
## Parameters of `pthread_create()`
### 1. `thread`
```c 
pthread_t *thread
```
* Stores thread ID.
* Used later with functions like `pthread_join()`.

Example:
```c 
pthread_t tid;
```
## 2. `attr`
```c 
const pthread_attr_t *attr
```
* Specifies thread attributes.
* Usually `NULL` for default settings.

Example:
```c 
NULL
```
## 3. `start_routine`
```c 
void *(*start_routine)(void *)
```
* Function executed by the thread.
* Called automatically after thread creation.

Example:
```c 
void *task(void *arg)
{
    printf("Thread Running\n");
    return NULL;
}
```
## 4. `arg`
```c 
void *arg
```
* Data passed to thread function.
* Can be `NULL`.

Example:
```c 
pthread_create(&tid, NULL, task, NULL);
```
## Return Value
`pthread_create()` returns:
| Value    | Meaning                     |
| -------- | --------------------------- |
| `0`      | Thread created successfully |
| Non-zero | Error occurred              |
Example:
```c 
if(pthread_create(&tid, NULL, task, NULL) != 0)
{
    printf("Thread creation failed\n");
}
```
## Basic Example: Creating One Thread
```c 
#include <stdio.h>
#include <pthread.h>
void *display(void *arg)
{
    printf("Hello from Thread\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, display, NULL);
    pthread_join(tid, NULL);
    printf("Main Thread Finished\n");
    return 0;
}
```
### Output
```
Hello from Thread
Main Thread Finished
```
## Explanation
Step-by-step:
1. Main thread starts.
2. `pthread_create()` creates a child thread.
3. Child thread executes `display()`.
4. `pthread_join()` waits for thread completion.
5. Main thread continues.

Execution flow:
```
Main Thread
      |
      |---- pthread_create()
                   |
                   |---- Child Thread Executes
                   |        printf()
                   |
      |---- pthread_join()
      |
Main Ends
```
## Creating Multiple Threads
Example:
```c 
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Running\n");
    return NULL;
}
int main()
{
    pthread_t t1, t2, t3;
    pthread_create(&t1, NULL, task, NULL);
    pthread_create(&t2, NULL, task, NULL);
    pthread_create(&t3, NULL, task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    pthread_join(t3, NULL);
    return 0;
}
```
Possible Output:
```
Thread Running
Thread Running
Thread Running
```
The order may change because threads execute concurrently.
## Creating Threads in a Loop
Example:
```c

    printf("Thread Created\n");

```
Output (order may vary):
```

Thread Created
Thread Created
Thread Created
Thread Created
```
## Passing Data During Thread Creation
Example:

Output:
```
Number = 100
```

# Thread Identity
In multithreaded programming, each thread has a unique identifier called a **Thread ID (Thread Identity)**.

This ID helps in:
* Identifying a thread
* Managing threads
* Comparing threads
* Synchronizing thread execution

In Linux Pthreads, thread identity is represented using:
```c
pthread_t
```
from POSIX Threads.
## What is Thread Identity?
When a thread is created, the system assigns it a unique ID.

Example:
```
Process
   |
   ├── Thread 1 (ID = T1)
   ├── Thread 2 (ID = T2)
   └── Thread 3 (ID = T3)
```
Each thread has a different identity so the operating system and program can distinguish them.
## Data Type Used: `pthread_t`
Thread ID is stored using:
```c
pthread_t
```
Example:
```c
pthread_t tid;
```
When a thread is created:
```c
pthread_create(&tid, NULL, task, NULL);
```
the thread ID is stored in `tid`.
## Getting Thread Identity: `pthread_self()`
### Purpose
Returns the calling thread’s ID.
### Syntax
```c
pthread_self();
```
### Prototype
```c
pthread_t pthread_self(void);
```
* Returns thread ID of the currently executing thread.
* No arguments required.
## Example: Display Thread Identity
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Child Thread ID = %lu\n",
            pthread_self());
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, task, NULL);
    printf("Main Thread ID = %lu\n",
            pthread_self());
    pthread_join(tid, NULL);
    return 0;
}
```
### Possible Output
```
Main Thread ID = 140243491239744
Child Thread ID = 140243482847040
```
The numbers vary for each execution and system.
## Explanation
1. Main thread begins execution.
2. `pthread_create()` creates child thread.
3. `pthread_self()` returns ID of the executing thread.
4. Main thread and child thread print different IDs.

Execution flow:
```
Main Thread (ID1)
       |
       |---- Create Thread
                    |
                    └── Child Thread (ID2)
```
## Comparing Thread IDs: `pthread_equal()`
Thread IDs should not always be compared using `==`.

Use:
```c
pthread_equal()
```
### Syntax
```c
pthread_equal(t1, t2);
```
### Prototype
```c
int pthread_equal(pthread_t t1,
                  pthread_t t2);
```
### Return Value
| Value    | Meaning               |
| -------- | --------------------- |
| Non-zero | Threads are same      |
| 0        | Threads are different |
## Example: Comparing Thread IDs
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    pthread_t t = pthread_self();
    if(pthread_equal(t,
        *(pthread_t *)arg))
    {
        printf("Same Thread\n");
    }
    else
    {
        printf("Different Thread\n");
    }
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL,
                   task, &tid);
    pthread_join(tid, NULL);
    return 0;
}
```
Output:
```
Same Thread
```
## Storing Thread IDs
Thread IDs can be stored in arrays.

Example:
```c
pthread_t tid[5];
```
Create multiple threads:
```c
for(int i = 0; i < 5; i++)
{
    pthread_create(&tid[i],
                   NULL,
                   task,
                   NULL);
}
```
Each array element stores one thread identity.
## Printing Thread ID Safely
Many systems print thread IDs using:
```c
printf("%lu", pthread_self());
```
However, `pthread_t` implementation may differ between systems.

Safer approach:
```c
printf("%lu",
       (unsigned long)pthread_self());
```
Example:
```c
printf("Thread ID = %lu\n",
       (unsigned long)pthread_self());
```
## Example: Multiple Threads with IDs
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread ID = %lu\n",
          (unsigned long)pthread_self());
    return NULL;
}
int main()
{
    pthread_t tid[3];
    for(int i = 0; i < 3; i++)
    {
        pthread_create(&tid[i],
                        NULL,
                        task,
                        NULL);
    }
    for(int i = 0; i < 3; i++)
    {
        pthread_join(tid[i], NULL);
    }
    return 0;
}
```
Possible Output:
```
Thread ID = 140372930963200
Thread ID = 140372922570496
Thread ID = 140372914177792
```
(Order may vary.)
## Important Functions Related to Thread Identity
| Function           | Purpose                     |
| ------------------ | --------------------------- |
| `pthread_self()`   | Get current thread ID       |
| `pthread_equal()`  | Compare thread IDs          |
| `pthread_create()` | Create thread and obtain ID |
## Key Notes
* Every thread has a unique identity.
* Thread ID type is `pthread_t`.
* `pthread_self()` gets current thread ID.
* `pthread_equal()` compares IDs safely.
* Thread IDs help manage and synchronize threads.

# Synchronizing by Joining
In multithreaded programming, **joining** is a synchronization technique where one thread waits for another thread to finish execution.

In Pthreads, synchronization by joining is done using:
```c
pthread_join()
```
from POSIX Threads.
## Why Synchronization is Needed?
Threads run concurrently, so execution order is unpredictable.

Without synchronization:
* Main thread may terminate before child threads finish.
* Shared results may be incomplete.
* Program behavior may become inconsistent.

Example problem:
```
Main Thread Starts
Child Thread Created
Main Thread Ends
```
The child thread may never complete execution.

To avoid this, we use **joining**.
## What is Joining?
Joining means:
> One thread waits until another thread terminates.

Usually:
* Main thread waits for child thread(s)
* Parent thread synchronizes with worker thread(s)

Execution flow:
```
Main Thread
     |
     |---- Create Thread
     |
     |---- Wait (pthread_join)
                    |
                Child Thread Executes
                    |
                Child Thread Ends
     |
Main Continues
```
## Function Used: `pthread_join()`
### Syntax
```c
pthread_join(thread, retval);
```
### Prototype
```c
int pthread_join(
        pthread_t thread,
        void **retval
);
```
## Parameters
### 1. `thread`
```c
pthread_t thread
```
* Thread ID to wait for.
* Usually obtained from `pthread_create()`.

Example:
```c
pthread_t tid;
pthread_create(&tid, NULL, task, NULL);
pthread_join(tid, NULL);
```
## 2. `retval`
```c
void **retval
```
* Used to receive thread return value.
* `NULL` if return value is not needed.

Example:
```c
pthread_join(tid, NULL);
```
## Return Value
| Value    | Meaning |
| -------- | ------- |
| `0`      | Success |
| Non-zero | Error   |
Example:
```c
if(pthread_join(tid, NULL) != 0)
{
    printf("Join Failed\n");
}
```
## Example 1: Joining One Thread
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Child Thread Running\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, task, NULL);
    pthread_join(tid, NULL);
    printf("Main Thread Resumes\n");
    return 0;
}
```
### Output
```
Child Thread Running
Main Thread Resumes
```
### Explanation
1. Main thread creates child thread.
2. Child thread starts execution.
3. Main thread waits at `pthread_join()`.
4. Child thread finishes.
5. Main thread resumes execution.
## Example 2: Without `pthread_join()`
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Child Thread Running\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, task, NULL);
    return 0;
}
```
Possible Output:
```
(no output)
```
or
```
Child Thread Running
```
Reason:
* Main thread exits before child thread finishes.
## Example 3: Joining Multiple Threads
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Running\n");
    return NULL;
}
int main()
{
    pthread_t t1, t2, t3;
    pthread_create(&t1, NULL, task, NULL);
    pthread_create(&t2, NULL, task, NULL);
    pthread_create(&t3, NULL, task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    pthread_join(t3, NULL);
    printf("All Threads Completed\n");
    return 0;
}
```
Possible Output:
```
Thread Running
Thread Running
Thread Running
All Threads Completed
```
The order of thread execution may vary.
## Joining in a Loop
When multiple threads exist:
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Executed\n");
    return NULL;
}
int main()
{
    pthread_t tid[5];
    for(int i = 0; i < 5; i++)
    {
        pthread_create(&tid[i],
                        NULL,
                        task,
                        NULL);
    }
    for(int i = 0; i < 5; i++)
    {
        pthread_join(tid[i],
                     NULL);
    }
    printf("All Threads Finished\n");
    return 0;
}
```
## Getting Return Value from a Thread
Thread function:
```c
void *task(void *arg)
{
    static int value = 100;
    return &value;
}
```
Receiving return value:
```c
void *ret;
pthread_join(tid, &ret);
printf("%d\n", *(int *)ret);
```
Output:
```
100
```
## Behavior of `pthread_join()`
When a thread calls:
```c
pthread_join(tid, NULL);
```
* Calling thread blocks (waits)
* CPU executes other runnable threads
* Control resumes only after target thread terminates

Thread state:
```
Runnable → Running → Waiting (join) → Runnable
```
## Important Rules
### 1. Join Only Once
Wrong:
```c
pthread_join(tid, NULL);
pthread_join(tid, NULL);
```
A thread cannot be joined multiple times.
## 2. Cannot Join Itself
Wrong:
```c
pthread_join(pthread_self(), NULL);
```
This causes deadlock.
## 3. Detached Threads Cannot Be Joined
Detached threads release resources automatically and cannot be joined.
## Join vs Detach
| Join                          | Detach                  |
| ----------------------------- | ----------------------- |
| Waits for thread completion   | Does not wait           |
| Synchronization possible      | Independent execution   |
| Resources released after join | Resources auto released |
## Advantages of Joining
1. Proper synchronization
2. Prevents premature termination
3. Allows retrieval of thread return value
4. Helps coordinate multiple threads
## Common Mistakes
### Forgetting `pthread_join()`
Problem:
```c
pthread_create(&tid,
               NULL,
               task,
               NULL);
return 0;
```
Main thread exits early.

Fix:
```c
pthread_join(tid, NULL);
```

# Detaching Threads
In multithreaded programming, a thread can be created as a **detached thread**, meaning it executes independently and releases its resources automatically when it terminates.

In Pthreads, detaching is done using:
```c 
pthread_detach()
```
from POSIX Threads.
## What is a Detached Thread?
Normally, threads are **joinable**:
* Another thread can wait for them using `pthread_join()`
* Resources are released only after joining

A **detached thread**:
* Runs independently
* Cannot be joined
* Automatically releases resources after termination

Execution idea:
```
Main Thread
      |
      |---- Create Thread
                    |
                    └── Detached Thread
                           |
                        Executes
                           |
                        Terminates
                           |
                  Resources released automatically
```
## Why Detach Threads?
Detaching is useful when:
* You do not need thread return values
* No synchronization is required
* Background work should run independently
* Automatic cleanup is desired

Example:
* Logging thread
* Background monitoring thread
* Notification handler
* Cleanup worker
## Joinable vs Detached Threads
| Joinable Thread           | Detached Thread          |
| ------------------------- | ------------------------ |
| Can be joined             | Cannot be joined         |
| Requires `pthread_join()` | Automatic cleanup        |
| Return value available    | Return value unavailable |
| Synchronization possible  | Independent execution    |
## Function Used: `pthread_detach()`
### Syntax
```c
pthread_detach(thread);
```
### Prototype
```c 
int pthread_detach(pthread_t thread);
```
### Parameter
```c 
pthread_t thread
```
* Thread ID to detach.

Example:
```c
pthread_detach(tid);
```
## Return Value
| Value    | Meaning |
| -------- | ------- |
| `0`      | Success |
| Non-zero | Error   |
Example:
```c 
if(pthread_detach(tid) != 0)
{
    printf("Detach failed\n");
}
```
## Example 1: Detaching a Thread
```c
#include <stdio.h>
#include <unistd.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Detached Thread Running\n");
    sleep(2);
    printf("Detached Thread Finished\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, task, NULL);
    pthread_detach(tid);
    sleep(3);
    printf("Main Thread Ends\n");
    return 0;
}
```
### Output
```
Detached Thread Running
Detached Thread Finished
Main Thread Ends
```
### Explanation
1. Main thread creates child thread.
2. `pthread_detach()` marks it detached.
3. Thread runs independently.
4. After completion, resources are automatically freed.
5. No `pthread_join()` needed.
## Example 2: Error Using `pthread_join()` on Detached Thread
Wrong:
```c
pthread_create(&tid, NULL, task, NULL);
pthread_detach(tid);
pthread_join(tid, NULL);
```
Reason:
* Detached threads cannot be joined.
## Creating Detached Thread Directly
Instead of calling `pthread_detach()` later, a thread can be created detached using thread attributes.

Example:
```c 
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Detached Thread\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_attr_t attr;
    pthread_attr_init(&attr);
    pthread_attr_setdetachstate(
            &attr,
            PTHREAD_CREATE_DETACHED);
    pthread_create(&tid,
                   &attr,
                   task,
                   NULL);
    pthread_attr_destroy(&attr);
    pthread_exit(NULL);
}
```
### Explanation
* `pthread_attr_init()` initializes attributes.
* `pthread_attr_setdetachstate()` sets detached state.
* `PTHREAD_CREATE_DETACHED` creates thread detached.
## `pthread_exit()` in Main Thread
In detached-thread programs:
```c 
pthread_exit(NULL);
```
is often used.

Reason:
* Keeps process alive until detached threads finish.
* Prevents main thread from ending process too early.

Example:
```c 
int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, task, NULL);
    pthread_detach(tid);
    pthread_exit(NULL);
}
```
## Execution Flow
```
Main Thread
     |
     |---- Create Thread
     |
     |---- Detach Thread
                    |
              Thread Executes
                    |
              Thread Terminates
                    |
            Resources Released
```
## When to Use Detached Threads?
Detached threads are best for:
1. Background processing
2. Independent worker tasks
3. Logging systems
4. Event handlers
5. Monitoring services

Example:

A server may create detached threads to handle temporary client requests.
## Important Rules
### 1. Detached Threads Cannot Be Joined
Wrong:
```c 
pthread_detach(tid);
pthread_join(tid, NULL);
```
## 2. Cannot Detach Twice
Wrong:
```c 
pthread_detach(tid);
pthread_detach(tid);
```
May produce an error.
## 3. Detaching Releases Resources Automatically
No need:
```c 
pthread_join(tid, NULL);
```
## Join vs Detach
| Feature          | Join            | Detach        |
| ---------------- | --------------- | ------------- |
| Wait for thread  | Yes             | No            |
| Synchronization  | Yes             | No            |
| Resource cleanup | Manual (`join`) | Automatic     |
| Return value     | Available       | Not available |
## Common Mistake
Problem:
```c 
pthread_create(&tid,
               NULL,
               task,
               NULL);
return 0;
```
The process may terminate before thread finishes.

Fix:

Either:
```c 
pthread_join(tid, NULL);
```
or
```c 
pthread_detach(tid);
pthread_exit(NULL);
```

# Stopping Threads
Stopping a thread means terminating its execution. In Pthreads, a thread can stop in several ways:
1. Returning from the thread function
2. Calling `pthread_exit()`
3. Being cancelled using `pthread_cancel()`
4. Process termination (all threads stop)

These mechanisms are provided by POSIX Threads.
## 1. Stopping by Returning from Thread Function
The simplest way to stop a thread is to return from the thread routine.

Example:
```c 
void *task(void *arg)
{
    printf("Thread Running\n");
    return NULL;
}
```
Explanation:
* Thread starts execution.
* Function completes.
* Thread terminates automatically.

Example program:
```c 
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Finished\n");
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    pthread_join(tid, NULL);
    return 0;
}
```
Output:
```
Thread Finished
```
## 2. Stopping Using `pthread_exit()`
A thread can terminate explicitly using:
```c 
pthread_exit()
```
### Syntax
```c 
pthread_exit(value);
```
### Prototype
```c 
void pthread_exit(void *retval);
```
* Stops the calling thread immediately.
* Can return a value to another thread using `pthread_join()`.

Example:
```c 
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    printf("Thread Running\n");
    pthread_exit(NULL);
}
int main()
{
    pthread_t tid;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    pthread_join(tid, NULL);
    return 0;
}
```
Output:
```
Thread Running
```
## Returning a Value with `pthread_exit()`
Example:
```c 
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    static int result = 100;
    pthread_exit(&result);
}
int main()
{
    pthread_t tid;
    void *ret;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    pthread_join(tid, &ret);
    printf("Returned Value = %d\n",
           *(int *)ret);
    return 0;
}
```
Output:
```
Returned Value = 100
```
## 3. Stopping Using `pthread_cancel()`
One thread can request termination of another thread using:
```c 
pthread_cancel()
```
### Syntax
```c 
pthread_cancel(thread);
```
### Prototype
```c 
int pthread_cancel(pthread_t thread);
```
### Parameter
```c
pthread_t thread
```
* Thread ID to cancel.
## Example: Cancelling a Thread
```c
#include <stdio.h>
#include <unistd.h>
#include <pthread.h>
void *task(void *arg)
{
    while(1)
    {
        printf("Running...\n");
        sleep(1);
    }
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    sleep(3);
    pthread_cancel(tid);
    pthread_join(tid, NULL);
    printf("Thread Cancelled\n");
    return 0;
}
```
Possible Output:
```
Running...
Running...
Running...
Thread Cancelled
```
### Explanation
1. Thread runs continuously.
2. Main thread waits for 3 seconds.
3. `pthread_cancel()` sends cancellation request.
4. Thread terminates.
5. `pthread_join()` synchronizes.
## Cancellation is a Request
Important:
```c 
pthread_cancel(tid);
```
does **not guarantee immediate termination**.

It sends a cancellation request.

The target thread stops only at a **cancellation point**, such as:
* `sleep()`
* `read()`
* `write()`
* `pthread_join()`

Example:
```
Thread Running
      |
Cancellation Request
      |
Cancellation Point Reached
      |
Thread Stops
```
## 4. Process Termination Stops All Threads
If `main()` exits:
```c 
return 0;
```
the entire process terminates.

All threads stop immediately.

Example:
```c
#include <stdio.h>
#include <pthread.h>
void *task(void *arg)
{
    while(1)
    {
        printf("Thread Running\n");
    }
    return NULL;
}
int main()
{
    pthread_t tid;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    return 0;
}
```
Possible Output:
```
(no output)
```
Reason:
* Process exits before thread continues.
## `pthread_exit()` vs `return`
Inside thread:

Both are valid:
```c 
return NULL;
```
or
```c 
pthread_exit(NULL);
```
Difference:
* `return` exits at function end
* `pthread_exit()` exits immediately

Example:
```c 
void *task(void *arg)
{
    printf("Start\n");
    pthread_exit(NULL);
    printf("Never executes\n");
}
```
Output:
```
Start
```
## Main Thread and `pthread_exit()`
Sometimes main thread should not terminate process:

Example:
```c 
int main()
{
    pthread_t tid;
    pthread_create(&tid,
                   NULL,
                   task,
                   NULL);
    pthread_exit(NULL);
}
```
Meaning:
* Main thread exits
* Process stays alive until other threads finish
## Summary Table
| Method             | Description                    |
| ------------------ | ------------------------------ |
| `return`           | Thread ends normally           |
| `pthread_exit()`   | Explicitly terminate thread    |
| `pthread_cancel()` | Request another thread to stop |
| Process exit       | All threads terminate          |

# Synchronizing with Mutexes
When multiple threads access shared data simultaneously, problems such as **race conditions** and inconsistent results can occur.

To prevent this, threads are synchronized using a **mutex** (Mutual Exclusion).

In Pthreads, mutexes are provided by:

POSIX Threads
## What is a Mutex?
A **mutex** (Mutual Exclusion lock) is a synchronization mechanism that allows **only one thread at a time** to access a shared resource.

Think of it as a lock on a room:
* Thread enters → locks the room
* Other threads wait outside
* Thread finishes → unlocks the room

Execution idea:
```
Thread 1 ---- Lock ----> Shared Resource ---- Unlock
Thread 2 -------- waits --------> Lock ----> Resource
Thread 3 -------- waits --------> Lock ----> Resource
```
## Why Mutexes Are Needed?
Consider multiple threads updating the same variable.

Example without synchronization:
```c 
counter++;
```
Suppose:
```
counter = 5
```
Two threads execute:
```
Thread 1 reads 5
Thread 2 reads 5
Thread 1 writes 6
Thread 2 writes 6
```
Expected:
```
7
```
Actual:
```
6
```
This problem is called a **race condition**.

Mutexes solve this issue.
## Mutex Operations
Three common operations:
1. Initialize mutex
2. Lock mutex
3. Unlock mutex
4. Destroy mutex

Functions:
```c 
pthread_mutex_init()
pthread_mutex_lock()
pthread_mutex_unlock()
pthread_mutex_destroy()
```
## Declaring a Mutex
Syntax:
```c 
pthread_mutex_t mutex;
```
Example:
```c 
pthread_mutex_t lock;
```
## Initializing a Mutex
### Method 1: Static Initialization
```c
pthread_mutex_t lock =
        PTHREAD_MUTEX_INITIALIZER;
```
Simplest method.
## Method 2: Dynamic Initialization
Syntax:
```c 
pthread_mutex_init(&mutex, NULL);
```
Example:
```c 
pthread_mutex_t lock;
pthread_mutex_init(&lock, NULL);
```
## Locking a Mutex
Syntax:
```c 
pthread_mutex_lock(&mutex);
```
Meaning:
* If mutex is free → thread gets lock
* If locked → thread waits

Example:
```c 
pthread_mutex_lock(&lock);
```
## Unlocking a Mutex
Syntax:
```c 
pthread_mutex_unlock(&mutex);
```
Example:
```c 
pthread_mutex_unlock(&lock);
```
Unlock allows waiting thread to proceed.
## Destroying a Mutex
Syntax:
```c 
pthread_mutex_destroy(&mutex);
```
Example:
```c 
pthread_mutex_destroy(&lock);
```
Used after mutex is no longer required.
## Example Without Mutex (Race Condition)
```c 
#include <stdio.h>
#include <pthread.h>
int counter = 0;
void *task(void *arg)
{
    for(int i = 0; i < 100000; i++)
    {
        counter++;
    }
    return NULL;
}
int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, task, NULL);
    pthread_create(&t2, NULL, task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Counter = %d\n", counter);
    return 0;
}
```
Expected:
```
Counter = 200000
```
Possible Actual Output:
```
Counter = 163542
```
Reason:

Race condition.
## Example With Mutex (Correct Synchronization)
```c 
#include <stdio.h>
#include <pthread.h>
int counter = 0;
pthread_mutex_t lock =
        PTHREAD_MUTEX_INITIALIZER;
void *task(void *arg)
{
    for(int i = 0; i < 100000; i++)
    {
        pthread_mutex_lock(&lock);
        counter++;
        pthread_mutex_unlock(&lock);
    }
    return NULL;
}
int main()
{
    pthread_t t1, t2;
    pthread_create(&t1, NULL, task, NULL);
    pthread_create(&t2, NULL, task, NULL);
    pthread_join(t1, NULL);
    pthread_join(t2, NULL);
    printf("Counter = %d\n", counter);
    pthread_mutex_destroy(&lock);
    return 0;
}
```
Output:
```
Counter = 200000
```
## Critical Section
The code protected by a mutex is called a **critical section**.

Example:
```c 
pthread_mutex_lock(&lock);
counter++;
pthread_mutex_unlock(&lock);
```
Only one thread executes this section at a time.
## Execution Flow
```
Thread 1 → Lock → Critical Section → Unlock
Thread 2 → Wait → Lock → Critical Section → Unlock
```
## Common Mistakes
### 1. Forgetting Unlock
Wrong:
```c 
pthread_mutex_lock(&lock);
counter++;
```
Problem:

Other threads block forever.

Correct:
```c 
pthread_mutex_unlock(&lock);
```
## 2. Locking Twice (Deadlock)
Wrong:
```c 
pthread_mutex_lock(&lock);
pthread_mutex_lock(&lock);
```
May cause deadlock.
## 3. Destroying Before Use Ends
Wrong:
```c 
pthread_mutex_destroy(&lock);
```
before threads finish.

Correct:

Destroy after joins.
## Mutex vs Join
| Mutex                    | Join                         |
| ------------------------ | ---------------------------- |
| Protects shared data     | Waits for thread completion  |
| Prevents race conditions | Synchronizes execution order |
| Used during execution    | Used after execution         |
## Advantages of Mutexes
1. Prevent race conditions
2. Protect shared resources
3. Maintain data consistency
4. Ensure safe concurrent access
## Disadvantages
1. Performance overhead
2. Deadlock possibility
3. Improper locking causes bugs

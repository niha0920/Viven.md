# When Does Time Begin?
In **Time Functions** (in C/Linux system programming), **time begins at a fixed reference point called the Epoch**.

For most Unix/Linux systems, time begins at:
- **January 1, 1970, 00:00:00 UTC** (Coordinated Universal Time)

This is called the **Unix Epoch**.

Unix epoch
## Why is this important?
Many time functions count time as the **number of seconds elapsed since the Epoch**.

For example:
```c
#include <stdio.h>
#include <time.h>
int main() {
    time_t t;
    time(&t);
    printf("%ld\n", t);
    return 0;
}
```
Here:
* `time()` returns the **current time in seconds since Jan 1, 1970 (UTC)**.
* `time_t` stores this value.

Example output:
```
1747800000
```
That means approximately **1,747,800,000 seconds have passed since 1 Jan 1970**.

You can think of it like this:
```
1970 (Epoch) -----------------------> Current Time
        counting seconds continuously
```
Common time-related functions in C/Linux:
* `time()` → current time in seconds since epoch
* `ctime()` → converts time to readable string
* `localtime()` → converts epoch time to local time
* `gmtime()` → converts epoch time to UTC format
* `mktime()` → converts broken-down time to epoch seconds

In Linux system programming, whenever you hear **“When does time begin?”**, the answer is usually:

> Time begins from the Unix Epoch: **January 1, 1970, 00:00:00 UTC**.

# Time Data Types
In C/Linux **Time Functions**, there are a few important **time data types** used to store and manipulate time values.
## 1. `time_t`
Stores **calendar time** (seconds elapsed since the Unix Epoch).
```c 
time_t t;
```
* Defined in `<time.h>`
* Usually stores **seconds since Jan 1, 1970, 00:00:00 UTC**
* Used with `time()`

Example:
```c
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    time(&now);
    printf("%ld\n", now);
    return 0;
}
```
Example output:
```
1747805000
```
## 2. `struct tm`
Stores **broken-down time** (human-readable form).

It breaks time into:
* seconds
* minutes
* hours
* day
* month
* year

Structure:
```c 
struct tm {
    int tm_sec;   // seconds (0-60)
    int tm_min;   // minutes (0-59)
    int tm_hour;  // hours (0-23)
    int tm_mday;  // day of month (1-31)
    int tm_mon;   // month (0-11)
    int tm_year;  // years since 1900
    int tm_wday;  // day since Sunday (0-6)
    int tm_yday;  // day since Jan 1 (0-365)
    int tm_isdst; // daylight saving flag
};
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    struct tm *ptr;
    time(&now);
    ptr = localtime(&now);
    printf("Hour = %d\n", ptr->tm_hour);
    printf("Minute = %d\n", ptr->tm_min);
    printf("Second = %d\n", ptr->tm_sec);
    return 0;
}
```
## 3. `clock_t`
Used to measure **processor (CPU) time** consumed by a program.
```c 
clock_t start, end;
```
Used with `clock()`.

Example:
```c
#include <stdio.h>
#include <time.h>
int main() {
    clock_t start, end;
    start = clock();
    for(int i = 0; i < 1000000; i++);
    end = clock();
    printf("CPU Time = %lf\n",
           (double)(end - start) / CLOCKS_PER_SEC);
    return 0;
}
```
Here:
* `clock()` → returns CPU ticks
* `CLOCKS_PER_SEC` → converts ticks into seconds
## Summary Table
| Data Type   |                              Purpose |
| ----------- | -----------------------------------: |
| `time_t`    |   Stores time in seconds since epoch |
| `struct tm` | Stores readable date/time components |
| `clock_t`   |            Stores CPU execution time |

So in Linux system programming, the **main time data types** are:

`time_t`, `struct tm`, and `clock_t`.

# Determining Real Time
**Real time** means the **current calendar time** of the system (date and clock time).

In C, real time is determined using the `time()` function from `<time.h>`.
## Syntax
```c 
time_t time(time_t *tloc);
```
* Returns current time in **seconds since the Unix Epoch**
* Epoch = **Jan 1, 1970, 00:00:00 UTC**
### Example Program
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t current_time;
    time(&current_time);
    printf("Current Time = %ld\n", current_time);
    return 0;
}
```
Example output:
```
Current Time = 1747805000
```
This number represents **seconds elapsed since Jan 1, 1970**.
## Getting Human Readable Real Time
To display time in readable format, use `ctime()`.
```c
#include <stdio.h>
#include <time.h>
int main() {
    time_t current_time;
    time(&current_time);
    printf("Real Time = %s", ctime(&current_time));
    return 0;
}
```
Example output:
```
Real Time = Thu May 21 12:30:45 2026
```
### How it works
1. `time()` gets current system time in seconds.
2. `ctime()` converts it into readable format.

Flow:
```
Current System Clock
        ↓
      time()
        ↓
   seconds since epoch
        ↓
      ctime()
        ↓
Human readable time
```
### Important Functions for Determining Real Time
| Function      |                           Purpose |
| ------------- | --------------------------------: |
| `time()`      |        Gets current calendar time |
| `ctime()`     |  Converts time to readable string |
| `localtime()` | Converts to local timezone format |
| `gmtime()`    |          Converts to UTC/GMT time |
So, **determining real time** in Linux programming usually means using **`time()` to get current system time**, and optionally `ctime()` or `localtime()` to display it in readable form.

# Converting time_t
`time_t` stores **time as seconds since the Unix Epoch** (Jan 1, 1970).

To make it useful for humans, we convert it into readable date/time formats.

Common conversions of `time_t`:
## 1. `ctime()` – Convert `time_t` to Readable String
Converts `time_t` into a human-readable string.
### Syntax
```c 
char *ctime(const time_t *timer);
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    time(&now);
    printf("%s", ctime(&now));
    return 0;
}
```
Example output:
```
Thu May 21 12:30:45 2026
```
## 2. `localtime()` – Convert `time_t` to Local Time
Converts `time_t` into `struct tm` based on **system local timezone**.
### Syntax
```c 
struct tm *localtime(const time_t *timer);
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    struct tm *ptr;
    time(&now);
    ptr = localtime(&now);
    printf("Hour   = %d\n", ptr->tm_hour);
    printf("Minute = %d\n", ptr->tm_min);
    printf("Day    = %d\n", ptr->tm_mday);
    printf("Month  = %d\n", ptr->tm_mon + 1);
    printf("Year   = %d\n", ptr->tm_year + 1900);
    return 0;
}
```
Output (example):
```
Hour   = 12
Minute = 30
Day    = 21
Month  = 5
Year   = 2026
```
**Important:**
* `tm_mon` → range `0–11`, so add `1`
* `tm_year` → years since `1900`, so add `1900`
## 3. `gmtime()` – Convert `time_t` to UTC/GMT Time
Converts `time_t` into UTC format.
### Syntax
```c 
struct tm *gmtime(const time_t *timer);
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    struct tm *ptr;
    time(&now);
    ptr = gmtime(&now);
    printf("UTC Hour = %d\n", ptr->tm_hour);
    return 0;
}
```
## 4. `mktime()` – Convert `struct tm` to `time_t`
Performs reverse conversion.
### Syntax
```c 
time_t mktime(struct tm *timeptr);
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    struct tm t = {0};
    t.tm_year = 2026 - 1900;
    t.tm_mon  = 4;   // May
    t.tm_mday = 21;
    time_t result = mktime(&t);
    printf("%ld\n", result);
    return 0;
}
```
## Summary Table
| Function      | Conversion                        |
| ------------- | --------------------------------- |
| `ctime()`     | `time_t → string`                 |
| `localtime()` | `time_t → struct tm (local time)` |
| `gmtime()`    | `time_t → struct tm (UTC)`        |
| `mktime()`    | `struct tm → time_t`              |
So, **converting `time_t`** means changing epoch seconds into readable formats using functions like `ctime()`, `localtime()`, `gmtime()`, or converting back using `mktime()`.

# Converting tm Structure
`struct tm` stores **broken-down time** (human-readable format such as year, month, day, hour, minute, second).

Sometimes we need to convert `struct tm` into:
* `time_t` (epoch seconds)
* formatted string representation
## 1. `mktime()` – Convert `struct tm` → `time_t`
This is the **most common conversion**.
### Syntax
```c 
time_t mktime(struct tm *timeptr);
```
It converts a `struct tm` object into **seconds since the Unix Epoch**.

Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    struct tm t = {0};
    time_t result;
    t.tm_year = 2026 - 1900; // years since 1900
    t.tm_mon  = 4;           // May (0-11)
    t.tm_mday = 21;
    t.tm_hour = 10;
    t.tm_min  = 30;
    t.tm_sec  = 45;
    result = mktime(&t);
    printf("time_t value = %ld\n", result);
    return 0;
}
```
Example output:
```
time_t value = 1779359445
```
**Important:**
* `tm_year` = current year − `1900`
* `tm_mon` = `0–11` (January = 0)
## 2. `asctime()` – Convert `struct tm` → Readable String
Converts `struct tm` into a human-readable string.
### Syntax
```c 
char *asctime(const struct tm *timeptr);
``
Example:
```c 
#include <stdio.h>
#include <time.h>

    struct tm t = {0};

    t.tm_mon  = 4;
    t.tm_mday = 21;

    t.tm_sec  = 45;
    printf("%s", asctime(&t));
    return 0;
}
```
Example output:
```
Thu May 21 10:30:45 2026
```
## Flow of Conversion
```
struct tm
   ↓ mktime()
time_t (epoch seconds)
struct tm
   ↓ asctime()
Readable String
```
### Summary Table
| Function    | Conversion                    |
| ----------- | ----------------------------- |
| `mktime()`  | `struct tm → time_t`          |
| `asctime()` | `struct tm → readable string` |
So, **converting `struct tm`** mainly means:
* `mktime()` → convert to `time_t`
* `asctime()` → convert to readable text

# Process Time
**Process time** is the amount of **CPU time used by a process** while executing.

It is **not real (wall-clock) time**.

For example:
* If a program runs for **10 seconds** but actually uses CPU for only **2 seconds**, then:
  * Real time = **10 sec**
  * Process time = **2 sec**

Process time measures **CPU usage of the program**.
## Determining Process Time
In C, process time is determined using:
```c
clock()
```
Function from `<time.h>`.
### Syntax
```c 
clock_t clock(void);
```
* Returns processor time consumed by the program
* Value is in **clock ticks**

To convert ticks into seconds:
```c
CPU Time = ticks / CLOCKS_PER_SEC
```
## Example Program
```c 
#include <stdio.h>
#include <time.h>
int main() {
    clock_t start, end;
    double cpu_time;
    start = clock();
    for (long int i = 0; i < 100000000; i++);
    end = clock();
    cpu_time =
        (double)(end - start) / CLOCKS_PER_SEC;
    printf("CPU Time = %lf seconds\n",
           cpu_time);
    return 0;
}
```
Example output:
```
CPU Time = 0.245000 seconds
```
## How It Works
1. `clock()` at start → stores beginning CPU ticks
2. Execute program
3. `clock()` at end → stores ending CPU ticks
4. Difference gives CPU time consumed

Formula:
```
CPU Time =
(end_clock - start_clock)
/ CLOCKS_PER_SEC
```
## Real Time vs Process Time
| Feature           |           Real Time |  Process Time |
| ----------------- | ------------------: | ------------: |
| Meaning           | Actual elapsed time | CPU time used |
| Function          |            `time()` |     `clock()` |
| Unit              | Seconds since epoch |   Clock ticks |
| Includes waiting? |                 Yes |            No |
Example:
```
Program runs for 10 sec
CPU actively works for 2 sec
Real Time    = 10 sec
Process Time = 2 sec
```
So, **process time** is the **CPU execution time consumed by a process**, measured using `clock()` and stored in `clock_t`.

# Time arithmetic
**Time arithmetic** means performing **mathematical operations on time values**, such as:
* finding time difference
* adding/subtracting time
* comparing times

In C, this is usually done using `time_t`.

Since `time_t` stores **seconds since the Unix Epoch**, arithmetic becomes simple numeric operations.
## 1. Finding Difference Between Two Times — `difftime()`
Used to calculate difference between two `time_t` values.
### Syntax
```c 
double difftime(time_t end, time_t beginning);
```
Returns:
```
end - beginning
```
(in seconds)

Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t start, end;
    time(&start);
    for(long int i = 0; i < 1000000000; i++);
    time(&end);
    printf("Time Difference = %.2lf seconds\n",
           difftime(end, start));
    return 0;
}
```
Example output:
```
Time Difference = 3.00 seconds
```
## 2. Adding Time
Since `time_t` stores seconds, we can directly add seconds.

Example: add **1 hour**
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    time(&now);
    now = now + 3600;
    printf("%s", ctime(&now));
    return 0;
}
```
Here:
```
3600 sec = 1 hour
```
## 3. Subtracting Time
Example: subtract **30 minutes**
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t now;
    time(&now);
    now = now - 1800;
    printf("%s", ctime(&now));
    return 0;
}
```
Here:
```
1800 sec = 30 minutes
```
## 4. Comparing Time
You can compare two `time_t` values directly.

Example:
```c 
if (t1 > t2)
    printf("t1 is later\n");
else
    printf("t2 is later\n");
```
Because `time_t` behaves like an integer storing seconds.
## Common Time Conversion Values
```
1 minute = 60 sec
1 hour   = 3600 sec
1 day    = 86400 sec
```
## Summary Table
| Operation                | Method             |
| ------------------------ | ------------------ |
| Difference between times | `difftime()`       |
| Add time                 | `time_t + seconds` |
| Subtract time            | `time_t - seconds` |
| Compare time             | `>`, `<`, `==`     |
So, **time arithmetic** means performing calculations on `time_t` values to measure, add, subtract, or compare time.

# Second Resolution Timers
A **second resolution timer** measures time with **precision in seconds**.

It cannot measure milliseconds or microseconds — only whole seconds.

In C/Linux, second-resolution timing is commonly done using:
```c 
time()
```
Function from `<time.h>`.
## Why called “Second Resolution”?
Because the timer value changes **once every second**.

Example:
```
10:20:30
10:20:31
10:20:32
```
It does **not** show fractions of a second.
## `time()` Function
### Syntax
```c 
time_t time(time_t *tloc);
```
Returns:
* current system time
* measured in **seconds since Unix Epoch**
### Example Program
```c 
#include <stdio.h>
#include <time.h>
int main() {
    time_t start, end;
    time(&start);
    printf("Waiting for 5 seconds...\n");
    while (1) {
        time(&end);
        if (difftime(end, start) >= 5)
            break;
    }
    printf("5 seconds completed\n");
    return 0;
}
```
### Output
```
Waiting for 5 seconds...
5 seconds completed
```
## How It Works
1. `time()` gets start time
2. Program continuously checks current time
3. `difftime()` calculates difference
4. Stops after required seconds

Flow:
```
start time
    ↓
time()
    ↓
check elapsed seconds
    ↓
stop after target time
```
## Characteristics of Second Resolution Timers
| Feature          | Description                                 |
| ---------------- | ------------------------------------------- |
| Precision        | Seconds                                     |
| Function used    | `time()`                                    |
| Data type        | `time_t`                                    |
| Suitable for     | Delays of seconds, elapsed time measurement |
| Not suitable for | Milliseconds/microseconds                   |
Example uses:
* timeout after 10 seconds
* session timing
* execution duration in seconds

So, a **second resolution timer** is a timer that measures time in **whole seconds**, typically implemented using `time()` and `time_t` in Linux system programming.

# Fine-Grained Timers
**Fine-grained timers** measure time with **higher precision than seconds**, such as:
* milliseconds (**ms**) = `1/1000` second
* microseconds (**µs**) = `1/1,000,000` second
* nanoseconds (**ns**) = `1/1,000,000,000` second

They are used when second resolution is not accurate enough.

Examples:
* measuring execution time of small code sections
* embedded/Linux real-time applications
* benchmarking performance
* precise delays
## 1. `gettimeofday()` – Microsecond Resolution
Used to get time with **microsecond precision**.

Header:
```c 
#include <sys/time.h>
```
### Syntax
```c 
int gettimeofday(struct timeval *tv,
                 struct timezone *tz);
```
Uses `struct timeval`:
```c 
struct timeval {
    long tv_sec;   // seconds
    long tv_usec;  // microseconds
};
```
Example:
```c 
#include <stdio.h>
#include <sys/time.h>
int main() {
    struct timeval t;
    gettimeofday(&t, NULL);
    printf("Seconds      = %ld\n", t.tv_sec);
    printf("Microseconds = %ld\n", t.tv_usec);
    return 0;
}
```
Example output:
```
Seconds      = 1747805000
Microseconds = 452311
```
## 2. `clock_gettime()` – Nanosecond Resolution
Used for **high precision timing**.

Header:
```c 
#include <time.h>
```
### Syntax
```c 
int clock_gettime(clockid_t clk_id,
                  struct timespec *tp);
```
Uses `struct timespec`:
```c 
struct timespec {
    time_t tv_sec; // seconds
    long tv_nsec;  // nanoseconds
};
```
Example:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    struct timespec t;
    clock_gettime(CLOCK_REALTIME, &t);
    printf("Seconds     = %ld\n", t.tv_sec);
    printf("Nanoseconds = %ld\n", t.tv_nsec);
    return 0;
}
```
Example output:
```
Seconds     = 1747805000
Nanoseconds = 245678921
```
## Measuring Execution Time
Example using `clock_gettime()`:
```c 
#include <stdio.h>
#include <time.h>
int main() {
    struct timespec start, end;
    double elapsed;
    clock_gettime(CLOCK_REALTIME, &start);
    for(long int i = 0; i < 100000000; i++);
    clock_gettime(CLOCK_REALTIME, &end);
    elapsed =
        (end.tv_sec - start.tv_sec) +
        (end.tv_nsec - start.tv_nsec) / 1e9;
    printf("Elapsed Time = %lf seconds\n",
           elapsed);
    return 0;
}
```
## Second Resolution vs Fine-Grained Timers
| Feature      | Second Resolution |                        Fine-Grained |
| ------------ | ----------------: | ----------------------------------: |
| Precision    |           Seconds |                        ms / µs / ns |
| Function     |          `time()` | `gettimeofday()`, `clock_gettime()` |
| Suitable for |       Long delays |                      Precise timing |
### Summary
| Function          | Precision    |
| ----------------- | ------------ |
| `time()`          | Seconds      |
| `gettimeofday()`  | Microseconds |
| `clock_gettime()` | Nanoseconds  |
So, **fine-grained timers** are timers used for **high-precision time measurement**, typically implemented using `gettimeofday()` or `clock_gettime()` in Linux programming.

# Real Time Clock (RTC)
A **Real Time Clock (RTC)** is a **hardware clock** that keeps track of the current date and time **even when the computer is powered OFF**.

It works using a **small battery (CMOS battery)** on the motherboard.

Real-Time Clock (RTC)
## Why RTC is Needed?
Normally, when a system shuts down, RAM loses power. Without RTC, the system would forget time.

RTC continues running and stores:
* Year
* Month
* Date
* Hour
* Minute
* Second

When the system boots:
```
RTC (Hardware Clock)
        ↓
Kernel reads RTC time
        ↓
System Clock initialized
        ↓
Programs use system time
```
## RTC vs System Clock
| Feature                   | RTC            | System Clock   |
| ------------------------- | -------------- | -------------- |
| Type                      | Hardware clock | Software clock |
| Runs when power OFF       | Yes            | No             |
| Powered by battery        | Yes            | No             |
| Used during boot          | Yes            | No             |
| Updated while system runs | Usually no     | Yes            |
Example:
```
Laptop switched OFF for 2 days
        ↓
RTC continues counting time
        ↓
Power ON
        ↓
OS reads RTC and restores time
```
## Accessing RTC in Linux
Linux exposes RTC device as:
```
/dev/rtc
```
or
```
/dev/rtc0
```
You can read RTC time using commands like:
```
date
```
or
```
hwclock
```
`hwclock` reads the hardware clock (RTC).

Example:
```
hwclock
```
Output:
```
2026-05-21 12:45:10+05:30
```
## RTC in C Programming
RTC time can be accessed using Linux device interfaces (`/dev/rtc`).

Simple example:
```c 
#include <stdio.h>
#include <fcntl.h>
#include <unistd.h>
int main() {
    int fd;
    fd = open("/dev/rtc", O_RDONLY);
    if (fd < 0) {
        printf("RTC open failed\n");
        return 1;
    }
    printf("RTC device opened\n");
    close(fd);
    return 0;
}
```
## Important Points
* RTC is a **hardware clock**
* Keeps time even after shutdown
* Battery-backed
* Used to initialize system time at boot
* Linux device: `/dev/rtc` or `/dev/rtc0`

So, **RTC (Real Time Clock)** is the **hardware clock that maintains date and time even when the system is powered off**.
